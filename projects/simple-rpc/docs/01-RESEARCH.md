# 简易 RPC 框架 - 市场调研

## 1. RPC 概述

### 1.1 什么是 RPC

RPC（Remote Procedure Call，远程过程调用）是一种计算机通信协议，允许程序调用另一个地址空间（通常是共享网络的另一台机器上）的过程或函数，而不需要显式编写网络通信代码。

### 1.2 RPC 核心原理

```
客户端 → 序列化 → 网络传输 → 反序列化 → 服务端处理
```

1. **客户端调用**：客户端调用本地代理方法
2. **序列化**：将方法名、参数等信息序列化为字节流
3. **网络传输**：通过网络发送到服务端
4. **反序列化**：服务端接收并反序列化请求
5. **服务处理**：服务端执行实际方法
6. **返回结果**：将结果序列化并返回给客户端

### 1.3 RPC 与 REST 对比

| 特性 | RPC | REST |
|------|-----|------|
| 协议 | 通常使用 TCP | HTTP |
| 性能 | 更高 | 较低 |
| 耦合度 | 较高 | 较低 |
| 适用场景 | 内部服务调用 | 公开 API |
| 序列化 | 二进制（Protobuf） | 文本（JSON） |

## 2. 主流 RPC 框架

### 2.1 gRPC

- **开发公司**：Google
- **协议**：HTTP/2
- **序列化**：Protocol Buffers
- **特点**：
  - 高性能
  - 支持多种语言
  - 流式传输
  - 拦截器机制

### 2.2 Thrift

- **开发公司**：Facebook
- **协议**：自定义二进制协议
- **序列化**：Thrift IDL
- **特点**：
  - 跨语言支持
  - 代码生成
  - 多种传输层

### 2.3 Dubbo

- **开发公司**：阿里巴巴
- **协议**：自定义协议
- **序列化**：Hessian2、JSON 等
- **特点**：
  - 服务治理
  - 负载均衡
  - 服务发现
  - 集群容错

### 2.4 JSON-RPC

- **协议**：HTTP/HTTPS
- **序列化**：JSON
- **特点**：
  - 简单易用
  - 跨语言
  - 人类可读

## 3. 序列化技术

### 3.1 Protocol Buffers

- **开发公司**：Google
- **特点**：
  - 高效的二进制格式
  - 跨语言支持
  - 向后兼容
  - 代码生成

### 3.2 JSON

- **特点**：
  - 人类可读
  - 广泛支持
  - 轻量级
  - 无模式

### 3.3 MessagePack

- **特点**：
  - 二进制格式
  - 高效紧凑
  - 跨语言支持

### 3.4 Avro

- **开发公司**：Apache
- **特点**：
  - 动态模式
  - 高效序列化
  - 适合大数据场景

## 4. 服务注册与发现

### 4.1 核心概念

- **服务注册**：服务启动时将自身信息注册到注册中心
- **服务发现**：客户端从注册中心获取服务实例列表
- **健康检查**：定期检测服务实例是否可用
- **负载均衡**：在多个实例间分配请求

### 4.2 主流注册中心

| 注册中心 | 开发公司 | 一致性 | 特点 |
|----------|----------|--------|------|
| ZooKeeper | Apache | 强一致性 | 成熟稳定 |
| Consul | HashiCorp | 最终一致性 | 多数据中心支持 |
| etcd | CoreOS | 强一致性 | Kubernetes 使用 |
| Nacos | Alibaba | 最终一致性 | 支持配置管理 |

## 5. 负载均衡策略

### 5.1 轮询（Round Robin）

按顺序将请求分配给每个服务器。

### 5.2 随机（Random）

随机选择一个服务器。

### 5.3 加权轮询（Weighted Round Robin）

根据服务器权重分配请求。

### 5.4 最少连接（Least Connections）

选择当前连接数最少的服务器。

### 5.5 IP Hash

根据客户端 IP 地址哈希选择服务器。

## 6. 超时与重试策略

### 6.1 超时类型

- **连接超时**：建立连接的超时时间
- **读取超时**：等待响应的超时时间
- **写入超时**：发送请求的超时时间

### 6.2 重试策略

- **固定间隔重试**：每次重试间隔固定
- **指数退避重试**：重试间隔指数增长
- **随机退避重试**：重试间隔随机

### 6.3 熔断器

当错误率达到阈值时，熔断器打开，停止调用服务，一段时间后进入半开状态，尝试恢复。
