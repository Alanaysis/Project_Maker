# 需求分析

## 1. 用户画像

### 1.1 主要用户：游戏后端开发者

| 特征 | 描述 |
|------|------|
| 技能水平 | 中级开发者，有 Go/C++ 基础 |
| 学习目标 | 理解分布式游戏服务器核心原理 |
| 使用场景 | 学习、面试准备、技术研究 |
| 痛点 | 缺乏从零实现的教程，现有框架太复杂 |

### 1.2 次要用户：游戏独立开发者

| 特征 | 描述 |
|------|------|
| 技能水平 | 初中级，有游戏开发经验 |
| 学习目标 | 快速搭建多人游戏后端 |
| 使用场景 | 原型开发、小型多人游戏 |
| 痛点 | 现有框架学习成本高，不理解底层原理 |

## 2. 功能需求

### 2.1 核心功能 (P0)

| 编号 | 功能 | 描述 |
|------|------|------|
| F-001 | 玩家连接管理 | 支持玩家连接、断开、重连 |
| F-002 | 玩家移动同步 | 实时同步玩家位置和方向 |
| F-003 | 攻击动作同步 | 同步玩家攻击动作和伤害计算 |
| F-004 | 客户端预测 | 本地预测移动和攻击结果 |
| F-005 | 服务器校正 | 状态不一致时的平滑校正 |
| F-006 | 状态快照 | 定期发送世界状态快照 |

### 2.2 重要功能 (P1)

| 编号 | 功能 | 描述 |
|------|------|------|
| F-007 | 一致性哈希 | 玩家到服务器的分布式分配 |
| F-008 | 多服务器支持 | 支持多个游戏服务器实例 |
| F-009 | 负载均衡 | 基于玩家数量的动态负载均衡 |
| F-010 | 延迟补偿 | 基于 RTT 的延迟补偿机制 |

### 2.3 扩展功能 (P2)

| 编号 | 功能 | 描述 |
|------|------|------|
| F-011 | 房间系统 | 创建/加入/离开游戏房间 |
| F-012 | 聊天系统 | 房间内文字聊天 |
| F-013 | 断线重连 | 网络断开后自动重连 |
| F-014 | 观战模式 | 观看其他玩家的游戏 |

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 同时在线人数 | 100+ | 单服务器实例 |
| 状态同步频率 | 20-60Hz | 可配置 |
| 移动延迟 | < 100ms | 客户端预测后感知延迟 |
| 服务器 CPU | < 50% | 100 人在线时 |
| 内存占用 | < 512MB | 100 人在线时 |

### 3.2 可靠性需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 服务可用性 | 99.9% | 排除计划维护 |
| 数据一致性 | 最终一致 | 状态最终同步 |
| 断线容忍 | 30秒 | 断线后可重连恢复 |

### 3.3 可扩展性需求

| 维度 | 要求 |
|------|------|
| 水平扩展 | 支持多服务器实例 |
| 功能扩展 | 模块化设计，易于添加新功能 |
| 协议扩展 | Protobuf 支持协议演进 |

## 4. 技术约束

| 约束 | 说明 |
|------|------|
| 语言 | Go 1.21+ |
| 序列化 | Protocol Buffers 3 |
| 传输层 | UDP (游戏数据) + TCP (控制信令) |
| 依赖 | 最小化第三方依赖 |
| 部署 | 支持单机和多机部署 |

## 5. 用例场景

### 5.1 玩家加入游戏

```
1. 玩家启动客户端
2. 输入服务器地址和玩家名称
3. 客户端连接服务器
4. 服务器分配玩家到合适的服务器节点
5. 服务器发送当前世界状态
6. 客户端渲染游戏画面
7. 玩家可以开始游戏
```

### 5.2 实时战斗

```
1. 玩家 A 发起攻击
2. 客户端预测攻击结果并显示
3. 发送攻击输入到服务器
4. 服务器验证攻击合法性
5. 服务器计算伤害并更新状态
6. 服务器广播状态更新到所有客户端
7. 客户端校正预测结果
```

### 5.3 网络波动处理

```
1. 玩家网络出现 200ms 延迟
2. 客户端继续使用预测状态
3. 延迟恢复后收到服务器状态
4. 客户端检测到预测偏差
5. 平滑校正到服务器状态
```

## 6. 验收标准

### 6.1 最小可用版本 (MVP)

- [ ] 支持 100 人同时在线
- [ ] 玩家移动实时同步
- [ ] 基本攻击动作
- [ ] 客户端预测正常工作
- [ ] 服务器校正平滑无抖动
- [ ] 一致性哈希分配玩家

### 6.2 质量标准

- [ ] 单元测试覆盖率 > 70%
- [ ] 集成测试通过
- [ ] 压力测试 100 人无崩溃
- [ ] 文档完整清晰
