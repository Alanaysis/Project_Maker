# 产品思维文档

## 1. 产品定位

### 1.1 产品定义

**端侧极致量化模型框架**是一个面向 AI 工程师和嵌入式开发者的量化推理工具链，帮助用户将深度学习模型高效部署到边缘设备，特别针对车载场景优化。

### 1.2 核心价值主张

**"让模型量化不再神秘，让边缘部署不再困难"**

- **学习友好**：深入理解量化原理，而非简单使用工具
- **开箱即用**：提供完整的量化和推理解决方案
- **场景导向**：专注车载场景，满足实际需求

### 1.3 目标用户

| 用户类型 | 核心需求 | 使用场景 |
|----------|----------|----------|
| AI 工程师 | 学习量化原理 | 模型优化、部署 |
| 嵌入式工程师 | 部署 AI 模型 | 边缘设备开发 |
| 自动驾驶工程师 | 优化推理性能 | 车载系统集成 |
| 学生/研究者 | 理解技术原理 | 学术研究、项目实践 |

## 2. 用户吸引力分析

### 2.1 用户痛点

#### 痛点 1：量化原理不清晰

- **现状**：很多开发者只会使用量化工具，不理解底层原理
- **影响**：遇到问题无法排查，无法优化
- **解决方案**：提供详细的原理讲解和代码实现

#### 痛点 2：部署门槛高

- **现状**：模型部署涉及多个环节，技术栈复杂
- **影响**：学习成本高，开发周期长
- **解决方案**：提供一站式工具链，简化部署流程

#### 痛点 3：性能优化困难

- **现状**：性能优化需要深入了解硬件和算法
- **影响**：难以达到预期性能
- **解决方案**：提供自动化优化工具和最佳实践

#### 痛点 4：车载场景特殊需求

- **现状**：通用框架难以满足车规要求
- **影响**：需要大量定制开发
- **解决方案**：专注车载场景，提供针对性优化

### 2.2 用户收益

#### 学习收益

- **深入理解**：量化原理、推理优化、边缘部署
- **动手能力**：从零实现核心算法
- **工程经验**：完整的项目开发经验

#### 实践收益

- **技能提升**：掌握模型优化和部署技能
- **项目经验**：可写入简历的项目经历
- **技术视野**：了解行业前沿技术

#### 职业收益

- **就业竞争力**：边缘 AI 是热门方向
- **薪资提升**：稀缺技能，薪资溢价
- **职业发展**：AI 部署是必经之路

### 2.3 吸引力设计

#### 差异化优势

1. **深度 vs 广度**
   - 竞品：提供工具，浅层使用
   - 本项目：深入原理，理解本质

2. **学习 vs 生产**
   - 竞品：面向生产环境
   - 本项目：面向学习成长

3. **通用 vs 专注**
   - 竞品：通用场景
   - 本项目：专注车载场景

4. **黑盒 vs 白盒**
   - 竞品：封装复杂，难以理解
   - 本项目：代码清晰，易于学习

#### 用户留存设计

1. **渐进式学习路径**
   - 入门 → 进阶 → 高级
   - 每个阶段都有成就感

2. **实践驱动**
   - 理论 + 实践结合
   - 看得见的效果

3. **社区互动**
   - 学习交流
   - 问题解答

### 3.2 竞品详细分析

#### TensorRT

**优势**：
- 性能极致，GPU 推理标杆
- 自动化程度高
- NVIDIA 生态完善

**劣势**：
- 闭源，无法学习内部实现
- 仅支持 NVIDIA GPU
- 学习曲线陡峭

**用户反馈**：
- "性能很好，但遇到问题很难排查"
- "文档虽然详细，但理解起来困难"
- "绑定 NVIDIA 平台，不够灵活"

**本项目差异化**：
- 开源实现，可学习原理
- 支持多平台
- 提供详细原理讲解

#### NCNN

**优势**：
- 轻量级，无依赖
- 移动端优化好
- 开源，可定制

**劣势**：
- 功能相对简单
- 文档不够完善
- 社区相对较小

**用户反馈**：
- "简单易用，但功能有限"
- "性能不错，但优化空间不大"
- "适合移动端，但不适合复杂场景"

**本项目差异化**：
- 更深入的原理讲解
- 更完善的学习路径
- 专注车载场景

#### MNN

**优势**：
- 高性能，多后端
- 支持 INT4 量化
- 阿里内部验证

**劣势**：
- 文档相对较少
- 社区相对较小
- 学习资源有限

**用户反馈**：
- "性能很好，但文档不够详细"
- "功能强大，但学习成本高"
- "适合生产，但不适合学习"

**本项目差异化**：
- 详细的学习文档
- 渐进式学习路径
- 注重原理理解

## 5. 用户体验设计

### 5.1 快速开始体验

**目标**：5 分钟内完成第一次量化

```python
# 1. 安装
pip install edge-quant

# 2. 量化
from edge_quant import quantize
model = load_model("model.onnx")
quant_model = quantize(model, quant_type="int8")

# 3. 推理
from edge_quant import InferenceEngine
engine = InferenceEngine(quant_model)
result = engine.inference(input_data)
```

### 5.2 学习路径设计

**阶段 1：入门（1-2 天）**
- 理解量化概念
- 运行第一个示例
- 查看量化效果

**阶段 2：基础（1 周）**
- 学习量化原理
- 实现简单量化
- 理解校准方法

**阶段 3：进阶（2 周）**
- 实现算子融合
- 优化推理性能
- 学习内存管理

**阶段 4：高级（1 个月）**
- 混合精度量化
- 自定义算子
- 车载场景应用

### 5.3 错误处理体验

**友好错误信息**

```python
# 不好的错误信息
Error: quantization failed

# 好的错误信息
Error: 量化失败 - 模型包含不支持的算子 'CustomOp'
建议：
1. 使用 skip_ops 参数跳过该算子
2. 自定义该算子的量化实现
3. 参考文档：docs/custom_operator.md
```

**调试支持**

```python
# 启用详细日志
from edge_quant import set_log_level
set_log_level("DEBUG")

# 量化过程可视化
quantizer.visualize_calibration()
```
