# 02_REQUIREMENTS.md - 需求分析

## 功能需求

### 核心目标

通过实践示例系统学习 C++20 所有主要新特性，每个特性一个独立的可编译运行的示例文件。

### 特性清单

#### 1. 四大核心特性（必做）

| 编号 | 特性 | 优先级 | 复杂度 | 说明 |
|------|------|--------|--------|------|
| F01 | Concepts | P0 | 中 | 模板约束、requires 表达式、概念组合 |
| F02 | Ranges | P0 | 高 | 视图、管道操作、范围算法、惰性求值 |
| F03 | Coroutines | P0 | 高 | Generator、Task、co_await/yield/return |
| F04 | Modules | P1 | 中 | 模块声明、导出、导入（编译器支持有限） |

#### 2. 语言特性（必做）

| 编号 | 特性 | 优先级 | 复杂度 | 说明 |
|------|------|--------|--------|------|
| F05 | <=> 三向比较 | P0 | 低 | 自动生成比较、三种排序类别 |
| F06 | consteval/constinit | P0 | 中 | 立即函数、编译期初始化 |
| F07 | 聚合初始化改进 | P1 | 低 | 指定初始化器、括号初始化 |
| F08 | 范围 for 初始化 | P1 | 低 | 初始化语句 |
| F09 | Lambda 改进 | P0 | 中 | 模板 Lambda、捕获改进 |
| F10 | [[no_unique_address]] | P1 | 低 | 空成员优化 |
| F11 | using enum | P1 | 低 | 枚举成员引入作用域 |

#### 3. 标准库特性（必做）

| 编号 | 特性 | 优先级 | 复杂度 | 说明 |
|------|------|--------|--------|------|
| F12 | std::format | P0 | 中 | 格式化语法、自定义格式化 |
| F13 | std::span | P0 | 低 | 非拥有视图、子 span |
| F14 | std::jthread | P0 | 中 | 自动 join、stop_token |
| F15 | 同步原语 | P1 | 高 | latch、barrier、semaphore |
| F16 | std::source_location | P1 | 低 | 替代宏的源位置信息 |
| F17 | std::stop_token | P1 | 中 | 协作式取消机制 |

#### 4. 综合示例（必做）

| 编号 | 特性 | 优先级 | 复杂度 | 说明 |
|------|------|--------|--------|------|
| F18 | 综合示例 | P0 | 高 | 多特性联合使用 |

## 非功能需求

### 代码质量
- 每个文件有详细注释，解释核心概念
- 代码风格统一，遵循现代 C++ 最佳实践
- 所有示例可独立编译运行

### 文档完整性
- README.md：项目概述和快速开始
- 每个源文件：内嵌详细注释
- 构建配置：CMakeLists.txt 完整

### 可维护性
- 文件命名清晰有序（01-18）
- 每个示例独立，可单独编译
- 测试覆盖所有示例

---

## 约束条件

### 编译器要求
- GCC 10+ 或 Clang 13+ 或 MSVC 19.29+
- CMake 3.16+
- C++20 标准支持

### 限制
- Modules 示例需要编译器特殊支持，可能需要单独处理
- 部分特性（如 std::format）在较老编译器上不可用
