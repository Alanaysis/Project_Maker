# 01 - 市场调研：粒子群优化算法

## 1. 算法起源

### 1.1 历史背景

粒子群优化（Particle Swarm Optimization, PSO）由 **James Kennedy** 和 **Russell Eberhart** 于 **1995 年**提出。灵感来源于：

- **鸟群觅食行为**：鸟群如何找到食物源
- **鱼群游动模式**：鱼群如何协调移动
- **社会心理学**：个体如何从群体中学习

### 1.2 核心思想

模拟鸟群觅食过程：
1. 每只鸟（粒子）在搜索空间中飞行
2. 每只鸟记住自己找到的最好位置（个体最佳）
3. 整个鸟群共享信息，记住群体找到的最好位置（全局最佳）
4. 每只鸟根据自身经验和群体经验调整飞行方向

## 2. 算法原理

### 2.1 速度更新公式

```
v_new = w * v + c1 * r1 * (pbest - x) + c2 * r2 * (gbest - x)
```

其中：
- `w * v`：**惯性部分**，保持原有运动趋势
- `c1 * r1 * (pbest - x)`：**认知部分**，向个体最佳移动
- `c2 * r2 * (gbest - x)`：**社会部分**，向全局最佳移动
- `w`：惯性权重（控制探索与开发的平衡）
- `c1, c2`：学习因子（通常 c1 = c2 = 2.0）
- `r1, r2`：[0, 1] 之间的随机数

### 2.2 位置更新公式

```
x_new = x + v_new
```

### 2.3 算法流程

```
初始化粒子群
    ↓
评估适应度 → 更新个体最佳和全局最佳
    ↓
更新速度和位置
    ↓
是否满足终止条件？
    ↓ 否
返回上一步
    ↓ 是
输出全局最佳
```

## 3. 关键参数分析

### 3.1 惯性权重 w

| 值 | 效果 | 适用场景 |
|---|---|---|
| w > 1 | 全局探索能力强，但可能发散 | 不推荐 |
| w ≈ 0.9 | 平衡探索和开发 | 早期迭代 |
| w ≈ 0.4 | 局部开发能力强 | 后期迭代 |
| w = 0 | 无记忆，容易陷入局部最优 | 不推荐 |

**线性递减策略**：w 从 0.9 线性递减到 0.4，兼顾全局探索和局部开发。

### 3.2 学习因子 c1 和 c2

| 设置 | 效果 |
|---|---|
| c1 = 0 | 无个体记忆，只向群体学习 |
| c2 = 0 | 无社会学习，粒子独立搜索 |
| c1 = c2 = 2.0 | 平衡个体和群体学习（经典设置） |
| c1 > c2 | 更依赖个体经验 |
| c1 < c2 | 更依赖群体经验 |

### 3.3 粒子数量

- **太少**（< 10）：搜索不充分，容易陷入局部最优
- **适中**（20-50）：平衡计算成本和搜索能力
- **太多**（> 100）：计算成本高，但边际收益递减

## 4. 变体与改进

### 4.1 惯性权重策略

| 策略 | 公式 | 特点 |
|---|---|---|
| 固定权重 | w = const | 简单，但难以平衡 |
| 线性递减 | w = w_max - (w_max - w_min) * t/T | 经典，效果好 |
| 自适应 | 根据收敛情况动态调整 | 智能，但复杂 |
| 随机权重 | w = 0.5 + rand()/2 | 增加随机性 |

### 4.2 拓扑结构

| 结构 | 描述 | 特点 |
|---|---|---|
| 全局（gbest） | 所有粒子共享全局最佳 | 收敛快，但易陷入局部最优 |
| 环形（lbest） | 每个粒子只与邻居通信 | 收敛慢，但多样性好 |
| 星形 | 中心粒子与所有粒子通信 | 平衡收敛和多样性 |

### 4.3 其他改进

- **混沌 PSO**：用混沌序列替代随机数
- **量子 PSO**：引入量子力学概念
- **多目标 PSO**：处理多目标优化问题
- **离散 PSO**：处理离散/组合优化问题

## 5. 与其他优化算法对比

| 算法 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| PSO | 简单、参数少、收敛快 | 易陷入局部最优 | 连续优化 |
| 遗传算法 | 全局搜索能力强 | 参数多、收敛慢 | 组合优化 |
| 模拟退火 | 能跳出局部最优 | 收敛慢 | 离散优化 |
| 梯度下降 | 收敛快、精度高 | 需要梯度信息 | 凸优化 |
