# C++ 技巧需求分析文档

## 1. 文档概述

本文档定义了 C++ 奇技淫巧集锦项目的功能需求和非功能需求，为项目开发提供明确的目标和验收标准。

## 2. 功能需求

### 2.1 类型技巧 (Type Tricks) - 5 个技巧

#### T01: 类型标识 (Type Identification)
- **描述**: 使用 `typeid` 和 `type_info` 在运行时获取类型信息
- **功能点**:
  - 基本类型识别
  - 多态类型识别
  - 类型名称获取
  - 类型比较操作
- **应用场景**: 日志系统、序列化、调试工具

#### T02: 类型转换 (Type Conversion)
- **描述**: 安全的类型转换技术
- **功能点**:
  - `static_cast` 编译期转换
  - `dynamic_cast` 运行时转换
  - `reinterpret_cast` 底层转换
  - `const_cast` 常量转换
  - 自定义类型转换运算符
- **应用场景**: 类型适配、接口兼容、底层操作

#### T03: SFINAE 基础 (SFINAE Basics)
- **描述**: 替换失败不是错误的基本应用
- **功能点**:
  - `std::enable_if` 使用
  - `std::void_t` 技巧
  - 返回类型 SFINAE
  - 模板参数 SFINAE
- **应用场景**: 条件编译、接口约束、类型检测

#### T04: 类型萃取 (Type Traits)
- **描述**: 使用 `<type_traits>` 进行编译期类型查询
- **功能点**:
  - 类型分类 (`is_integral`, `is_floating_point`)
  - 类型属性 (`is_const`, `is_volatile`)
  - 类型关系 (`is_same`, `is_base_of`)
  - 类型变换 (`remove_const`, `add_pointer`)
- **应用场景**: 泛型编程、模板特化选择

#### T05: if constexpr
- **描述**: 编译期条件分支
- **功能点**:
  - 模板函数中的条件编译
  - 编译期类型分支
  - 消除运行时开销
  - 简化模板代码
- **应用场景**: 类型特定优化、编译期决策

### 2.2 模板技巧 (Template Tricks) - 5 个技巧

#### T06: 变参模板 (Variadic Templates)
- **描述**: 可变参数模板的使用
- **功能点**:
  - 参数包展开
  - 折叠表达式 (C++17)
  - 递归展开技巧
  - 完美转发
- **应用场景**: 通用容器、格式化输出、事件系统

#### T07: 模板特化 (Template Specialization)
- **描述**: 模板的全特化和偏特化
- **功能点**:
  - 函数模板全特化
  - 类模板全特化
  - 类模板偏特化
  - 默认模板参数
- **应用场景**: 类型特定实现、性能优化

#### T08: CRTP (Curiously Recurring Template Pattern)
- **描述**: 奇异递归模板模式
- **功能点**:
  - 静态多态实现
  - 基类调用派生类方法
  - 编译期接口实现
  - 混入类设计
- **应用场景**: 性能敏感的多态、策略模式

#### T09: 高级 SFINAE (Advanced SFINAE)
- **描述**: SFINAE 的高级应用技巧
- **功能点**:
  - 检测惯用法 (Detection Idiom)
  - `std::is_detected` 实现
  - 表达式 SFINAE
  - 模板约束替代方案
- **应用场景**: 接口检测、类型能力查询

#### T10: Concepts (C++20)
- **描述**: C++20 概念约束
- **功能点**:
  - 概念定义语法
  - `requires` 表达式
  - 约束模板参数
  - 概念组合和子sumption
- **应用场景**: 清晰的接口约束、更好的错误信息

### 2.3 内存技巧 (Memory Tricks) - 5 个技巧

#### T11: 智能指针 (Smart Pointers)
- **描述**: 现代 C++ 智能指针使用
- **功能点**:
  - `std::unique_ptr` 独占所有权
  - `std::shared_ptr` 共享所有权
  - `std::weak_ptr` 弱引用
  - 自定义删除器
  - `make_unique` 和 `make_shared`
- **应用场景**: 资源管理、所有权语义

#### T12: 内存池 (Memory Pool)
- **描述**: 自定义内存池实现
- **功能点**:
  - 固定大小块分配
  - 对象池设计
  - 内存复用策略
  - 线程安全内存池
- **应用场景**: 游戏引擎、高频分配场景

#### T13: 自定义分配器 (Custom Allocator)
- **描述**: STL 兼容的自定义分配器
- **功能点**:
  - 分配器接口实现
  - `allocator_traits` 使用
  - 有状态分配器
  - 内存池分配器
- **应用场景**: 特定内存布局、性能优化

#### T14: Placement New
- **描述**: 在指定内存位置构造对象
- **功能点**:
  - placement new 语法
  - `std::construct_at` (C++20)
  - 手动构造和析构
  - 内存对齐处理
- **应用场景**: 容器实现、内存映射

#### T15: RAII (Resource Acquisition Is Initialization)
- **描述**: 资源获取即初始化模式
- **功能点**:
  - 文件句柄管理
  - 锁管理
  - 网络连接管理
  - 自定义 RAII 类
- **应用场景**: 异常安全、资源管理

### 2.4 并发技巧 (Concurrency Tricks) - 5 个技巧

#### T16: 原子操作 (Atomics)
- **描述**: 原子类型和操作
- **功能点**:
  - `std::atomic` 基本使用
  - 内存顺序 (memory order)
  - 原子操作组合
  - 原子标志使用
- **应用场景**: 计数器、标志位、无锁数据结构

#### T17: 锁守卫 (Lock Guards)
- **描述**: RAII 风格的锁管理
- **功能点**:
  - `std::lock_guard` 使用
  - `std::unique_lock` 使用
  - `std::scoped_lock` (C++17)
  - 死锁避免策略
- **应用场景**: 多线程同步、临界区保护

#### T18: 线程池 (Thread Pool)
- **描述**: 线程池实现
- **功能点**:
  - 固定大小线程池
  - 任务队列设计
  - 任务提交和结果获取
  - 线程池关闭策略
- **应用场景**: 并行任务处理、服务器编程

#### T19: 无锁编程 (Lock-Free Programming)
- **描述**: 无锁数据结构和算法
- **功能点**:
  - CAS 操作
  - 无锁栈实现
  - 无锁队列实现
  - ABA 问题解决
- **应用场景**: 高性能并发、实时系统

#### T20: 异步编程 (Async Programming)
- **描述**: 异步任务和 future 模式
- **功能点**:
  - `std::async` 使用
  - `std::future` 和 `std::promise`
  - `std::packaged_task`
  - 协程基础 (C++20)
- **应用场景**: 异步 I/O、后台任务

### 2.5 优化技巧 (Optimization Tricks) - 5 个技巧

#### T21: 移动语义 (Move Semantics)
- **描述**: 移动语义和右值引用
- **功能点**:
  - 右值引用绑定

#### T24: SSO (Small String Optimization)

- **应用场景**: 字符串密集型应用

#### T25: 循环优化 (Loop Optimization)
- **描述**: 循环性能优化技巧
- **功能点**:

## 3. 非功能需求

### 3.1 可编译性 (Compilability)

#### NF-01: 编译器兼容性
- **要求**: 支持主流 C++ 编译器
- **目标编译器**:
  - GCC 10.0+
  - Clang 12.0+
  - MSVC 2019+ (19.20+)
- **验收标准**: 所有技巧在三个编译器上都能编译通过

#### NF-02: C++ 标准兼容
- **要求**: 支持 C++17 和 C++20 标准
- **C++17 必需特性**:
  - `if constexpr`
  - 结构化绑定
  - `std::optional`, `std::variant`, `std::any`
  - 折叠表达式
- **C++20 可选特性**:
  - Concepts
  - Ranges
  - Coroutines
  - Modules
- **验收标准**: C++17 特性必须可用，C++20 特性提供条件编译

#### NF-03: 构建系统
- **要求**: 使用 CMake 构建系统
- **最低版本**: CMake 3.16+
- **功能**:
  - 支持 out-of-source 构建
  - 支持单独编译每个技巧
  - 支持编译所有技巧
  - 支持运行测试
- **验收标准**: `cmake .. && cmake --build .` 能成功编译所有示例

### 3.2 文档质量 (Documentation Quality)

#### NF-04: 代码注释
- **要求**: 每个技巧文件必须包含详细注释
- **注释内容**:
  - 技巧概述 (文件头部)
  - 实现原理说明
  - 关键代码注释
  - 使用示例说明
  - 注意事项和陷阱
- **验收标准**: 注释覆盖率 > 80%

#### NF-05: README 文档
- **要求**: 每个技巧分类目录必须有 README
- **内容**:
  - 分类概述
  - 技巧列表
  - 学习建议
  - 参考资源
- **验收标准**: 所有分类目录都有 README.md

#### NF-06: 示例完整性
- **要求**: 每个技巧必须有完整可运行示例
- **示例要求**:
  - 独立可编译
  - 包含 `main` 函数
  - 有清晰的输出
  - 展示实际应用场景
- **验收标准**: 所有示例都能独立编译运行

### 3.3 代码质量 (Code Quality)

#### NF-07: 代码风格一致性
- **要求**: 遵循统一的代码风格
- **规范**:
  - 使用 4 空格缩进
  - 函数命名使用 snake_case
  - 类命名使用 PascalCase
  - 常量命名使用 UPPER_SNAKE_CASE
- **验收标准**: 所有代码风格一致

#### NF-08: 现代 C++ 实践
- **要求**: 使用现代 C++ 特性
- **实践**:
  - 优先使用 `auto`
  - 使用范围 for 循环
  - 使用智能指针
  - 使用 RAII
- **验收标准**: 不使用已弃用特性

### 3.4 可维护性 (Maintainability)

#### NF-09: 模块化设计
- **要求**: 每个技巧独立、低耦合
- **设计原则**:
  - 单一职责
  - 接口清晰
  - 依赖最小化
- **验收标准**: 可以单独添加新技巧而不影响现有代码

#### NF-10: 扩展性
- **要求**: 易于添加新技巧
- **机制**:
  - 清晰的目录结构
  - 标准的文件模板
  - 自动化构建支持
- **验收标准**: 新增技巧只需添加文件和更新 CMakeLists.txt

### 3.5 性能 (Performance)

#### NF-11: 编译性能
- **要求**: 合理的编译时间
- **目标**:
  - 单个技巧编译 < 10 秒
  - 全部技巧编译 < 5 分钟
- **验收标准**: 在标准开发机器上满足时间要求

#### NF-12: 运行性能
- **要求**: 示例运行时间合理
- **目标**:
  - 单个示例运行 < 1 秒
  - 性能测试示例有明确的时间输出
- **验收标准**: 示例能快速运行并显示结果
