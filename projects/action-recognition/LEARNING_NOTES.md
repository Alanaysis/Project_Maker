# 动作识别学习笔记

## 核心概念

### 1. 什么是动作识别？

动作识别是从视频中自动识别人体动作的任务。与图像分类不同，动作识别需要同时理解**空间信息**（外观）和**时间信息**（运动）。

**核心挑战**：
- 同一动作可以有不同的执行方式
- 不同动作可能外观相似
- 需要捕捉长时间的时序依赖
- 视频数据量大，计算成本高

### 2. 空间特征提取

使用CNN（如ResNet）从每帧图像中提取空间特征。

**关键理解**：
- CNN捕捉的是单帧的外观信息（人物姿态、物体形状等）
- 使用预训练模型（ImageNet）可以加速收敛
- 可以冻结CNN只训练时序模型，减少计算量

**特征维度**：
```
输入: (B, T, 3, 224, 224)  # T帧RGB图像
输出: (B, T, 512)          # 每帧一个512维特征向量
```

### 3. 时序建模

将帧级特征序列输入时序模型，捕捉帧与帧之间的关系。

**LSTM vs GRU vs Transformer**：

| 模型 | 参数量 | 训练速度 | 长期依赖 | 并行性 |
|------|--------|----------|----------|--------|
| LSTM | 最多 | 慢 | 好 | 差 |
| GRU | 较少 | 中 | 较好 | 差 |
| Transformer | 中 | 快 | 最好 | 好 |

**为什么用RNN处理视频？**
- 视频是天然的序列数据
- 动作通常有时间顺序（如"起跳"→"空中"→"落地"）
- RNN可以记住之前的信息来理解当前帧

### 4. 采样策略

视频可能有几百到几千帧，不可能全部处理。需要采样。

**均匀采样**：等间距选取帧，覆盖整个视频
**随机采样**：随机选取帧，增加多样性
**密集采样**：连续选取帧，适合短时动作
**时序抖动**：均匀采样+随机偏移，训练时的数据增强

**关键洞察**：采样策略对结果影响很大！如果只采样视频开头的帧，可能错过关键动作。

## 关键实现细节

### 1. 批量帧处理

将5D视频张量reshape为4D进行CNN处理：

```python
# 输入: (B, T, C, H, W)
B, T, C, H, W = x.shape
x = x.view(B * T, C, H, W)  # 变成 (B*T, C, H, W)
features = self.cnn(x)        # CNN处理
features = features.view(B, T, -1)  # 恢复为 (B, T, feat_dim)
```

**为什么要这样？**
- CNN期望4D输入 (batch, channels, height, width)
- 把时间维度合并到batch维度，复用CNN权重
- 处理完再恢复时间维度

### 2. 变长序列处理

不同视频的帧数可能不同，需要特殊处理：

```python
# 打包变长序列
packed = nn.utils.rnn.pack_padded_sequence(
    x, lengths, batch_first=True, enforce_sorted=False
)
output, hidden = self.lstm(packed)

# 解包
output, _ = nn.utils.rnn.pad_packed_sequence(output, batch_first=True)
```

**不处理会怎样？**
- RNN会处理padding的零向量，产生错误的隐藏状态
- 影响模型学习效果

### 3. 预训练权重

使用ImageNet预训练的CNN权重：

```python
model = models.resnet18(weights="IMAGENET1K_V1")
```

**为什么有效？**
- ImageNet学到的特征（边缘、纹理、形状）对视频帧也有效
- 大幅加速收敛
- 小数据集上效果提升明显

## 常见问题

### Q1: 为什么模型不收敛？

**可能原因**：
1. 学习率太大或太小
2. 数据标签错误
3. 数据预处理不正确（如没有归一化）
4. 模型太简单或太复杂

**调试方法**：
- 先用小数据集过拟合，确认模型能学习
- 检查数据加载是否正确
- 打印loss变化曲线

### Q2: GPU内存不足怎么办？

**解决方法**：
1. 减小batch_size
2. 减少num_frames
3. 使用更小的backbone（resnet18而非resnet50）
4. 冻结backbone参数
5. 使用gradient checkpointing

### Q3: 为什么用LSTM而不是简单平均池化？

**简单平均池化**：
```python
features = spatial_features.mean(dim=1)  # 丢失时序信息
```

**LSTM**：
- 可以记住之前的帧信息
- 理解动作的先后顺序
- 捕捉长期依赖关系

**但平均池化也有优点**：
- 计算快
- 不需要训练
- 某些场景下效果够用

## 最佳实践

1. **从小开始**：先用小数据集、小模型验证流程
2. **合成数据调试**：使用synthetic模式快速测试代码
3. **特征缓存**：大规模训练时先提取特征缓存到磁盘
4. **多策略采样**：训练时用随机/抖动采样，测试时用均匀采样
5. **渐进式解冻**：先冻结backbone训练时序模型，再微调整个网络

## 延伸学习

- **3D卷积**：直接在时空维度上卷积，如C3D、I3D
- **双流网络**：分别处理RGB和光流，再融合
- **SlowFast**：双路径捕捉慢速和快速运动
- **Video Transformer**：用注意力机制替代RNN
- **时序动作检测**：不仅分类，还要定位动作时间区间
