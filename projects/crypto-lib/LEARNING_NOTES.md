# 学习笔记

## 项目概述

本项目实现了一个密码学库，涵盖三种核心密码学技术：
1. AES对称加密
2. RSA非对称加密
3. 椭圆曲线密码学（ECC）

## 学习目标达成情况

### 1. 理解对称加密和非对称加密

**对称加密（AES）：**
- 加密和解密使用相同密钥
- 速度快，适合大量数据加密
- 密钥分发是主要挑战

**非对称加密（RSA/ECC）：**
- 使用公钥/私钥对
- 速度慢，适合密钥交换和数字签名
- 解决了密钥分发问题

**关键理解：**
- 实际应用中通常结合使用：用非对称加密传输对称密钥，用对称密钥加密数据

### 2. 掌握RSA算法原理

**核心数学概念：**
- 大数分解难题：给定n = p × q，难以分解出p和q
- 欧拉定理：a^φ(n) ≡ 1 (mod n)
- 模逆运算：找到d使得e × d ≡ 1 (mod φ(n))

**实现要点：**
- Miller-Rabin素性测试用于生成大素数
- 扩展欧几里得算法计算模逆
- 模幂运算用于加密/解密

**安全考虑：**
- 密钥长度至少2048位
- 需要填充方案（OAEP）防止攻击
- 随机数生成必须是密码学安全的

### 3. 学会椭圆曲线密码学

**椭圆曲线基础：**
- 曲线方程：y² = x³ + ax + b (mod p)
- 点加法和倍点运算
- 标量乘法：kP = P + P + ... + P (k次)

**ECDH密钥交换：**
- 基于椭圆曲线离散对数问题
- 比RSA更高效（更短的密钥提供相同安全性）
- 256位ECC ≈ 3072位RSA

**ECDSA数字签名：**
- 签名过程：选择随机k，计算R = kG，r = R.x mod n，s = k⁻¹(z + r×d) mod n
- 验证过程：计算u1, u2，验证点的x坐标

## 技术难点与解决方案

### 1. 大数运算

**问题：** RSA和ECC需要处理大整数（256位以上）

**解决方案：** 使用`num-bigint`库
```rust
use num_bigint::{BigInt, BigUint};

let a = BigUint::from(12345u32);
let b = BigUint::from(67890u32);
let c = a.modpow(&b, &n);
```

### 2. 椭圆曲线点运算

**问题：** 点加法涉及模逆运算，计算复杂

**解决方案：** 实现扩展欧几里得算法
```rust
fn extended_gcd(a: &BigInt, b: &BigInt) -> (BigInt, BigInt, BigInt) {
    if a.is_zero() {
        return (b.clone(), BigInt::zero(), BigInt::one());
    }
    let (g, x, y) = extended_gcd(&(b % a), a);
    (g, y - (b / a) * &x, x)
}
```

### 3. 随机数生成

**问题：** 密码学需要安全的随机数

**解决方案：** 使用`rand`库的密码学安全随机数生成器
```rust
use rand::Rng;

let mut rng = rand::thread_rng();
let random_bytes: Vec<u8> = (0..32).map(|_| rng.gen()).collect();
```

### 4. AES S-box实现

**问题：** SubBytes变换需要快速查找

**解决方案：** 预计算查找表
```rust
const S_BOX: [[u8; 16]; 16] = [
    [0x63, 0x7C, 0x77, ...],
    // 256个条目
];

fn sub_bytes(&self, state: &mut [u8; 16]) {
    for byte in state.iter_mut() {
        let row = (*byte >> 4) as usize;
        let col = (*byte & 0x0F) as usize;
        *byte = S_BOX[row][col];
    }
}
```

## 设计决策

### 1. 为什么选择CBC模式？

- 相比ECB更安全（隐藏模式）
- 实现相对简单
- 适合学习目的

### 2. 为什么使用num-bigint？

- Rust原生不支持大整数运算
- num-bigint提供完整的任意精度整数支持
- 性能满足学习需求

### 3. 为什么实现secp256k1和P-256？

- secp256k1：比特币使用，了解区块链密码学
- P-256：NIST标准，广泛使用

## 性能观察

### AES性能
- 查找表优化使S-box操作O(1)
- CBC模式需要串行处理
- 大数据量时瓶颈在内存分配

### RSA性能
- 密钥生成最耗时（需要找大素数）
- 加密/解密使用快速模幂运算
- 2048位密钥生成需要几秒

### ECC性能
- 点运算比RSA的模幂运算更快
- 标量乘法使用double-and-add优化
- 相同安全级别下密钥更短

## 安全教训

1. **不要自己实现密码学**：生产环境应使用经过验证的库
2. **随机数很重要**：弱随机数会导致密钥可预测
3. **填充是必要的**：无填充的RSA容易受到攻击
4. **常量时间操作**：避免时序攻击
5. **密钥管理**：密钥的安全存储和传输同样重要
