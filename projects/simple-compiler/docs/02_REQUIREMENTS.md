# 需求分析

## 1. 项目概述

### 1.1 项目目标

构建一个教学级的简易编译器，涵盖编译器的核心技术，帮助学习者理解编译原理。

### 1.2 目标用户

- 计算机科学学生
- 编译器爱好者
- 编程语言开发者
- 对底层技术感兴趣的程序员

### 1.3 设计原则

- **完整性**：涵盖编译器的所有核心阶段
- **可读性**：代码清晰，注释详细
- **可扩展性**：易于添加新特性
- **实用性**：包含实际应用示例

## 2. 功能需求

### 2.1 词法分析器 (Lexer)

**核心功能**
- 将源代码转换为 token 流
- 支持多种 token 类型
- 处理空白字符和注释
- 报告词法错误

**支持的 token 类型**

| 类别 | Token | 示例 |
|------|-------|------|
| 关键字 | let, var, const, if, else, while, for, fn, return, class | `let x = 10;` |
| 标识符 | 变量名、函数名 | `myVariable`, `calculateSum` |
| 整数 | INTEGER | `42`, `100`, `-5` |
| 浮点数 | FLOAT | `3.14`, `2.0`, `-1.5` |
| 字符串 | STRING | `"hello"`, `"world"` |
| 布尔值 | TRUE, FALSE | `true`, `false` |
| 运算符 | +, -, *, /, %, ==, !=, <, >, <=, >=, &&, \|\|, ! | `x + y`, `a == b` |
| 分隔符 | (, ), {, }, [, ], ,, ;, : | `fn add(a, b)` |

**特殊处理**
- 单行注释：`// this is a comment`
- 多行注释：`/* this is a comment */`
- 字符串转义：`\n`, `\t`, `\\`, `\"`
- 科学计数法：`1.5e10`

### 2.2 语法分析器 (Parser)

**核心功能**
- 将 token 流转换为抽象语法树 (AST)
- 使用递归下降法
- 处理运算符优先级和结合性
- 报告语法错误

**支持的语法结构**

**变量声明**
```
let x: int = 10;
var y = "hello";
const PI: float = 3.14;
```

**函数定义**
```
fn add(a: int, b: int): int {
    return a + b;
}
```

**类定义**
```
class Point {
    x: int;
    y: int;

    fn init(x: int, y: int) {
        this.x = x;
        this.y = y;
    }
}
```

**控制流**
```
if (condition) {
    // ...
} else {
    // ...
}

while (condition) {
    // ...
}

for (let i = 0; i < 10; i++) {
    // ...
}
```

**表达式**
```
x + y * z          // 算术
a == b && c != d   // 逻辑
foo(1, 2)          // 函数调用
arr[i]             // 数组访问
obj.field          // 成员访问
```

### 2.3 语义分析器

**核心功能**
- 类型检查
- 作用域分析
- 名称解析
- 类型推导

**类型系统**

| 类型 | 描述 | 示例 |
|------|------|------|
| int | 整数类型 | `42`, `-10` |
| float | 浮点类型 | `3.14`, `-2.5` |
| string | 字符串类型 | `"hello"` |
| bool | 布尔类型 | `true`, `false` |
| void | 空类型 | 函数无返回值 |
| array | 数组类型 | `int[]`, `string[]` |

**类型检查规则**
- 变量类型一致性
- 函数参数类型匹配
- 返回值类型检查
- 运算符操作数类型检查

**作用域规则**
- 块作用域
- 函数作用域
- 类作用域
- 全局作用域

### 2.4 中间代码生成

**核心功能**
- 生成三地址码
- SSA（静态单赋值）形式
- 基本块划分
- 控制流图构建

**IR 指令类型**

| 类别 | 指令 | 描述 |
|------|------|------|
| 算术 | ADD, SUB, MUL, DIV, MOD | 算术运算 |
| 比较 | CMP_EQ, CMP_NE, CMP_LT, CMP_GT | 比较运算 |
| 逻辑 | AND, OR, NOT | 逻辑运算 |
| 赋值 | LOAD, MOVE, STORE | 赋值操作 |
| 控制 | JUMP, BRANCH, CALL, RETURN | 控制流 |
| 内存 | ALLOCA, LOAD_MEM, STORE_MEM | 内存操作 |

### 2.5 代码优化

**优化 Pass**

**常量折叠**
```
// 优化前
x = 2 + 3

// 优化后
x = 5
```

**死代码消除**
```
// 优化前
x = 10      // x 未使用
y = 20
return y

// 优化后
y = 20
return y
```

**公共子表达式消除**
```
// 优化前
a = b + c
d = b + c

// 优化后
a = b + c
d = a
```

**强度削减**
```
// 优化前
x = y * 2

// 优化后
x = y << 1
```

**循环优化**
```
// 优化前
while (i < 10) {
    x = a + b  // 循环不变量
    i = i + 1
}

// 优化后
t = a + b
while (i < 10) {
    x = t
    i = i + 1
}
```

### 2.6 目标代码生成

**核心功能**
- 生成 x86-64 汇编代码
- 寄存器分配
- 栈帧管理
- 函数调用约定

**寄存器分配**
- 线性扫描算法
- 溢出到栈
- 寄存器复用

**栈帧布局**
```
+------------------+
| 返回地址         |
+------------------+
| 保存的寄存器     |
+------------------+
| 局部变量         |
+------------------+
| 临时变量         |
+------------------+
| 参数区域         |
+------------------+
```

**函数调用约定**
- 参数传递：前6个参数通过寄存器，其余通过栈
- 返回值：通过 RAX 寄存器
- 调用者保存：RAX, RCX, RDX, RSI, RDI, R8-R11
- 被调用者保存：RBX, R12-R15, RBP, RSP

### 2.7 解释器

**直接解释执行**
- 遍历 AST 执行
- 管理运行时环境
- 处理函数调用
- 支持闭包

**栈式虚拟机**
- 基于栈的字节码
- 操作码定义
- 栈操作
- 调用栈管理

**寄存式虚拟机**
- 基于寄存器的字节码
- 寄存器文件
- 指令执行
- 性能优化
