# Implementation - Simple Compiler

## Implementation Overview

The Simple Compiler is implemented in Rust, consisting of 5 main modules:
1. **Lexer** - Tokenization
2. **Parser** - AST construction
3. **AST** - Tree data structures
4. **Code Generator** - Bytecode emission
5. **VM** - Bytecode execution

## Module Structure

```
src/
├── lib.rs          # Module declarations
├── main.rs         # CLI entry point (REPL, file execution)
├── lexer.rs        # Lexer implementation
├── parser.rs       # Parser implementation
├── ast.rs          # AST node definitions
├── codegen.rs      # Code generator
└── vm.rs           # Virtual machine
```

## Lexer Implementation

### Key Design: Hand-Written Scanner

The lexer is hand-written rather than using regex, giving us:
- Full control over error messages
- Efficient single-pass scanning
- Easy to extend with new tokens

### Core Structure

```rust
pub struct Lexer {
    source: Vec<char>,  // Source code as character array
    pos: usize,         // Current position
    line: usize,        // Current line number
    column: usize,      // Current column number
}
```

### Tokenization Process

1. Skip whitespace and comments
2. Read the next character
3. Based on the character, determine the token type
4. For multi-character tokens (identifiers, numbers, strings), read until the token is complete
5. Return the token with line/column information

### Notable Implementation Details

**String Escapes:**
```rust
// Support \n, \t, \\, \"
match escaped {
    'n' => value.push('\n'),
    't' => value.push('\t'),
    '\\' => value.push('\\'),
    '"' => value.push('"'),
}
```

**Comment Handling:**
- Line comments: `// ...`
- Block comments: `/* ... */` (with nesting support)

**Number Parsing:**
- Integers: `42`, `0`, `100000`
- Floats: `3.14`, `0.5`, `100.0`
- No support for hex, octal, or scientific notation

## Parser Implementation

### Recursive Descent Parsing

Each grammar rule maps to a function:

```rust
fn parse_statement(&mut self) -> Result<Statement, String> {
    match &self.peek().kind {
        TokenKind::Let => self.parse_let(),
        TokenKind::If => self.parse_if(),
        TokenKind::While => self.parse_while(),
        TokenKind::Fn => self.parse_fn_def(),
        // ...
    }
}
```

### Operator Precedence

Implemented via precedence climbing (nested function calls):

```
parse_expression()
  └── parse_or()           // ||
      └── parse_and()      // &&
          └── parse_equality()  // == !=
              └── parse_comparison()  // < <= > >=
                  └── parse_addition()  // + -
                      └── parse_multiplication()  // * / %
                          └── parse_unary()  // - !
                              └── parse_call()  // function calls
                                  └── parse_primary()  // literals, identifiers, (expr)
```

### Error Recovery

Currently, the parser returns an error on the first unexpected token. Future improvements could include:
- Panic-mode recovery (skip to next statement)
- Error productions (handle common mistakes)

## AST Implementation

### Enum-Based Design

Rust's enums with data are perfect for AST nodes:

```rust
pub enum Expression {
    IntegerLiteral(i64),
    BinaryOp {
        op: BinaryOp,
        left: Box<Expression>,
        right: Box<Expression>,
    },
    // ...
}
```

### Box for Recursive Types

Since AST nodes are recursive (expressions contain expressions), we use `Box<Expression>` for heap allocation:

```rust
BinaryOp {
    left: Box<Expression>,  // Heap-allocated
    right: Box<Expression>,
}
```

## Code Generator Implementation

### Stack-Based Bytecode

The code generator produces instructions for a stack-based VM:

```
Expression: 1 + 2 * 3

Generated bytecode:
  PushInt(1)      // Push 1
  PushInt(2)      // Push 2
  PushInt(3)      // Push 3
  Mul             // Pop 2,3; push 6
  Add             // Pop 1,6; push 7
```

### Backpatching for Control Flow

For if/else and while loops, we need to jump to addresses that aren't known yet:

```rust
// Emit placeholder jump
let jump_addr = self.current_addr();
self.emit(Instruction::JumpIfFalse(0));  // 0 is placeholder

// Generate then block
// ...

// Patch the jump address
let end_addr = self.current_addr();
self.instructions[jump_addr] = Instruction::JumpIfFalse(end_addr);
```

### Short-Circuit Evaluation

Logical operators use short-circuit evaluation:

```rust
// a && b
// If a is false, don't evaluate b
self.gen_expression(left)?;
let jump_addr = self.current_addr();
self.emit(Instruction::JumpIfFalse(0));  // Skip if false
self.emit(Instruction::Pop);  // Pop left value
self.gen_expression(right)?;
// Patch jump to here
```

### Function Code Generation

Functions are embedded in the main instruction stream with a jump over them:

```
Jump(10)        // Skip function body
// Function body starts here
LoadLocal("x")
LoadLocal("y")
Add
Return
// Function body ends here
// Main code continues at address 10
```

## VM Implementation

### Stack Machine

The VM maintains:
- **Value stack**: For expression evaluation
- **Call stack**: For function calls (with local variables)
- **Globals**: Top-level variable storage
- **Program counter (PC)**: Current instruction address

### Execution Loop

```rust
while self.pc < self.instructions.len() {
    let inst = self.instructions[self.pc].clone();
    self.pc += 1;
    self.execute(inst)?;
}
```

### Function Calls

When calling a function:
1. Pop arguments from the stack
2. Create a new stack frame with local variables
3. Push the frame onto the call stack
4. Set PC to the function's start address

When returning:
1. Pop the return value (if any)
2. Pop the stack frame
3. Restore PC to the return address

### Type System

Types are checked at runtime:

```rust
pub enum Value {
    Int(i64),
    Float(f64),
    Bool(bool),
    Str(String),
}
```

Operations check types and return errors for mismatches:
```rust
Instruction::Add => self.binary_op(|a, b| match (a, b) {
    (Value::Int(a), Value::Int(b)) => Ok(Value::Int(a + b)),
    (Value::Float(a), Value::Float(b)) => Ok(Value::Float(a + b)),
    (Value::Str(a), Value::Str(b)) => Ok(Value::Str(format!("{}{}", a, b))),
    (a, b) => Err(format!("Cannot add {:?} and {:?}", a, b)),
})?,
```

## CLI Implementation

The main binary (`sc`) supports three modes:

1. **REPL**: Interactive mode (default)
2. **Run**: Execute a file (`sc run file.sl`)
3. **Compile**: Show compilation details (`sc compile file.sl`)

### REPL Features
- Line-by-line input
- Immediate execution
- Token/AST inspection commands
- Multi-line support (accumulates until complete statement)

## Performance Considerations

### Current Implementation
- No optimization passes
- Dynamic type checking
- Heap allocation for strings
- Simple stack-based execution

### Potential Optimizations
- Constant folding at compile time
- Type inference to reduce runtime checks
- Register-based VM for fewer instructions
- JIT compilation for hot code paths
