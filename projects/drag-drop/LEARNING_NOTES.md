## 2. 重点难点

### 难点1：拖拽事件处理

**问题**：如何正确处理拖拽相关的 DOM 事件？

**解决过程**：
1. 尝试1：直接使用 HTML5 原生拖拽 API
   - 问题：样式定制困难，兼容性问题
2. 尝试2：使用鼠标事件模拟拖拽
   - 优点：完全可控
   - 问题：需要处理很多细节
3. 最终方案：封装统一的拖拽管理器
   - 支持鼠标和触摸事件
   - 提供事件订阅机制
   - 处理延迟和距离阈值

**关键收获**：
- 事件处理需要考虑多种设备
- 预绑定事件处理器避免重复创建
- 使用事件总线解耦模块

### 难点2：排序位置计算

**问题**：如何根据鼠标位置计算元素的新位置？

**解决过程**：
1. 尝试1：遍历所有元素检查重叠
   - 问题：效率太低
2. 尝试2：只检查相邻元素
   - 优点：效率高
   - 问题：快速移动时可能跳过元素
3. 最终方案：计算鼠标与元素中心点的关系
   - 根据排序方向判断
   - 支持垂直和水平排序
   - 使用占位符指示位置

**关键收获**：
- 位置计算要考虑排序方向
- 占位符是排序的关键
- 动画效果提升用户体验

### 难点3：文件验证

**问题**：如何验证拖拽上传的文件？

**解决过程**：
1. 尝试1：只检查文件扩展名
   - 问题：不可靠
2. 尝试2：检查 MIME 类型
   - 优点：更可靠
   - 问题：某些文件类型可能没有正确的 MIME 类型
3. 最终方案：组合使用多种验证方式
   - 检查 MIME 类型
   - 检查文件扩展名
   - 检查文件大小
   - 支持自定义验证

**关键收获**：
- 文件验证要多层防护
- MIME 类型更可靠
- 文件大小限制很重要

## 4. 代码片段收藏

### 片段1：事件总线

```typescript
class EventBus<T extends Record<string, unknown>> {
  private listeners = new Map<keyof T, Set<EventHandler<unknown>>>();

  on<K extends keyof T>(event: K, handler: EventHandler<T[K]>): RemoveEventListener {
    if (!this.listeners.has(event)) {
      this.listeners.set(event, new Set());
    }
    this.listeners.get(event)!.add(handler as EventHandler<unknown>);

    return () => {
      this.off(event, handler);
    };
  }

  emit<K extends keyof T>(event: K, data: T[K]): void {
    this.listeners.get(event)?.forEach((handler) => {
      (handler as EventHandler<T[K]>)(data);
    });
  }
}
```

**为什么收藏**：事件总线是解耦模块的关键

**使用场景**：模块间通信

### 片段3：节流函数

```typescript
function throttle<T extends (...args: unknown[]) => unknown>(
  fn: T,
  delay: number
): T {
  let lastTime = 0;
  let timer: ReturnType<typeof setTimeout> | null = null;

  return ((...args: unknown[]) => {
    const now = Date.now();
    const remaining = delay - (now - lastTime);

    if (remaining <= 0) {
      if (timer) {
        clearTimeout(timer);
        timer = null;
      }
      lastTime = now;
      fn(...args);
    } else if (!timer) {
      timer = setTimeout(() => {
        lastTime = Date.now();
        timer = null;
        fn(...args);
      }, remaining);
    }
  }) as unknown as T;
}
```

**为什么收藏**：节流是性能优化的关键

**使用场景**：频繁事件处理
