# 需求分析文档

## 1. 项目概述

### 1.1 项目目标
实现一个网络防火墙系统，支持包过滤、状态检测和入侵检测功能。该项目主要用于学习网络协议和防火墙原理。

### 1.2 项目范围
- 包过滤（IPv4）
- 连接状态跟踪
- 基本入侵检测
- 日志记录

### 1.3 不在范围内
- IPv6 支持
- NAT 功能
- 应用层过滤
- 图形界面

## 2. 用户画像

### 2.1 主要用户

**用户类型**：计算机科学学生/网络工程师学习者

**特征**：
- 有一定 C 语言基础
- 了解基本网络概念（IP、TCP、UDP）
- 希望深入理解防火墙原理
- 学习热情高，但实践经验有限

**目标**：
- 理解数据包结构
- 掌握包过滤机制
- 学习状态检测技术
- 获得实际项目经验

### 2.2 使用场景

**场景 1：学习网络协议**
```
用户：我想理解 IP 包的结构

```

**场景 2：测试过滤规则**
```
用户：我想测试一条过滤规则是否正确
系统：加载规则并显示匹配结果
```

**场景 3：观察连接状态**
```
用户：我想看 TCP 连接是如何建立的

```

```
用户：我想看防火墙如何检测 SYN Flood
系统：检测并告警异常流量
```

## 3. 功能需求

### 3.1 包过滤 (P0 - 必须实现)

#### FR-001: IP 地址过滤
- **描述**：支持基于源/目的 IP 地址的过滤
- **输入**：IP 地址或 CIDR
- **输出**：允许/拒绝
- **示例**：
  ```
  允许 192.168.1.0/24
  拒绝 10.0.0.5
  ```

#### FR-002: 端口过滤
- **描述**：支持基于源/目的端口的过滤
- **输入**：端口号或端口范围
- **输出**：允许/拒绝
- **示例**：
  ```
  允许 目的端口 80
  拒绝 目的端口 22
  允许 目的端口 1024-65535
  ```

#### FR-003: 协议过滤
- **描述**：支持基于协议类型的过滤
- **输入**：协议类型 (TCP/UDP/ICMP/ALL)
- **输出**：允许/拒绝
- **示例**：
  ```
  允许 TCP
  拒绝 ICMP
  ```

#### FR-004: 规则匹配
- **描述**：按顺序匹配规则，执行第一个匹配的动作
- **输入**：数据包
- **输出**：动作 (ACCEPT/DROP/REJECT)
- **规则**：
  - 规则按顺序匹配
  - 第一个匹配的规则生效
  - 默认策略：DROP

### 3.2 状态检测 (P1 - 重要)

#### FR-005: TCP 连接跟踪
- **描述**：跟踪 TCP 连接的状态
- **状态**：
  ```
  NEW → SYN_SENT → ESTABLISHED → FIN_WAIT → CLOSED
  ```
- **功能**：
  - 记录连接状态
  - 验证状态转换合法性
  - 超时清理

#### FR-006: UDP "连接" 跟踪
- **描述**：跟踪 UDP 流量（虽然 UDP 是无连接的）
- **功能**：
  - 记录源/目的地址和端口
  - 设置超时时间
  - 匹配请求/响应

#### FR-007: ICMP 跟踪
- **描述**：跟踪 ICMP 请求/响应
- **功能**：
  - 匹配 Echo Request/Reply
  - 记录 ICMP 类型和代码

#### FR-008: 连接超时管理
- **描述**：自动清理超时的连接
- **超时时间**：
  - TCP ESTABLISHED: 300 秒
  - TCP NEW: 30 秒
  - UDP: 30 秒
  - ICMP: 10 秒

### 3.3 入侵检测 (P2 - 可选)

#### FR-009: SYN Flood 检测
- **描述**：检测 SYN Flood 攻击
- **规则**：
  - 同一源 IP 在短时间内发送大量 SYN 包
  - 阈值：100 个 SYN/秒
- **动作**：告警并记录

#### FR-010: 端口扫描检测
- **描述**：检测端口扫描行为
- **规则**：
  - 同一源 IP 访问多个不同端口
  - 阈值：20 个端口/分钟
- **动作**：告警并记录

#### FR-011: 异常包检测
- **描述**：检测异常的数据包
- **规则**：
  - 包大小异常（过大或过小）
  - TTL 异常
  - 标志位异常
- **动作**：告警并记录

### 3.4 日志记录 (P1 - 重要)

#### FR-012: 包日志
- **描述**：记录所有捕获的数据包
- **格式**：
  ```
  [时间戳] [动作] [协议] [源IP:端口] → [目的IP:端口] [长度]
  ```
- **示例**：
  ```
  [2024-01-01 12:00:00] [ACCEPT] [TCP] [192.168.1.100:12345] → [8.8.8.8:80] [64]
  ```

#### FR-013: 规则日志
- **描述**：记录规则匹配信息
- **格式**：
  ```
  [时间戳] [规则ID] [匹配字段] [动作]
  ```

#### FR-014: 告警日志
- **描述**：记录入侵检测告警
- **格式**：
  ```
  [时间戳] [告警类型] [源IP] [详情]
  ```

## 4. 非功能需求

### 4.1 性能需求

#### NFR-001: 吞吐量
- **目标**：处理 10,000 包/秒
- **度量**：在 1Gbps 网络上无明显延迟

#### NFR-002: 延迟
- **目标**：每包处理延迟 < 1ms
- **度量**：99% 的包在 1ms 内处理完成

#### NFR-003: 内存使用
- **目标**：运行时内存 < 50MB
- **度量**：跟踪 10,000 个连接时的内存使用

### 4.2 可靠性需求

#### NFR-004: 稳定性
- **目标**：连续运行 24 小时不崩溃
- **度量**：无内存泄漏，无段错误

#### NFR-005: 错误处理
- **目标**：优雅处理所有错误情况
- **包括**：
  - 权限不足
  - 网络接口不存在
  - 配置文件错误

### 4.3 可用性需求

#### NFR-006: 配置简单
- **目标**：5 分钟内完成基本配置
- **方式**：配置文件 + 命令行参数

#### NFR-007: 日志可读
- **目标**：日志易于理解和分析
- **方式**：结构化日志，支持不同级别

## 5. 接口需求

### 5.1 命令行接口

```
Usage: firewall [OPTIONS]

Options:
  -c, --config FILE    配置文件路径
  -i, --interface IF   网络接口
  -v, --verbose        详细输出
  -d, --daemon         后台运行
  -l, --log FILE       日志文件路径
  -h, --help           显示帮助
```

### 5.2 配置文件接口

```conf
# 规则配置
[RULES]
# 允许已建立的连接
ACCEPT tcp ESTABLISHED

# 允许 HTTP
ACCEPT tcp dst_port 80

# 允许 DNS
ACCEPT udp dst_port 53

# 默认拒绝
DROP all all

# 日志配置
[LOG]
level = INFO
file = /var/log/firewall.log

# IDS 配置
[IDS]
syn_flood_threshold = 100
port_scan_threshold = 20
```

### 5.3 日志输出接口

```json
{
  "timestamp": "2024-01-01T12:00:00Z",
  "action": "ACCEPT",
  "protocol": "TCP",
  "src_ip": "192.168.1.100",
  "src_port": 12345,
  "dst_ip": "8.8.8.8",
  "dst_port": 80,
  "length": 64,
  "rule_id": 2
}
```

## 6. 约束条件

### 6.1 技术约束
- 使用 C 语言实现
- 依赖 Linux Netfilter 框架
- 需要 root 权限运行

### 6.2 平台约束
- 仅支持 Linux 操作系统
- 需要内核版本 >= 3.10
- 需要安装 libnetfilter-queue

### 6.3 资源约束
- 单线程实现（简化版本）
- 内存使用 < 100MB
- CPU 使用 < 50%
