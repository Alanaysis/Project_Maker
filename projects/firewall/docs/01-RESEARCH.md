# 市场调研报告

## 1. 防火墙技术概述

防火墙是网络安全的第一道防线，用于监控和控制进出网络的流量。根据实现方式，防火墙可分为：

- **软件防火墙**：运行在操作系统上的应用程序
- **硬件防火墙**：专用的网络设备
- **内核级防火墙**：集成在操作系统内核中的过滤模块

## 2. 同类型项目分析

### 2.1 iptables

**项目地址**：https://netfilter.org/iptables/

**核心特点**：
- Linux 内核的包过滤框架
- 基于表（table）、链（chain）、规则（rule）的三层结构
- 支持 IPv4 和 IPv6
- 支持 NAT、连接跟踪等功能

**技术架构**：
```
┌─────────────────────────────────────────────┐
│                  用户空间                    │
│  ┌─────────────────────────────────────┐   │
│  │           iptables 命令行           │   │
│  └─────────────────────────────────────┘   │
│                    ↓ libiptc                │
│  ┌─────────────────────────────────────┐   │
│  │           xtables 模块              │   │
│  └─────────────────────────────────────┘   │
├─────────────────────────────────────────────┤
│                  内核空间                    │
│  ┌─────────────────────────────────────┐   │
│  │         Netfilter 框架              │   │
│  │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐  │   │
│  │  │Pre  │ │Forw │ │Local│ │Post │  │   │
│  │  │Rout │ │ard  │ │In/Out│ │Rout │  │   │
│  │  └─────┘ └─────┘ └─────┘ └─────┘  │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

**规则表**：
- **filter**：过滤表，用于包过滤
- **nat**：网络地址转换
- **mangle**：修改包头
- **raw**：连接跟踪豁免

**优缺点**：
- ✅ 成熟稳定，社区支持好
- ✅ 功能丰富，扩展性强
- ❌ 规则复杂，学习曲线陡峭
- ❌ 规则更新需要刷新整个链

### 2.2 nftables

**项目地址**：https://netfilter.org/projects/nftables/

**核心特点**：
- iptables 的继任者
- 统一的命令行工具 `nft`
- 基于虚拟机的字节码执行
- 支持集合（set）和映射（map）

**技术架构**：
```
┌─────────────────────────────────────────────┐
│                  用户空间                    │
│  ┌─────────────────────────────────────┐   │
│  │           nft 命令行工具            │   │
│  └─────────────────────────────────────┘   │
│                    ↓ libnftables            │
│  ┌─────────────────────────────────────┐   │
│  │         nftables 虚拟机             │   │
│  └─────────────────────────────────────┘   │
├─────────────────────────────────────────────┤
│                  内核空间                    │
│  ┌─────────────────────────────────────┐   │
│  │         Netfilter 框架              │   │
│  │  ┌─────────────────────────────┐   │   │
│  │  │      nftables 内核模块      │   │   │
│  │  └─────────────────────────────┘   │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

**与 iptables 的对比**：
| 特性 | iptables | nftables |
|------|----------|----------|
| 命令工具 | 多个 (iptables, ip6tables, etc.) | 单个 (nft) |
| 规则更新 | 刷新整个链 | 原子更新 |
| 匹配机制 | 线性扫描 | 集合/映射 |
| 性能 | 一般 | 更好 |
| 扩展性 | 模块化 | 更灵活 |

### 2.3 Suricata

**项目地址**：https://github.com/OISF/suricata

**核心特点**：
- 高性能入侵检测/防御系统 (IDS/IPS)
- 多线程架构
- 支持多种流量获取方式
- 强大的规则语言

**技术架构**：
```
┌─────────────────────────────────────────────────────┐
│                    Suricata 引擎                     │
├─────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐         │
│  │ 数据包   │  │ 规则     │  │ 日志     │         │
│  │ 获取层   │→│ 检测层   │→│ 输出层   │         │
│  └──────────┘  └──────────┘  └──────────┘         │
│       ↑              ↑              ↑               │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐         │
│  │ libpcap  │  │ 流量     │  │ EVE JSON │         │
│  │ AF_PACKET│  │ 引擎     │  │ Syslog   │         │
│  │ Netmap   │  │ 协议解析 │  │ 文件     │         │
│  └──────────┘  └──────────┘  └──────────┘         │
└─────────────────────────────────────────────────────┘
```

**检测能力**：
- 签名检测（规则匹配）
- 协议异常检测
- 文件提取和分析
- TLS/SSL 检测

**优缺点**：
- ✅ 高性能，多线程
- ✅ 检测能力强
- ✅ 社区活跃
- ❌ 资源消耗大
- ❌ 配置复杂

### 2.4 Snort

**项目地址**：https://github.com/snort3/snort3

**核心特点**：
- 老牌入侵检测系统
- 规则语言成熟
- 广泛的社区支持

**与 Suricata 的对比**：
| 特性 | Snort | Suricata |
|------|-------|----------|
| 多线程 | 有限 | 完整 |
| 性能 | 一般 | 更好 |
| 规则兼容 | 原创 | 兼容 Snort |
| 协议支持 | 基础 | 更丰富 |
| 活跃度 | 维护模式 | 活跃开发 |

## 3. 技术变体和演进路径

### 3.1 包过滤技术演进

```
静态包过滤 (1980s)
    ↓
动态包过滤 (1990s)
    ↓
状态检测 (Stateful Inspection) (2000s)
    ↓
应用层过滤 (Application Layer) (2010s)
    ↓
下一代防火墙 (NGFW) (2020s)
```

### 3.2 Linux 防火墙演进

```
ipchains (Linux 2.2)
    ↓
iptables (Linux 2.4)
    ↓
nftables (Linux 3.13+)
    ↓
eBPF (现代)
```

### 3.3 关键技术点

**状态检测 (Stateful Inspection)**：
- 跟踪连接状态（TCP 状态机）
- 动态打开/关闭端口
- 防止 IP 欺骗

**应用层过滤**：
- 深度包检测 (DPI)
- 协议解析（HTTP, DNS, TLS）
- 内容过滤

**硬件加速**：
- FPGA 实现
- 网卡卸载
- DPDK 用户态网络

## 4. 技术选型建议

### 对于本项目

**推荐方案**：
- 使用 `libnetfilter_queue` 实现用户态包处理
- 实现基本的包过滤和状态检测
- 添加简单的入侵检测规则

**理由**：
1. 用户态实现，调试方便
2. 不需要编写内核模块
3. 性能足够用于学习
4. 代码可读性好

**备选方案**：
- 使用 raw socket + libpcap：更底层，但功能受限
- 编写内核模块：性能最好，但风险高

### 学习路径建议

```
初级：理解 IP/TCP/UDP 协议
    ↓
中级：使用 libpcap 捕获包
    ↓
高级：使用 Netfilter 拦截包
    ↓
专家：实现状态检测和入侵检测
```

## 6. 总结

通过市场调研，我们发现：

1. **Netfilter 仍是主流**：iptables/nftables 是 Linux 防火墙的基石
2. **状态检测是标配**：现代防火墙都支持有状态检测
3. **IDS/IPS 融合**：防火墙和入侵检测的界限越来越模糊
4. **性能是关键**：高性能包处理是核心挑战

对于学习项目，建议从简单的包过滤开始，逐步添加状态检测和入侵检测功能。
