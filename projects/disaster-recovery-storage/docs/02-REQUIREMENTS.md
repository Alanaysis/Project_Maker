# 需求分析文档：容灾数据存储系统

## 1. 项目概述

### 1.1 项目目标
构建一个用于学习分布式存储系统核心原理的教学项目，通过实践理解：
- 数据冗余和纠删码
- 跨数据中心复制
- 故障检测和自动恢复

### 1.2 项目定位
**学习型项目** - 非生产环境使用，重点在于理解原理和实现细节

## 3. 功能需求

### 3.1 核心功能

#### FR-001: 数据写入
**描述**: 支持将数据写入存储系统
**优先级**: P0 (必须)
**详细需求**:
- 支持任意大小的数据块
- 数据自动分片
- 支持纠删码编码
- 支持多副本写入
- 返回写入状态

**验收标准**:
- [ ] 可以写入1KB-1GB的数据
- [ ] 数据正确编码和存储
- [ ] 写入操作有确认返回

#### FR-002: 数据读取
**描述**: 支持从存储系统读取数据
**优先级**: P0 (必须)
**详细需求**:
- 根据数据ID读取
- 自动重组分片
- 支持纠删码解码
- 处理部分节点不可用

**验收标准**:
- [ ] 可以正确读取写入的数据
- [ ] 部分节点故障时仍可读取
- [ ] 数据完整性校验

#### FR-003: 纠删码编码
**描述**: 实现Reed-Solomon纠删码
**优先级**: P0 (必须)
**详细需求**:
- 实现GF(2^8)有限域运算
- 支持配置k(数据分片数)和m(校验分片数)
- 编码和解码功能
- 支持任意数据大小

**验收标准**:
- [ ] 正确实现有限域四则运算
- [ ] 编码后可以正确解码
- [ ] 支持丢失最多m个分片后恢复

#### FR-004: 多副本存储
**描述**: 支持数据的多副本冗余
**优先级**: P0 (必须)
**详细需求**:
- 可配置副本数(默认3)
- 副本放置策略
- 副本一致性检查
- 副本修复

**验收标准**:
- [ ] 数据按配置存储多个副本
- [ ] 可以检测副本不一致
- [ ] 可以修复缺失的副本

#### FR-005: 故障检测
**描述**: 检测存储节点的故障
**优先级**: P1 (重要)
**详细需求**:
- 心跳检测机制
- 可配置超时时间
- 故障节点标记
- 故障恢复检测

**验收标准**:
- [ ] 可以检测节点停机
- [ ] 可以检测节点恢复
- [ ] 误报率低于5%

#### FR-006: 数据分片
**描述**: 将大数据分割成小块
**优先级**: P0 (必须)
**详细需求**:
- 可配置分片大小(默认64KB)
- 分片元数据管理
- 分片重组
- 处理最后一个分片

**验收标准**:
- [ ] 大数据正确分片
- [ ] 分片可以正确重组
- [ ] 支持任意大小数据

### 3.2 扩展功能

#### FR-007: 数据删除
**描述**: 支持删除已存储的数据
**优先级**: P2 (可选)
**详细需求**:
- 根据ID删除
- 清理所有分片和副本
- 确认删除

#### FR-008: 数据更新
**描述**: 支持更新已存储的数据
**优先级**: P2 (可选)
**详细需求**:
- 原子更新
- 版本管理
- 并发控制

#### FR-009: 数据校验
**描述**: 验证数据完整性
**优先级**: P1 (重要)
**详细需求**:
- CRC32校验
- SHA256校验
- 自动校验

## 6. 用例场景

### 6.1 基本用例

#### UC-001: 存储文件
```
参与者: 用户
前置条件: 存储系统已启动
主要流程:
1. 用户调用write(data)
2. 系统分片数据
3. 系统纠删码编码
4. 系统写入多个节点
5. 系统返回成功
异常流程:
- 节点故障: 重试或使用其他节点
- 写入失败: 返回错误
```

#### UC-002: 读取文件
```
参与者: 用户
前置条件: 数据已存储
主要流程:
1. 用户调用read(id)
2. 系统定位数据分片
3. 系统从节点读取
4. 系统重组数据
5. 系统返回数据
异常流程:
- 节点故障: 从其他副本读取
- 数据损坏: 使用纠删码恢复
```

#### UC-003: 节点故障恢复
```
参与者: 系统
前置条件: 数据已存储
主要流程:
1. 系统检测到节点故障
2. 系统标记节点为不可用
3. 系统启动数据重建
4. 系统将数据复制到新节点
5. 系统更新元数据
```

### 6.2 边界用例

#### UC-004: 并发写入同一数据
- 使用乐观锁或版本控制
- 最后写入者胜出或合并

#### UC-005: 网络分区
- 检测网络分区
- 降级服务或等待恢复

#### UC-006: 磁盘空间不足
- 检测磁盘空间
- 触发数据迁移或清理

## 7. 数据模型

### 7.1 核心实体

#### Object (数据对象)
```
{
    id: string (唯一标识)
    size: uint64 (数据大小)
    checksum: string (校验和)
    create_time: timestamp
    update_time: timestamp
    version: uint32
}
```

#### Chunk (数据分片)
```
{
    id: string (唯一标识)
    object_id: string (所属对象)
    index: uint32 (分片序号)
    data: bytes (分片数据)
    checksum: string (校验和)
    is_parity: bool (是否校验块)
}
```

#### Node (存储节点)
```
{
    id: string (唯一标识)
    address: string (网络地址)
    port: uint16 (端口)
    status: enum (ONLINE, OFFLINE, DEGRADED)
    capacity: uint64 (总容量)
    used: uint64 (已用容量)
    last_heartbeat: timestamp
}
```

#### Replica (副本信息)
```
{
    chunk_id: string
    node_id: string
    status: enum (HEALTHY, DEGRADED, LOST)
    last_verify: timestamp
}
```

## 8. 接口设计

### 8.1 客户端接口

```cpp
class StorageClient {
public:
    // 写入数据
    Status write(const std::string& key, const std::vector<uint8_t>& data);

    // 读取数据
    Status read(const std::string& key, std::vector<uint8_t>& data);

    // 删除数据
    Status remove(const std::string& key);

    // 检查数据是否存在
    bool exists(const std::string& key);

    // 获取数据信息
    ObjectInfo stat(const std::string& key);
};
```

### 8.2 存储节点接口

```cpp
class StorageNode {
public:
    // 存储数据块
    Status storeChunk(const Chunk& chunk);

    // 读取数据块
    Status readChunk(const std::string& chunk_id, Chunk& chunk);

    // 删除数据块
    Status removeChunk(const std::string& chunk_id);

    // 心跳
    Status heartbeat();

    // 获取状态
    NodeStatus getStatus();
};
```

### 8.3 纠删码接口

```cpp
class ErasureCoder {
public:
    // 编码
    Status encode(const std::vector<uint8_t>& data,
                  std::vector<std::vector<uint8_t>>& shards);

    // 解码
    Status decode(const std::vector<std::vector<uint8_t>>& shards,
                  std::vector<uint8_t>& data);

    // 配置
    void configure(int k, int m);
};
```
