# 流媒体服务器产品思考

## 学习目标

### 1. 协议理解

通过本项目，深入理解以下协议：

- **RTMP 协议**：握手、消息分块、AMF 编码、推流/拉流流程
- **HLS 协议**：M3U8 格式、TS 切片、自适应码率机制
- **DASH 协议**：MPD 格式、分段 MP4、多码率支持
- **WebRTC 协议**：信令、ICE/STUN/TURN、SRTP 加密
- **RTSP 协议**：请求/响应、RTP/RTCP、SDP 协商

### 2. 媒体处理

掌握音视频处理核心技术：

- **视频编码**：H.264/H.265 编码原理、参数配置
- **音频编码**：AAC/Opus 编码原理、采样率转换
- **容器封装**：MP4/FLV/TS 格式、封装流程
- **切片生成**：关键帧对齐、切片时长控制

### 3. 网络编程

掌握高性能网络编程技术：

- **异步 I/O**：epoll 事件驱动、非阻塞 socket
- **连接管理**：连接池、超时处理、状态跟踪
- **流量控制**：拥塞控制、带宽估计、自适应码率

### 4. 系统设计

理解分布式系统设计：

- **负载均衡**：多种算法、健康检查、动态调度
- **集群管理**：节点发现、状态同步、故障恢复
- **缓存策略**：LRU 淘汰、多级缓存、预加载

## 关键要点

### 1. 协议设计要点

#### RTMP 协议
```
要点1: 握手流程
- C0/C1: 客户端发送版本和随机数据
- S0/S1/S2: 服务器响应版本和回显
- C2: 客户端确认

要点2: 消息分块
- 将大消息分成小块传输
- 支持不同的块头格式（fmt 0-3）
- 减少网络开销

要点3: AMF 编码
- 支持多种数据类型
- 紧凑的二进制格式
- 用于命令和数据传输
```

#### HLS 协议
```
要点1: M3U8 格式
- 主播放列表：多码率流
- 媒体播放列表：切片列表
- 标签：#EXTINF, #EXT-X-TARGETDURATION

要点2: TS 切片
- 关键帧对齐
- PAT/PMT 表
- PES 封装

要点3: 自适应码率
- 根据网络状况切换
- 多码率流
- 带宽估计
```

#### WebRTC 协议
```
要点1: 信令流程
- Offer/Answer 交换
- ICE 候选交换
- 会话建立

要点2: ICE/STUN/TURN
- NAT 穿透
- 候选收集
- 连通性检查

要点3: SRTP 加密
- 密钥交换
- 数据加密
- 完整性保护
```
