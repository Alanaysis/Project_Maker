# 风险管理引擎 - 研究文档

## 1. 风险管理概述

### 1.1 什么是风险管理？

风险管理是识别、评估和控制对组织资本和收益的威胁的过程。这些威胁或风险可能来自多种来源，包括财务不确定性、法律负债、战略管理错误、事故和自然灾害。

### 1.2 金融风险管理

在金融领域，风险管理涉及识别和分析投资决策中的损失风险，并采取适当的步骤来管理这些风险。

## 2. 风险度量方法

### 2.1 Value at Risk (VaR)

VaR 是最广泛使用的风险度量方法之一，用于量化在给定置信水平下，特定时间范围内投资组合可能遭受的最大损失。

**定义**：在给定的置信水平 α 下，VaR 是使得损失超过该值的概率为 1-α 的损失水平。

**数学表示**：
```
P(Loss > VaR_α) = 1 - α
```

**示例**：95% 的日 VaR 为 100 万美元意味着：
- 在 95% 的情况下，每日损失不会超过 100 万美元
- 在 5% 的情况下，损失可能超过 100 万美元

### 2.2 Conditional VaR (CVaR / Expected Shortfall)

CVaR 是 VaR 的扩展，它衡量的是当损失超过 VaR 时的预期损失。

**优势**：
- 比 VaR 更保守
- 满足一致性风险度量的次可加性
- 能更好地捕捉尾部风险

**数学表示**：
```
CVaR_α = E[Loss | Loss > VaR_α]
```

### 2.3 波动率

波动率是衡量资产价格变动幅度的统计指标，通常用收益率的标准差表示。

**类型**：
- 历史波动率：基于历史数据计算
- 隐含波动率：从期权价格反推

### 2.4 Beta

Beta 衡量资产相对于市场整体的系统性风险。

**解读**：
- Beta = 1：资产与市场波动一致
- Beta > 1：资产比市场波动更大
- Beta < 1：资产比市场波动更小

## 3. VaR 计算方法

### 3.1 历史模拟法 (Historical Simulation)

**原理**：直接使用历史收益率分布来估计 VaR。

**步骤**：
1. 收集历史收益率数据
2. 将收益率从小到大排序
3. 找到对应置信水平的分位数

**优点**：
- 不需要假设特定分布
- 能捕捉实际的分布特征
- 实现简单

**缺点**：
- 依赖历史数据的代表性
- 不能预测未发生过的事件
- 对数据窗口长度敏感

### 3.2 参数法 (Parametric / Variance-Covariance)

**原理**：假设收益率服从正态分布，使用均值和方差来估计 VaR。

**公式**：
```
VaR_α = μ + z_α × σ
```
其中 z_α 是标准正态分布的 α 分位数。

**优点**：
- 计算快速
- 易于理解和实现
- 适合大组合

**缺点**：
- 依赖正态分布假设
- 低估尾部风险
- 对非线性产品不准确

### 3.3 蒙特卡洛模拟法 (Monte Carlo Simulation)

**原理**：通过随机模拟生成大量可能的收益率路径，然后计算 VaR。

**步骤**：
1. 估计收益率分布的参数
2. 生成大量随机收益率
3. 计算模拟的损失分布
4. 从模拟分布中计算 VaR

**优点**：
- 可以处理复杂的分布
- 能模拟非线性产品
- 灵活性高

**缺点**：
- 计算量大
- 模型风险
- 需要选择合适的随机过程

## 4. 压力测试

### 4.1 什么是压力测试？

压力测试是一种风险管理技术，用于评估投资组合在极端市场条件下的表现。

### 4.2 压力测试类型

#### 历史情景压力测试
使用历史上发生过的危机事件作为情景。

**常见情景**：
- 2008年全球金融危机
- 2020年新冠疫情冲击
- 2000年互联网泡沫破裂
- 1987年黑色星期一

#### 假设情景压力测试
设计假设的极端情景进行测试。

**常见情景**：
- 股市大幅下跌
- 利率急剧上升
- 汇率剧烈波动
- 流动性枯竭

#### 反向压力测试
从目标损失出发，反向推导需要多大的冲击才能达到该损失。

**用途**：
- 识别脆弱点
- 设定风险限额
- 制定应急计划

### 4.3 敏感性分析

分析单个风险因子变化对投资组合的影响。

**用途**：
- 识别主要风险因子
- 评估风险集中度
- 优化风险敞口
