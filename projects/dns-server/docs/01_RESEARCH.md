# DNS 技术调研

## 1. DNS 技术历史

### 1.1 发展历程

| 时间 | 事件 | 说明 |
|------|------|------|
| 1983 | RFC 882/883 | DNS 首次定义 |
| 1987 | RFC 1034/1035 | DNS 标准化 |
| 1999 | RFC 2535 | DNSSEC 首次提出 |
| 2005 | RFC 4034 | DNSSEC 修订 |
| 2010 | RFC 5936 | AXFR 规范 |
| 2013 | RFC 6891 | EDNS0 扩展 |
| 2018 | RFC 8484 | DNS over HTTPS |
| 2020 | RFC 8490 | DNS over TLS |

### 1.2 协议演进

```
DNS (1983)
  ↓
Dynamic DNS (1997)
  ↓
DNSSEC (2005)
  ↓
EDNS0 (2013)
  ↓
DoH/DoT (2018)
  ↓
DNS over QUIC (2021)
```

## 3. DNS 记录类型

### 3.1 常用记录类型

| 类型 | 用途 | 示例 |
|------|------|------|
| A | IPv4 地址 | example.com → 93.184.216.34 |
| AAAA | IPv6 地址 | example.com → 2606:2800:220:1:248:1893:25c8:1946 |
| CNAME | 别名 | www.example.com → example.com |
| MX | 邮件交换 | example.com → mail.example.com (优先级 10) |
| NS | 域名服务器 | example.com → ns1.example.com |
| TXT | 文本记录 | "v=spf1 mx ~all" |
| SRV | 服务记录 | _http._tcp.example.com → www.example.com:80 |
| PTR | 反向解析 | 34.216.184.93.in-addr.arpa → example.com |

### 3.2 DNSSEC 记录类型

| 类型 | 用途 | 说明 |
|------|------|------|
| DNSKEY | 公钥 | 用于验证签名 |
| RRSIG | 签名 | RRSET 的数字签名 |
| DS | 委托签名 | 父区域对子区域的信任 |
| NSEC | 否定证明 | 证明域名不存在 |
| NSEC3 | 否定证明 v3 | 防止区域遍历 |

## 4. DNS 服务器类型

### 4.1 权威 DNS 服务器

**功能**：
- 存储域名的权威数据
- 响应来自递归服务器的查询
- 提供区域传输

**实现**：
- BIND (ISC)
- NSD/Unbound (NLnet Labs)
- PowerDNS
- Knot DNS

### 4.2 递归 DNS 服务器

**功能**：
- 接收客户端查询
- 执行递归解析
- 缓存结果

**实现**：
- Unbound
- BIND
- dnsmasq
- CoreDNS

### 4.3 转发 DNS 服务器

**功能**：
- 转发查询到上游服务器
- 缓存响应
- 提供过滤功能

**应用**：
- 企业内部 DNS
- ISP DNS
- 公共 DNS（8.8.8.8, 1.1.1.1）

## 5. 优缺点分析

### 5.1 优点

| 优点 | 说明 |
|------|------|
| 分布式 | 无单点故障 |
| 可扩展 | 层次化结构 |
| 高效 | 缓存机制 |
| 灵活 | 多种记录类型 |
| 标准化 | 广泛支持 |

### 5.2 缺点

| 缺点 | 说明 | 解决方案 |
|------|------|----------|
| 安全性 | 易受攻击 | DNSSEC |
| 隐私 | 查询明文 | DoH/DoT |
| 延迟 | 多级查询 | 缓存/预取 |
| 复杂性 | 配置复杂 | 自动化工具 |

## 6. 安全威胁

### 6.1 常见攻击

| 攻击类型 | 说明 | 防御措施 |

| DNS 欺骗 | 伪造响应 | DNSSEC |
| DDoS | 流量攻击 | 速率限制 |

| DNS 隧道 | 数据外泄 | 深度检测 |

| 技术 | 说明 | 标准 |
|------|------|------|
| DNSSEC | 签名验证 | RFC 4034 |

## 7. 性能优化

### 7.1 缓存策略

| 策略 | 说明 | 效果 |
|------|------|------|
| TTL 管理 | 合理设置过期时间 | 平衡新鲜度和性能 |
| 预取 | 提前刷新热点记录 | 减少延迟 |
| 负缓存 | 缓存否定回答 | 减少无效查询 |
| 分层缓存 | 多级缓存架构 | 提高命中率 |

### 7.2 并发处理

| 技术 | 说明 | 优势 |
|------|------|------|
| 异步 I/O | epoll/kqueue | 高并发 |
| 线程池 | 工作线程 | 资源复用 |
| 连接池 | TCP 连接 | 减少开销 |
| 无锁结构 | 原子操作 | 减少竞争 |
