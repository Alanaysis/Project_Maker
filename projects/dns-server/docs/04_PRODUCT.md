## 2. 关键要点

### 2.1 DNS 协议要点

#### 报文格式

```
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                      ID                           |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|QR|   Opcode  |AA|TC|RD|RA|   Z    |   RCODE      |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    QDCOUNT                        |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    ANCOUNT                        |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    NSCOUNT                        |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    ARCOUNT                        |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
```

**关键字段**：
- ID: 事务标识符
- QR: 查询(0)/响应(1)
- OPCODE: 操作码
- AA: 权威回答
- TC: 截断标志
- RD: 期望递归
- RA: 可用递归
- RCODE: 响应码

#### 域名编码

```
www.example.com
  ↓
[3]www[7]example[3]com[0]
```

**规则**：
- 每个标签以长度前缀
- 标签最大长度 63 字节
- 域名最大长度 255 字节
- 支持压缩指针

### 2.2 解析流程要点

#### 递归解析

```
Client → Recursive Server → Root Server
                           ↓
                    TLD Server
                           ↓
                    Auth Server
                           ↓
                    Response
```

**关键点**：
- 递归服务器承担解析责任
- 缓存减少查询次数
- CNAME 追踪
- Glue Records 处理

#### 迭代解析

```
Client → Recursive Server
              ↓
         Root Server → 返回 TLD NS
              ↓
         TLD Server → 返回 Auth NS
              ↓
         Auth Server → 返回结果
```

**关键点**：
- 客户端控制解析过程
- 每级返回下一级 NS
- 需要解析 NS 地址

### 2.3 缓存设计要点

#### LRU 策略

```
Recent ◄──► Entry1 ◄──► Entry2 ◄──► ... ◄──► EntryN ◄──► Old
   │                                                    │
   └──────────── Hash Map (O(1) lookup) ───────────────┘
```

**关键点**：
- O(1) 查找
- O(1) 插入/删除
- 自动淘汰旧数据
- TTL 管理

#### 负缓存

**作用**：
- 缓存否定回答
- 减少无效查询
- 防止缓存投毒

### 2.4 安全要点

#### DNSSEC 验证链

```
Root DNSKEY
    ↓ (DS)
TLD DNSKEY
    ↓ (DS)
Domain DNSKEY
    ↓ (RRSIG)
RRSET
```

**关键点**：
- 信任链传递
- 签名验证
- 时间有效性

#### 访问控制

**层次**：
1. IP 地址过滤
2. ACL 规则匹配
3. 记录类型过滤
4. 操作权限控制

### 2.5 性能要点

#### 异步 I/O 模型

```
┌─────────────────────────────────────┐
│          Event Loop (epoll)         │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐  │
│  │ FD1 │ │ FD2 │ │ FD3 │ │ FD4 │  │
│  └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘  │
│     │       │       │       │      │
│     ▼       ▼       ▼       ▼      │
│  Handle  Handle  Handle  Handle    │
│  Read    Read    Read    Read      │
└─────────────────────────────────────┘
```

**优势**：
- 单线程处理多连接
- 避免线程切换开销
- 高并发支持

#### 线程池模型

```
┌─────────────────────────────────────┐
│           Task Queue                │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐  │
│  │Task1│ │Task2│ │Task3│ │Task4│  │
│  └─────┘ └─────┘ └─────┘ └─────┘  │
└──────────────────┬──────────────────┘
                   │
    ┌──────────────┼──────────────┐
    │              │              │
    ▼              ▼              ▼
┌───────┐    ┌───────┐    ┌───────┐
│Worker1│    │Worker2│    │Worker3│
└───────┘    └───────┘    └───────┘
```

**优势**：
- 资源复用
- 负载均衡
- 避免线程爆炸

## 3. 设计模式应用

### 3.1 单例模式

```cpp
class Logger {
public:
    static Logger& instance() {
        static Logger logger;
        return logger;
    }
};
```

**应用**：日志器、统计收集器

### 3.2 工厂模式

```cpp
class DnsMessage {
public:
    static DnsMessage create_query(const std::string& name, RecordType type);
    static DnsMessage create_response(const DnsMessage& query, ResponseCode rcode);
};
```

**应用**：报文创建

### 3.3 策略模式

```cpp
enum class ForwardStrategy {
    FIRST,
    ROUND_ROBIN,
    RANDOM,
    FASTEST,
};
```

**应用**：转发策略、负载均衡

### 3.4 观察者模式

```cpp
using DnsRequestHandler = std::function<DnsMessage(...)>;
using EventCallback = std::function<void(int fd, EventType type)>;
```

**应用**：事件处理、回调机制

### 3.5 装饰器模式

```cpp
auto wrapped_handler = [this](...) {
    // 前置处理 (ACL, 限流)
    auto response = handler_(request, client_addr, client_port);
    // 后置处理 (日志, 统计)
    return response;
};
```

**应用**：请求处理链

## 4. 工程实践

### 4.1 代码组织

**原则**：
- 单一职责
- 高内聚低耦合
- 接口隔离
- 依赖倒置

**实践**：
- 头文件/源文件分离
- 命名空间组织
- 清晰的模块边界

### 4.2 错误处理

**策略**：
- 使用 `std::optional` 表示可能失败
- 异常用于真正异常情况
- 错误码用于可恢复错误
- 日志记录所有错误

### 4.3 内存管理

**策略**：
- 优先使用智能指针
- RAII 管理资源
- 避免裸指针
- 移动语义减少拷贝

### 4.4 线程安全

**策略**：
- 细粒度锁
- 避免死锁
- 使用原子操作
- 无锁数据结构
