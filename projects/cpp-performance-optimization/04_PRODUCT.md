## 2. 关键要点

### 2.1 内存优化要点

#### 缓存友好设计
- **关键洞察**: CPU 访问内存比访问缓存慢 100 倍
- **最佳实践**:
  - 使用连续内存容器 (vector, array)
  - 按行优先顺序访问二维数组
  - 避免随机访问模式
  - 利用数据局部性

#### 数据布局 (SoA vs AoS)
- **关键洞察**: 数据布局影响缓存利用率
- **最佳实践**:
  - 遍历场景使用 SoA
  - 对象场景使用 AoS
  - 混合场景考虑混合布局

#### 内存对齐
- **关键洞察**: 非对齐访问可能跨缓存行
- **最佳实践**:
  - 使用 alignas 确保对齐
  - SIMD 数据必须对齐
  - 理解结构体填充

#### 预取技术
- **关键洞察**: 预取可以隐藏内存延迟
- **最佳实践**:
  - 顺序访问时硬件自动预取
  - 随机访问时使用软件预取
  - 预取距离需要调优

#### 内存池
- **关键洞察**: 频繁分配小对象开销大
- **最佳实践**:
  - 对象池减少分配次数
  - 固定大小分配器高效
  - 减少内存碎片

### 2.2 编译器优化要点

#### 内联优化
- **关键洞察**: 函数调用有开销
- **最佳实践**:
  - 小函数标记 inline
  - 信任编译器的内联决策
  - 使用 LTO 跨模块内联

#### 向量化
- **关键洞察**: SIMD 可以同时处理多个数据
- **最佳实践**:
  - 使用连续内存
  - 避免循环依赖
  - 使用对齐内存
  - 帮助编译器识别向量化机会

#### 分支预测
- **关键洞察**: 分支预测失败代价高
- **最佳实践**:
  - 使用 [[likely]]/[[unlikely]]
  - 无分支编程技巧
  - 排序数据提高预测率

### 2.3 算法优化要点

#### 时间复杂度
- **关键洞察**: 算法选择影响数量级性能
- **最佳实践**:
  - O(n) 优于 O(n²)
  - 哈希表 O(1) 查找
  - 二分查找 O(log n)

#### 常数因子
- **关键洞察**: 常数因子也很重要
- **最佳实践**:
  - 减少循环内操作
  - 强度削减
  - 查找表

#### 并行化
- **关键洞察**: 多核可以线性加速
- **最佳实践**:
  - 使用并行算法
  - 避免数据竞争
  - 负载均衡

### 2.4 并发优化要点

#### 线程池
- **关键洞察**: 线程创建开销大
- **最佳实践**:
  - 使用线程池复用线程
  - 合理设置线程数
  - 避免线程饥饿

#### 无锁编程
- **关键洞察**: 锁是性能瓶颈
- **最佳实践**:
  - 使用原子操作
  - 理解内存序
  - 避免 ABA 问题

#### 伪共享
- **关键洞察**: 不同变量可能在同一缓存行
- **最佳实践**:
  - 使用填充隔离变量
  - 使用 thread_local
  - 使用硬件干扰距离

### 2.5 I/O 优化要点

#### 缓冲 I/O
- **关键洞察**: 系统调用开销大
- **最佳实践**:
  - 使用缓冲减少系统调用
  - 批量读写
  - 合理设置缓冲区大小

#### 异步 I/O
- **关键洞察**: I/O 等待浪费 CPU
- **最佳实践**:
  - 使用异步 I/O
  - 重叠计算和 I/O
  - 使用 I/O 线程池

#### 内存映射
- **关键洞察**: mmap 避免数据拷贝
- **最佳实践**:
  - 大文件使用 mmap
  - 只读访问使用共享映射
  - 注意页面错误

## 3. 优化原则

### 3.1 核心原则

#### 原则 1: 先测量后优化
```
不要猜测瓶颈，用数据说话。
使用性能分析工具定位真正的热点。
```

#### 原则 2: 80/20 法则
```
80% 的时间花在 20% 的代码上。
优化热点代码，忽略冷代码。
```

#### 原则 3: 算法优先
```
选择正确的算法比微优化更重要。
O(n) 算法再怎么优化也比不上 O(log n)。
```

#### 原则 4: 缓存友好
```
现代 CPU 的瓶颈是内存访问。
优化数据访问模式比优化计算更重要。
```

#### 原则 5: 避免过早优化
```
先让代码正确，再让代码快速。
可读性和可维护性同样重要。
```

### 3.2 权衡取舍

#### 性能 vs 可读性
- 热点代码: 优先性能
- 冷代码: 优先可读性
- 关键路径: 适当牺牲可读性

#### 时间 vs 空间
- 内存受限: 空间优先
- 计算密集: 时间优先
- 缓存友好: 两者兼顾

#### 开发效率 vs 运行效率
- 原型阶段: 开发效率优先
- 生产阶段: 运行效率优先
- 优化阶段: 针对性优化

## 4. 常见陷阱

### 4.1 过早优化
**问题**: 在不重要的地方花太多时间优化
**解决**: 先测量，找到真正的瓶颈

### 4.2 忽略编译器优化
**问题**: 手动优化不如编译器优化
**解决**: 使用合适的编译选项，信任编译器

### 4.3 过度优化
**问题**: 牺牲可读性换取微小性能提升
**解决**: 权衡性能收益和代码复杂度

### 4.4 平台相关优化
**问题**: 优化在其他平台无效或更慢
**解决**: 使用可移植的优化，测试多平台

### 4.5 忽略测量
**问题**: 优化后没有验证效果
**解决**: 每次优化都要测量对比

### 4.6 优化错误的代码
**问题**: 在冷代码上花时间优化
**解决**: 使用 profiler 找到热点

### 4.7 忽略算法复杂度
**问题**: 在 O(n²) 算法上做微优化
**解决**: 先选择正确的算法

### 4.8 过度使用 SIMD
**问题**: 所有代码都尝试向量化
**解决**: 只在热点代码使用 SIMD
