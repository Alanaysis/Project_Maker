# 需求分析文档

## 1. 用户画像

### 主要用户：计算机科学学生

**基本特征**:
- 年龄：18-25 岁
- 学历：本科/研究生
- 专业：计算机科学、软件工程
- 编程经验：1-3 年

**学习背景**:
- 学过 C 语言基础
- 了解基本的数据结构
- 学过计算机组成原理
- 对操作系统有初步认识

**学习目标**:
- 理解操作系统的工作原理
- 为考研/工作做准备
- 提升底层编程能力
- 满足技术好奇心

**痛点**:
- 现有资料多为英文
- xv6 代码量大，入门困难
- 缺少中文社区支持
- 理论与实践脱节

### 次要用户：自学者

**基本特征**:
- 已工作的程序员
- 想深入理解底层
- 学习时间碎片化

**需求**:
- 快速入门
- 随时可中断/继续
- 实用性强

## 2. 功能需求

### 2.1 核心功能 (Must Have)

#### 引导加载 (Boot)
- **F-B001**: 主引导扇区实现
  - BIOS 加载引导扇区到 0x7C00
  - 从实模式切换到保护模式
  - 加载内核到内存

- **F-B002**: 加载器实现
  - 读取内核文件
  - 设置 GDT
  - 跳转到内核入口

#### 内核初始化 (Kernel)
- **F-K001**: GDT 初始化
  - 设置代码段和数据段
  - 设置任务状态段

- **F-K002**: IDT 初始化
  - 设置中断描述符表
  - 注册中断处理函数

- **F-K003**: 内核主函数
  - 初始化各子系统
  - 启动第一个进程

#### 内存管理 (Memory)
- **F-M001**: 物理内存管理
  - 记录可用内存
  - 物理页分配和释放

- **F-M002**: 虚拟内存管理
  - 页目录和页表管理
  - 虚拟地址映射

- **F-M003**: 内核堆管理
  - kmalloc/kfree 实现
  - 简单的内存分配器

#### 进程管理 (Process)
- **F-P001**: 进程控制块
  - 进程 ID
  - 进程状态
  - 寄存器上下文
  - 内存映射

- **F-P002**: 进程创建
  - fork 系统调用
  - 进程初始化

- **F-P003**: 进程调度
  - 轮转调度算法
  - 时间片管理

- **F-P004**: 上下文切换
  - 保存/恢复寄存器
  - 切换页表

#### 中断处理 (Interrupt)
- **F-I001**: 异常处理
  - 除零错误
  - 页面错误
  - 通用保护错误

- **F-I002**: 硬件中断
  - 定时器中断
  - 键盘中断

- **F-I003**: 系统调用
  - int 0x80 实现
  - 系统调用分发

#### 设备驱动 (Driver)
- **F-D001**: 屏幕输出
  - VGA 文本模式
  - 字符输出
  - 光标控制

- **F-D002**: 键盘输入
  - 键盘中断处理
  - 扫描码转换
  - 输入缓冲区

#### 文件系统 (File System)
- **F-F001**: 简单文件系统
  - 文件创建/删除
  - 文件读写
  - 目录管理

### 2.2 扩展功能 (Nice to Have)

- **F-E001**: 多核支持
- **F-E002**: 网络协议栈
- **F-E003**: 图形界面
- **F-E004**: 设备热插拔

## 4. 用例场景

### 用例 1：学生首次使用

**参与者**: 计算机科学学生

**前置条件**:
- 已安装 Ubuntu Linux
- 有基本的 C 语言基础

**主要流程**:
1. 克隆项目仓库
2. 阅读 README.md
3. 安装依赖工具
4. 编译内核
5. 在 QEMU 中运行
6. 看到控制台输出 "Hello, Simple OS!"

**后置条件**:
- 成功运行第一个 OS 程序
- 对 OS 有初步认识

### 用例 2：学习引导过程

**参与者**: 想了解启动过程的学生

**前置条件**:
- 已成功运行示例

**主要流程**:
1. 阅读 boot/boot.asm
2. 理解引导扇区结构
3. 学习实模式到保护模式切换
4. 使用 GDB 调试引导过程
5. 修改代码实验

**后置条件**:
- 理解 BIOS 引导过程
- 掌握保护模式切换

### 用例 3：学习进程管理

**参与者**: 想理解进程调度的学生

**前置条件**:
- 已理解引导和内核初始化

**主要流程**:
1. 阅读 process/process.c
2. 理解进程控制块
3. 学习上下文切换
4. 修改调度算法
5. 观察调度行为

**后置条件**:
- 理解进程管理原理
- 能修改调度算法

## 5. 约束条件

### 5.1 技术约束

- 目标平台：x86 (32位)
- 开发环境：Linux
- 依赖工具：GCC, NASM, QEMU, Make
- 无第三方库依赖

### 5.2 学习约束

- 代码量控制在 5000 行以内
- 避免过度工程化
- 保持简单但完整
- 中文注释和文档

### 5.3 时间约束

- 最小可用版本：1 周
- 完整版本：4 周
- 文档完善：持续进行
