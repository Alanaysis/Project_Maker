#### 1. BIOS 启动过程

**什么是 BIOS?**
- BIOS (Basic Input/Output System) 是计算机启动时运行的第一个程序
- 存储在主板的 ROM 芯片中
- 负责硬件初始化和自检

**启动流程**:
1. 电源开启，CPU 重置
2. CPU 从 0xFFFF0 开始执行
3. BIOS 进行 POST (Power-On Self-Test)
4. BIOS 搜索引导设备
5. 加载引导扇区到 0x7C00
6. 跳转到 0x7C00 执行

#### 2. 实模式

**什么是实模式?**
- 8086 处理器的工作模式
- 地址空间 1MB
- 段地址:偏移地址 寻址方式
- 没有内存保护

**寻址方式**:
```
物理地址 = 段地址 * 16 + 偏移地址
例如: 0x1234:0x5678 = 0x12340 + 0x5678 = 0x179B8
```

#### 3. 保护模式

**什么是保护模式?**
- 80386 及以后处理器的工作模式
- 支持 4GB 地址空间
- 支持内存保护
- 支持多任务

**GDT (全局描述符表)**:
- 定义内存段的属性
- 每个段有基地址、限长、权限
- CPU 通过段选择子访问

**切换步骤**:
1. 禁用中断 (cli)
2. 加载 GDT (lgdt)
3. 设置 CR0 的 PE 位
4. 远跳转刷新流水线
5. 设置段寄存器

#### 2. GDT 初始化

**GDT 条目结构**:
```
+------------------+------------------+
| 段限长 (16位)    | 基地址 (16位)    |
+------------------+------------------+
| 基地址 (8位)     | 访问字节         |
+------------------+------------------+
| 粒度字节         | 基地址 (8位)     |
+------------------+------------------+
```

**访问字节**:
- 第 7 位: Present (段是否存在)
- 第 5-6 位: DPL (特权级)
- 第 4 位: Descriptor Type
- 第 0-3 位: Segment Type

#### 2. 分页机制

**什么是分页?**
- 将虚拟地址空间分成固定大小的页
- 通过页表映射到物理地址
- 实现虚拟内存

**地址转换**:
```
虚拟地址: [页目录索引(10位)] [页表索引(10位)] [页内偏移(12位)]

1. 用页目录索引找到页表
2. 用页表索引找到物理页
3. 物理页地址 + 页内偏移 = 物理地址
```

**页表项标志**:
- Present: 页是否在内存中
- Read/Write: 读写权限
- User/Supervisor: 用户/内核权限
- Accessed: 是否被访问
- Dirty: 是否被写入

#### 1. 系统调用概念

**什么是系统调用?**
- 用户程序请求内核服务的接口
- 从用户态切换到内核态
- 执行特权操作
- 返回结果给用户程序

**为什么需要系统调用?**
- 安全性: 限制用户程序的权限
- 抽象性: 提供统一的接口
- 隔离性: 用户程序不能直接访问硬件

**我的理解**:

#### 2. 系统调用实现

**实现步骤**:
1. 用户程序调用库函数
2. 库函数触发软中断 (int 0x80)
3. CPU 切换到内核态
4. 中断处理程序分发系统调用
5. 执行对应的内核函数
6. 返回结果给用户程序

**参数传递**:
- 通过寄存器传递
- EAX: 系统调用号
- EBX, ECX, EDX: 参数
