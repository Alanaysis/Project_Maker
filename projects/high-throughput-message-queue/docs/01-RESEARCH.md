# 市场调研报告

## 1. 问题定义

### 要解决的问题
在分布式系统中，服务间需要异步、可靠、高性能的消息传递机制。

### 为什么这个问题重要
消息队列是微服务架构的核心组件，用于解耦、削峰、异步处理。

## 2. 同类型项目概览

| 项目 | 核心特点 | 技术栈 |
|------|---------|--------|
| Apache Kafka | 分布式、持久化、高吞吐 | Java/Scala |
| RabbitMQ | 支持 AMQP、灵活路由 | Erlang |
| Redis Streams | 内存、轻量 | C |
| NATS | 高性能、简单 | Go |

## 3. 技术变体分析

**基础版本**：
```
消息生产 → 持久化 → 消息分发 → 消费确认
```

**变体1：日志存储**
- 类似 Kafka 的追加日志
- 顺序写入，零拷贝读取

**变体2：内存队列**
- 全部在内存中
- 极致延迟，重启丢失

## 4. 我们的选择

实现基于日志存储的消息队列：
- 顺序文件追加写入
- 分段存储（Segments）
- 消费偏移跟踪
- 崩溃恢复
