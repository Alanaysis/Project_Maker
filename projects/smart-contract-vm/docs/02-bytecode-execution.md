# 字节码执行

## 字节码基础

智能合约被编译成字节码，由一系列操作码（Opcode）组成。每个操作码是一个字节，代表一个特定的操作。

## 操作码分类

### 1. 停止操作
- `STOP (0x00)`: 停止执行

### 2. 算术操作
- `ADD (0x01)`: 加法
- `MUL (0x02)`: 乘法
- `SUB (0x03)`: 减法
- `DIV (0x04)`: 除法
- `MOD (0x05)`: 取模

### 3. 比较操作
- `LT (0x10)`: 小于比较
- `GT (0x11)`: 大于比较
- `EQ (0x12)`: 等于比较
- `ISZERO (0x13)`: 是否为零

### 4. 位运算
- `AND (0x16)`: 按位与
- `OR (0x17)`: 按位或
- `XOR (0x18)`: 按位异或
- `NOT (0x19)`: 按位取反

### 5. 栈操作
- `POP (0x50)`: 弹出栈顶
- `DUP1-DUP4 (0x80-0x83)`: 复制栈元素
- `SWAP1-SWAP4 (0x90-0x93)`: 交换栈元素

### 6. 内存操作
- `MLOAD (0x51)`: 从内存加载
- `MSTORE (0x52)`: 存储到内存

### 7. 存储操作
- `SLOAD (0x54)`: 从存储加载
- `SSTORE (0x55)`: 存储到存储

### 8. 跳转操作
- `JUMP (0x56)`: 无条件跳转
- `JUMPI (0x57)`: 条件跳转
- `JUMPDEST (0x5b)`: 跳转目标标记

### 9. Push 操作
- `PUSH1-PUSH32 (0x60-0x7f)`: 将数据压入栈

## 示例：简单加法

```bytecode
PUSH1 0x03  // 将 3 压入栈
PUSH1 0x02  // 将 2 压入栈
ADD         // 弹出 2 和 3，将 5 压入栈
STOP        // 停止执行
```

执行过程：
1. PC=0: 读取 `0x60` (PUSH1)，压入下一个字节 `0x03`
2. PC=2: 读取 `0x60` (PUSH1)，压入下一个字节 `0x02`
3. PC=4: 读取 `0x01` (ADD)，弹出 2 和 3，压入 5
4. PC=5: 读取 `0x00` (STOP)，停止执行
