# EVM 概述

## 什么是 EVM？

EVM（Ethereum Virtual Machine，以太坊虚拟机）是以太坊区块链的核心组件，它是一个图灵完备的虚拟机，负责执行智能合约字节码。

## 核心概念

### 1. 执行环境

EVM 运行在一个沙盒环境中，具有以下特点：
- **隔离性**: 合约执行不影响外部世界
- **确定性**: 相同输入始终产生相同输出
- **可终止性**: 通过 Gas 机制防止无限循环

### 2. 状态模型

EVM 维护以下状态：
- **栈 (Stack)**: 1024 个 256 位元素的后进先出栈
- **内存 (Memory)**: 线性字节数组，按字节寻址
- **存储 (Storage)**: 键值对存储，持久化在区块链上

### 3. Gas 机制

Gas 是 EVM 中的执行成本单位：
- 每个操作码消耗特定数量的 Gas
- Gas 限制防止恶意代码消耗过多资源
- 未使用的 Gas 会退还给调用者

## 与传统虚拟机的区别

| 特性 | EVM | 传统 VM (如 JVM) |
|------|-----|------------------|
| 数据宽度 | 256 位 | 32/64 位 |
| 内存模型 | 线性内存 | 堆栈分离 |
| 存储 | 持久化键值存储 | 文件系统 |
| 终止机制 | Gas 限制 | 无特殊限制 |
| 确定性 | 必须确定 | 可以不确定 |

## 简化实现

我们的实现基于 EVM 的核心概念，但做了以下简化：
- 使用 64 位整数代替 256 位
- 简化 Gas 成本计算
- 不实现完整的系统调用
- 内存大小限制更小
