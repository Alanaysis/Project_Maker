## 核心组件设计

### 1. 操作码 (Opcode)

采用枚举类型定义所有操作码，每个操作码包含：
- 操作码字节值
- Gas 成本
- 额外操作数字节数

```rust
pub enum Opcode {
    Stop = 0x00,
    Add = 0x01,
    // ...
}
```

### 2. 栈 (Stack)

使用 `Vec<u64>` 作为底层存储：
- 最大深度: 1024
- LIFO 顺序
- 支持 push/pop/dup/swap 操作

### 3. 内存 (Memory)

线性字节数组：
- 按字节寻址
- 动态扩展（最大 1MB）
- 支持 32 字节读写

### 4. 存储 (Storage)

持久化键值存储：
- 使用 `HashMap<u64, u64>`
- 写入 0 表示删除
- 支持 load/store 操作

### 5. Gas 计量器 (GasMeter)

跟踪执行成本：
- 记录已使用 Gas
- 检查 Gas 限制
- 按操作码计费

## 执行流程

```
1. 初始化 VM
   ├── 创建栈、内存、存储
   ├── 设置 Gas 限制
   └── 加载合约代码

2. 执行循环
   ├── 取指: 读取当前 PC 位置的操作码
   ├── 解码: 识别操作码类型
   ├── 计费: 消耗对应 Gas
   ├── 执行: 执行操作码逻辑
   └── 更新 PC

3. 终止条件
   ├── STOP 指令
   ├── RETURN 指令
   ├── REVERT 指令
   ├── Gas 耗尽
   └── PC 超出代码范围
```

## 合约功能设计

### 函数选择器

使用 calldata 的前 4 字节作为函数选择器：
```
calldata: [selector(4)] [param1(32)] [param2(32)] ...
```

### 事件日志

LOG0-LOG4 操作码实现：
- 从内存读取数据
- 从栈读取主题（topics）
- 生成日志条目

### 环境信息

新增操作码提供合约执行环境：
- ADDRESS: 合约地址
- CALLER: 调用者地址
- CALLVALUE: 转账金额
- CALLDATALOAD: 读取 calldata
- CALLDATASIZE: calldata 大小
- CALLDATACOPY: 复制 calldata 到内存
