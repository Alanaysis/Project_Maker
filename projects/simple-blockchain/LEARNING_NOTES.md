# LEARNING_NOTES.md - 区块链学习笔记

## 1. 区块链基础概念

### 1.1 什么是区块链
区块链是一种分布式数据库技术，通过密码学方法将数据块按时间顺序连接成链式结构。每个区块包含：
- 区块头（Block Header）
- 交易列表（Transactions）
- 时间戳（Timestamp）
- 前一区块哈希（Previous Hash）
- 随机数（Nonce）

### 1.2 核心特性
- **去中心化**：无需中心化机构，由网络节点共同维护
- **不可篡改**：通过密码学哈希保证数据完整性
- **透明性**：所有交易公开可查
- **共识机制**：节点间达成一致的算法

### 1.3 区块链类型
- **公有链**：完全去中心化，任何人可参与
- **私有链**：中心化控制，仅限特定成员
- **联盟链**：部分去中心化，由联盟成员共同维护

## 2. 密码学基础

### 2.1 哈希算法
哈希算法将任意长度的输入转换为固定长度的输出，具有以下特性：
- **单向性**：无法从哈希值反推原始数据
- **抗碰撞性**：难以找到两个不同输入产生相同哈希
- **雪崩效应**：输入微小变化导致输出巨大变化

常用哈希算法：
- SHA-256：比特币使用
- Keccak-256：以太坊使用
- RIPEMD-160：用于地址生成

### 2.2 非对称加密
非对称加密使用一对密钥：
- **私钥**：用于签名，必须保密
- **公钥**：用于验证，可以公开

常用算法：
- RSA：基于大数分解
- ECDSA：基于椭圆曲线
- Ed25519：基于 Edwards 曲线

### 2.3 数字签名
数字签名用于验证消息的真实性和完整性：
1. 发送方使用私钥对消息签名
2. 接收方使用公钥验证签名
3. 签名证明消息确实来自发送方

### 2.4 默克尔树
默克尔树是一种二叉树结构，用于高效验证大量数据：
- 叶子节点是数据块的哈希
- 非叶子节点是子节点哈希的哈希
- 根节点代表整个数据集的完整性

## 3. 共识机制

### 3.1 工作量证明（PoW）
PoW 要求节点通过计算哈希值来证明工作量：
- **优点**：简单易实现，安全性高
- **缺点**：能耗大，确认时间长
- **应用**：比特币、莱特币

### 3.2 权益证明（PoS）
PoS 根据持有代币数量和时间选择出块者：
- **优点**：能耗低，确认快
- **缺点**：可能导致中心化
- **应用**：以太坊 2.0、Cardano

### 3.3 委托权益证明（DPoS）
DPoS 通过投票选出代表进行出块：
- **优点**：高吞吐量，低能耗
- **缺点**：去中心化程度较低
- **应用**：EOS、TRON

### 3.4 实用拜占庭容错（PBFT）
PBFT 通过多轮投票达成共识：
- **优点**：确认快，能耗低
- **缺点**：节点数量有限
- **应用**：Hyperledger Fabric

## 4. 区块结构

### 4.1 区块头
```go
type BlockHeader struct {
    Version       int32    // 版本号
    PrevBlockHash [32]byte // 前一区块哈希
    MerkleRoot    [32]byte // 默克尔树根
    Timestamp     int64    // 时间戳
    Difficulty    uint32   // 难度目标
    Nonce         uint64   // 随机数
}
```

### 4.2 区块体
```go
type Block struct {
    Header       BlockHeader
    Transactions []*Transaction
    Hash         [32]byte
}
```

### 4.3 区块哈希计算
```go
func (b *Block) CalculateHash() [32]byte {
    data := bytes.Join(
        [][]byte{
            b.Header.PrevBlockHash[:],
            b.Header.MerkleRoot[:],
            IntToHex(b.Header.Timestamp),
            IntToHex(int64(b.Header.Difficulty)),
            IntToHex(int64(b.Header.Nonce)),
        },
        []byte{},
    )
    return sha256.Sum256(data)
}
```

## 5. 交易结构

### 5.1 交易输入
```go
type TxInput struct {
    TxID      [32]byte // 交易 ID
    OutIndex  int      // 输出索引
    Signature []byte   // 签名
    PubKey    []byte   // 公钥
}
```

### 5.2 交易输出
```go
type TxOutput struct {
    Value      float64  // 金额
    PubKeyHash []byte   // 公钥哈希
}
```

### 5.3 UTXO 模型
UTXO（未花费交易输出）模型：
- 每个交易消耗之前的 UTXO
- 创建新的 UTXO
- 余额 = 所有 UTXO 之和

## 6. 钱包实现

### 6.1 密钥对生成
```go
func NewWallet() *Wallet {
    private, public := newKeyPair()
    wallet := Wallet{private, public, ""}
    wallet.Address = wallet.GenerateAddress()
    return &wallet

```

### 6.2 地址生成
地址生成过程：
1. 生成私钥
2. 从私钥推导公钥
3. 对公钥进行 SHA-256 哈希
4. 对结果进行 RIPEMD-160 哈希
5. 添加版本前缀
6. 计算校验和
7. 进行 Base58 编码

## 7. 网络通信

### 7.1 P2P 网络
P2P 网络特点：
- 去中心化
- 节点直接连接
- 无中心服务器

### 7.2 消息类型
- Version：版本协商
- Verack：版本确认
- Tx：交易广播
- Block：区块广播
- GetBlocks：请求区块
- Inv：库存通知

### 7.3 广播机制
- 新交易广播到所有连接节点
- 新区块广播到所有连接节点
- 避免重复广播

## 8. 挖矿过程

### 8.1 挖矿步骤
1. 从交易池获取交易
2. 创建候选区块
3. 计算默克尔树根
4. 设置时间戳和难度
5. 循环尝试不同的 Nonce
6. 直到找到满足难度要求的哈希
7. 广播新区块

### 8.2 难度调整
- 每 2016 个区块调整一次
- 根据出块时间调整难度
- 目标出块时间：10 分钟

## 9. 安全性考虑

### 9.1 51% 攻击
当攻击者拥有超过 50% 的算力时，可以：
- 双花攻击
- 回滚交易
- 拒绝服务

### 9.2 防御措施
- 增加确认数
- 提高挖矿难度
- 分散算力

### 9.3 隐私保护
- 地址混淆
- 零知识证明
- 环签名
