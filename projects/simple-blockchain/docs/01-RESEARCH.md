# 01-RESEARCH.md - 区块链技术调研

## 1. 区块链核心概念

### 1.1 什么是区块链
区块链是一种分布式数据库技术，通过密码学方法将数据块按时间顺序连接成链式结构。每个区块包含：
- 区块头（Block Header）
- 交易列表（Transactions）
- 时间戳（Timestamp）
- 前一区块哈希（Previous Hash）
- 随机数（Nonce）

### 1.2 核心特性
- **去中心化**：无需中心化机构，由网络节点共同维护
- **不可篡改**：通过密码学哈希保证数据完整性
- **透明性**：所有交易公开可查
- **共识机制**：节点间达成一致的算法

## 2. 区块结构设计

### 2.1 区块头结构
```go
type BlockHeader struct {
    Version       int32    // 版本号
    PrevBlockHash [32]byte // 前一区块哈希
    MerkleRoot    [32]byte // 默克尔树根
    Timestamp     int64    // 时间戳
    Difficulty    uint32   // 难度目标
    Nonce         uint64   // 随机数
}
```

### 2.2 区块结构
```go
type Block struct {
    Header       BlockHeader
    Transactions []*Transaction
    Hash         [32]byte
}
```

## 3. 共识算法调研

### 3.1 工作量证明（PoW）
- **原理**：通过计算哈希值满足特定难度要求来证明工作量
- **优点**：简单易实现，安全性高
- **缺点**：能耗大，确认时间长
- **实现**：SHA-256 哈希算法，难度值调整

### 3.2 权益证明（PoS）
- **原理**：根据持有代币数量和时间选择出块者
- **优点**：能耗低，确认快
- **缺点**：可能导致中心化

### 3.3 本项目选择：PoW
- 学习目的，PoW 最直观易懂
- 实现简单，便于理解核心原理

## 4. 密码学基础

### 4.1 哈希算法
- **SHA-256**：比特币使用的哈希算法
- **特性**：单向性、抗碰撞性、雪崩效应

### 4.2 非对称加密
- **椭圆曲线加密（ECC）**：secp256k1 曲线
- **公钥/私钥对**：私钥签名，公钥验证
- **地址生成**：公钥 -> SHA-256 -> RIPEMD-160 -> Base58

### 4.3 默克尔树
- **结构**：二叉树，叶子节点为交易哈希
- **用途**：快速验证交易存在性
- **实现**：递归计算哈希

## 5. P2P 网络

### 5.1 网络拓扑
- **去中心化**：节点直接连接
- **广播机制**：新交易和区块广播到所有节点
- **节点发现**：通过种子节点发现新节点

### 5.2 通信协议
- **TCP 连接**：可靠传输
- **消息类型**：
  - Version：版本协商
  - Verack：版本确认
  - Tx：交易广播
  - Block：区块广播
  - GetBlocks：请求区块
  - Inv：库存通知
