## 核心概念

### 1. 场景图（Scene Graph）

**定义**：场景图是一种树形数据结构，用于组织 3D 场景中的物体及其空间关系。每个节点代表一个物体或变换，父子关系表示空间上的包含关系。

**关键点**：
- 树形结构：根节点 → 子节点 → 孙节点
- 层级变换：子节点继承父节点的变换
- 层级裁剪：父节点被裁剪时，所有子节点也被裁剪

### 2. 变换层级（Transform Hierarchy）

**定义**：子物体的最终位置由其自身变换和所有祖先节点的变换共同决定。

**关键点**：
- 世界变换 = 父世界变换 × 局部变换
- TRS 顺序：先缩放 → 再旋转 → 最后平移
- 脏标记：只在需要时重新计算

**变换矩阵组合**：
```
M_world = M_parent × T × R × S
```

### 3. 视锥裁剪（Frustum Culling）

**定义**：判断物体是否在摄像机的可见范围内，不可见的物体不需要渲染。

**关键点**：
- 视锥体由 6 个平面定义
- AABB 与视锥体的相交测试
- p-vertex 优化：只测试最远的顶点

**裁剪判断**：
```
对于每个平面：
  if (AABB在法向量方向最远的顶点) 在平面背面:
    return false  // 被裁剪
return true  // 可见
```

### 4. 四元数（Quaternion）

**定义**：用四个数 (x, y, z, w) 表示旋转，避免万向锁问题。

- 旋转操作：q * v * q^-1

## 重点难点

### 难点 1：变换矩阵的组合顺序

**问题描述**：为什么是 TRS（缩放-旋转-平移）而不是其他顺序？

**解决过程**：
1. 尝试 SRT：平移后缩放，位置也会被缩放 → 错误
2. 尝试 RTS：缩放后旋转，缩放轴会改变 → 错误
3. TRS：先在模型空间缩放，再旋转，最后平移 → 正确

### 难点 2：视锥体平面提取

**问题描述**：如何从 View-Projection 矩阵中提取 6 个裁剪平面？

**解决过程**：
1. 研究 Gribb/Hartmann 方法
2. 理解矩阵行与平面的关系
3. 归一化平面方程

**关键收获**：

_________________________________

### 难点 3：脏标记传播

**问题描述**：当一个节点变换改变时，如何高效地更新所有受影响的节点？

**解决过程**：
1. 最初方案：每次更新都遍历整棵树 → 性能差
2. 改进方案：只标记脏节点，按需更新
3. 最终方案：标记脏 + 递归更新子节点

## 代码片段收藏

### 片段 1：p-vertex 优化

```cpp
Vec3 p_vertex;
p_vertex.x = (plane.normal.x >= 0) ? aabb.max.x : aabb.min.x;
p_vertex.y = (plane.normal.y >= 0) ? aabb.max.y : aabb.min.y;
p_vertex.z = (plane.normal.z >= 0) ? aabb.max.z : aabb.min.z;
```

**收藏原因**：这是视锥裁剪的核心优化，只测试一个顶点而不是 8 个。
**使用场景**：任何 AABB 与半空间的相交测试。

### 片段 2：四元数旋转

```cpp
Vec3 rotate(const Vec3& v) const {
    Quaternion qv(v.x, v.y, v.z, 0);
    Quaternion result = (*this) * qv * conjugate();
    return {result.x, result.y, result.z};
}
```

**收藏原因**：简洁地实现了四元数旋转向量的公式 q*v*q^-1。
**使用场景**：任何需要旋转向量的场景。

### 片段 3：脏标记传播

```cpp
void mark_dirty() {
    world_transform_dirty_ = true;
    bounds_dirty_ = true;
    for (auto& child : children_) {
        child->mark_dirty();
    }
}
```

**收藏原因**：简单有效的脏标记传播模式。
**使用场景**：任何需要增量更新的层级结构。
