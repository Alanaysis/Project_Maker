# 01 - 响应式框架市场调研

## 1. 响应式编程概述

响应式编程是一种面向数据流和变化传播的编程范式。在前端框架中，响应式系统的核心目标是：
- 当数据变化时，自动更新依赖该数据的视图或计算结果
- 开发者只需声明数据之间的关系，无需手动操作 DOM

## 2. 主流响应式框架对比

### Vue 2.x - Object.defineProperty

```
优点：
  - 兼容性好（IE9+）
  - 实现成熟稳定

缺点：
  - 无法检测属性的添加和删除
  - 无法检测数组索引和长度变化
  - 需要 Vue.set() / Vue.delete() 特殊 API
  - 初始化时需要递归遍历所有属性
```

### Vue 3.x - Proxy

```
优点：
  - 可以检测属性的添加和删除
  - 可以检测数组的所有变化
  - 惰性代理：只在访问嵌套对象时才代理
  - 性能更好（无需初始化时递归）

缺点：
  - 不支持 IE
  - 不能 polyfill
```

### MobX - Object.defineProperty + 自动追踪

```
优点：
  - 自动追踪所有可观测状态
  - 极少的样板代码
  - 支持装饰器语法

缺点：
  - 魔法较多，调试困难
  - 学习曲线较陡
```

### Solid.js - 编译时优化

```
优点：
  - 无虚拟 DOM，直接操作真实 DOM
  - 极致的性能
  - 细粒度更新

缺点：
  - 需要编译器支持
  - 模板语法限制较多
```

## 3. 核心概念分析

### 3.1 数据劫持（Data Hijacking）

数据劫持是响应式系统的基础，通过拦截数据的读写操作来实现：

```
Object.defineProperty (Vue 2)
  - 拦截 getter/setter
  - 需要预先知道属性名

Proxy (Vue 3, 本项目)
  - 拦截所有操作（get/set/delete/has/ownKeys...）
  - 不需要预先知道属性名
  - 支持数组、Map、Set 等
```

### 3.2 依赖收集（Dependency Tracking）

```
核心思想：
  1. 属性被读取时 → 记录"谁在读"（收集依赖）
  2. 属性被修改时 → 通知"读过它的人"（触发更新）

数据结构：
  - Dep（依赖管理器）：每个属性一个，存储订阅者列表
  - Watcher（观察者）：代表一个依赖关系（如一个 computed 或 watch）
  - 两者是多对多关系
```

### 3.3 发布-订阅模式

```
数据（发布者）→ Dep（调度中心）→ Watcher（订阅者）
    ↑                                    ↓
  set()                            callback()
```

## 4. 设计决策

本项目选择使用 Proxy 实现，原因：

1. **现代性**：Proxy 是 ES2015+ 标准，代表未来方向
2. **完整性**：能拦截更多操作，实现更完整的响应式
3. **学习价值**：理解 Proxy 有助于理解 Vue 3 的设计
4. **简洁性**：代码更简洁，逻辑更清晰
