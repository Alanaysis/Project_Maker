# 01 - 研究笔记：目标检测与 YOLO

## 目标检测概述

### 什么是目标检测？

目标检测是计算机视觉中的核心任务，需要同时完成：
1. **定位**：找出图像中物体的位置（边界框）
2. **分类**：识别每个物体的类别

与图像分类不同，目标检测需要处理：
- 可变数量的物体
- 不同大小和比例的物体
- 物体之间的遮挡
- 背景干扰

### 传统方法 vs 深度学习

| 方法 | 代表 | 优点 | 缺点 |
|------|------|------|------|
| 传统方法 | HOG+SVM, DPM | 可解释性强 | 精度低，特征工程复杂 |
| 两阶段检测 | R-CNN 系列 | 精度高 | 速度慢 |
| 单阶段检测 | YOLO, SSD | 速度快 | 精度略低 |

## YOLO 算法

### 核心思想

YOLO (You Only Look Once) 将目标检测视为回归问题：
- 将图像划分为 S×S 网格
- 每个网格单元预测 B 个边界框
- 同时预测类别概率

### YOLO v1 架构

```
输入图像 (448×448×3)
    ↓
卷积层 (特征提取)
    ↓
全连接层 (预测)
    ↓
输出张量 (S×S×(B×5+C))
```

输出格式：
- S×S 网格，每个单元预测 B 个框
- 每个框：(x, y, w, h, confidence)
- C 个类别概率

### 关键创新

1. **全局推理**：一次前向传播完成检测
2. **端到端训练**：直接优化检测性能
3. **实时性能**：45 FPS (基础版)，155 FPS (快速版)

## 锚框机制

### 为什么需要锚框？

YOLO v1 直接预测坐标，存在以下问题：
- 训练不稳定
- 难以处理不同形状的物体

### 锚框概念

锚框是预定义的参考框：
- 不同大小和比例
- 作为预测的起点
- 简化学习目标

### YOLO v2/v3 的改进

引入锚框后：
- 预测相对于锚框的偏移
- 使用 k-means 聚类确定锚框尺寸
- 多尺度预测

## NMS (非极大值抑制)

### 问题背景

检测器通常会为同一物体产生多个重叠的检测框：
- 滑动窗口的特性
- 多个锚框匹配同一物体
- 需要去重

### NMS 算法

```
1. 按置信度排序所有检测框
2. 选择置信度最高的框
3. 删除与该框 IoU > 阈值的其他框
4. 重复直到没有框剩余
```

### IoU (交并比)

```
IoU = 交集面积 / 并集面积
```

- IoU = 1：完全重叠
- IoU = 0：完全不重叠
- 通常阈值设为 0.5
