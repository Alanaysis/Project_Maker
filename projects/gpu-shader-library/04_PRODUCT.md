## 关键要点

### 1. 着色器编程思维

#### 数据并行
```glsl
// 理解 GPU 并行执行模型
// 每个片段独立处理
void main() {
    // 每个像素独立执行这段代码
    vec3 color = calculateLighting();
    FragColor = vec4(color, 1.0);
}
```

#### 向量化思维
```glsl
// 利用 SIMD 特性
// 避免分支
vec3 color = mix(colorA, colorB, step(threshold, value));

// 而不是
vec3 color;
if (value > threshold) {
    color = colorA;
} else {
    color = colorB;
}
```

### 2. 光照物理基础

#### 能量守恒
```
入射光 = 反射光 + 折射光 + 吸收光
反射率 + 透射率 <= 1
```

#### 微表面理论
```
表面由无数微小镜面组成
粗糙度决定微表面朝向分布
```

#### Fresnel 效应
```
观察角度越接近掠射角
反射越强
```

### 3. 性能意识

#### 带宽限制
```glsl
// 纹理采样是昂贵操作
// 减少采样次数
vec4 data = texture(gBuffer, uv);  // 一次采样获取多通道
```

#### 计算复杂度
```glsl
// 避免昂贵运算
float invLength = inversesqrt(dot(v, v));  // 比 1.0/sqrt() 快
```

#### 分支开销
```glsl
// GPU 不擅长分支
// 使用 step/smoothstep 替代 if
float factor = smoothstep(edge0, edge1, x);
```

### 4. 调试技巧

#### 可视化调试
```glsl
// 输出中间结果用于调试
FragColor = vec4(normal * 0.5 + 0.5, 1.0);  // 法线可视化
FragColor = vec4(vec3(depth), 1.0);          // 深度可视化
```

#### 颜色编码
