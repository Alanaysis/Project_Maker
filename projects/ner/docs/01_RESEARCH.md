# 01 - 调研文档

## 序列标注概述

序列标注 (Sequence Labeling) 是 NLP 中的基础任务，目标是为输入序列中的每个 token 分配一个标签。

### 应用场景

| 任务 | 输入 | 输出 | 应用 |
|------|------|------|------|
| NER | 文本 | 实体标签 | 信息抽取 |
| POS Tagging | 文本 | 词性标签 | 句法分析 |
| Chunking | 文本 | 短语标签 | 语义分析 |
| Slot Filling | 语音/文本 | 槽位标签 | 对话系统 |

### 标注方案

#### BIO 标注

```
B-XXX: 实体开始 (Begin)
I-XXX: 实体内部 (Inside)
O:     非实体 (Outside)

优点: 简单直观
缺点: 无法区分相邻同类实体，无法区分单 token 和多 token 实体
```

#### BIOES 标注

```
B-XXX: 多 token 实体的开始 (Begin)
I-XXX: 多 token 实体的内部 (Inside)
O:     非实体 (Outside)
E-XXX: 多 token 实体的结束 (End)
S-XXX: 单 token 实体 (Single)

优点: 信息更丰富，边界更明确
缺点: 标签数量更多
```

#### 方案对比

| 特性 | BIO | BIOES |
|------|-----|-------|
| 标签数量 | 2n+1 | 4n+1 |
| 边界信息 | 部分 | 完整 |
| 单实体标记 | 无 | S- |
| 结束标记 | 无 | E- |
| 通常效果 | 基线 | 更好 |

## NER 方法调研

### 1. 基于规则的方法

#### 正则匹配

适用场景: 格式化实体 (日期、电话、邮箱、网址等)

```
优点:
- 无需训练数据
- 精确度高 (在规则覆盖范围内)
- 可解释性强

缺点:
- 召回率低
- 需要人工编写规则
- 难以处理歧义
```

#### 词典匹配

适用场景: 已知实体集合 (人名库、地名库、机构名库)

```
优点:
- 速度快
- 精确度高 (在词典覆盖范围内)
- 易于维护

缺点:
- 召回率受词典覆盖限制
- 无法识别未登录实体
- 歧义切分问题
```

匹配算法:
- 正向最大匹配 (Forward Maximum Matching)
- 逆向最大匹配 (Backward Maximum Matching)
- 双向匹配 (Bidirectional Matching)

### 2. 统计模型方法

#### HMM (隐马尔可夫模型)

HMM 是生成式概率图模型，建模联合概率 P(X, Y)。

```
三个核心参数:
- 初始概率 pi: P(y_1)
- 转移概率 A: P(y_t | y_{t-1})
- 发射概率 B: P(x_t | y_t)

训练方法: 极大似然估计 (MLE) + Laplace 平滑
解码方法: 维特比算法
```

优缺点:
```
优点:
- 训练速度快
- 实现简单
- 适合小数据集

缺点:
- 生成式模型需要建模 P(X)
- 特征工程受限于生成过程
- 独立性假设过强
```

#### CRF (条件随机场)

CRF 是判别式概率图模型，直接建模条件概率 P(Y|X)。

```
核心公式:
P(Y|X) = exp(Score(X, Y)) / Z(X)
Score(X, Y) = sum(emission_score) + sum(transition_score)

训练方法: 梯度下降优化对数似然
解码方法: 维特比算法
```

与 HMM 的对比:

| 特性 | HMM | CRF |
|------|-----|-----|
| 模型类型 | 生成式 | 判别式 |
| 建模目标 | P(X, Y) | P(Y\|X) |
| 需要建模 P(X) | 是 | 否 |
| 特征工程 | 受限 | 灵活 |
| 训练速度 | 快 | 较慢 |
| 通常效果 | 基线 | 更好 |

### 3. 深度学习方法

#### BiLSTM

使用双向 LSTM 捕获上下文信息，独立预测每个位置的标签。

```
架构: Input -> Embedding -> BiLSTM -> Linear -> Output

优点:
- 自动学习特征
- 捕获长距离依赖
- 处理变长序列

缺点:
- 独立预测，不考虑标签间依赖
- 可能产生非法标签序列
```

#### BiLSTM-CRF

结合 BiLSTM 和 CRF 的优势。

```
架构: Input -> Embedding -> BiLSTM -> Linear -> CRF -> Output

优点:
- BiLSTM 学习高质量上下文表示
- CRF 保证输出序列的合法性
- 两者结合效果更好

缺点:
- 训练较慢 (需要前向算法)
- 实现较复杂
```

历史背景:

| 年份 | 方法 | 贡献 |
|------|------|------|
| 2001 | MEMM | 最大熵马尔可夫模型 |
| 2001 | CRF | 条件随机场 |
| 2015 | LSTM-CRF | Huang et al. |
| 2016 | BiLSTM-CRF | Lample et al. |

## NER 数据集调研

### 常用数据集

| 数据集 | 语言 | 实体类型 | 规模 |

| MSRA NER | 中 | PER, LOC, ORG | 46K 句 |
| People Daily | 中 | PER, LOC, ORG | 20K 句 |

### 评估指标

```
准确率 (Accuracy) = 正确预测的 token 数 / 总 token 数
精确率 (Precision) = TP / (TP + FP)
召回率 (Recall)    = TP / (TP + FN)
F1 分数            = 2 * P * R / (P + R)

TP: 正确识别的实体
FP: 错误识别的实体
FN: 漏掉的实体
```

实体级别的评估: 只有当实体的边界和类型都正确时，才算正确。
