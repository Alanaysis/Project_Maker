# 02 - 需求文档

## 功能需求

### 1. 基于规则的 NER

#### 正则匹配 (RegexNER)

- 支持预定义的正则模式: 日期、电话、邮箱、网址、身份证号、IP 地址、货币金额
- 支持自定义正则模式
- 支持文本级别的实体识别
- 支持 token 序列级别的标注
- 自动去除重叠匹配

#### 词典匹配 (DictNER)

- 支持 Trie 树高效匹配
- 支持正向最大匹配
- 支持逆向最大匹配
- 支持双向匹配
- 支持从文件加载词典
- 支持批量添加实体

### 2. 统计模型 NER

#### HMM

- 支持监督学习训练 (MLE + Laplace 平滑)
- 支持维特比解码
- 支持批量预测
- 提供转移概率矩阵和发射概率矩阵的访问

#### CRF

- 支持特征提取 (当前词、词形、前缀/后缀、上下文)
- 支持梯度下降训练
- 支持维特比解码
- 支持 L2 正则化

### 3. 深度学习 NER

#### BiLSTM

- 支持独立预测每个位置的标签
- 支持变长序列 (pack_padded_sequence)
- 支持 Dropout 正则化

#### BiLSTM-CRF

- 支持 PyTorch CRF 层
- 支持前向算法 (计算配分函数)
- 支持维特比算法 (解码最优路径)
- 支持梯度裁剪
- 支持早停机制
- 支持学习率调度

### 4. 标注方案

#### BIO

- 编码: 实体列表 -> BIO 标签序列
- 解码: BIO 标签序列 -> 实体列表
- 获取标签集合

#### BIOES

- 编码: 实体列表 -> BIOES 标签序列
- 解码: BIOES 标签序列 -> 实体列表
- 获取标签集合
- BIO -> BIOES 转换
- BIOES -> BIO 转换

### 5. 评估

- 准确率 (Accuracy): token 级别
- 精确率 (Precision): 实体级别
- 召回率 (Recall): 实体级别
- F1 分数: 精确率和召回率的调和平均
- 支持按实体类型分别评估
- 支持整体评估 (micro-average)

### 6. 实际应用

- 人名识别 (PER)
- 地名识别 (LOC)
- 机构名识别 (ORG)

## 非功能需求

### 性能需求

- HMM 训练: 支持万级句子
- CRF 训练: 支持千级句子
- BiLSTM-CRF 训练: 支持 GPU 加速
- 推理速度: 单句 < 100ms

### 可扩展性

- 支持自定义实体类型
- 支持自定义特征
- 支持自定义正则模式
- 支持自定义词典

### 易用性

- 统一的 API 接口
- 清晰的错误信息
- 完整的文档和示例
