# 01 - 调研文档

## 1. 项目背景

### 1.1 什么是视频理解?

视频理解 (Video Understanding) 是计算机视觉的一个重要分支，旨在让机器理解视频中的内容、动作和事件。与图像理解不同，视频理解需要同时处理空间信息和时间信息。

### 1.2 应用领域

- **视频摘要**：自动生成视频的关键帧和文字摘要
- **视频分类**：识别视频的内容类别（如体育、新闻、电影）
- **动作识别**：识别视频中的人物动作
- **视频检索**：根据内容搜索视频
- **异常检测**：检测视频中的异常行为
- **自动驾驶**：理解道路场景和交通状况

### 1.3 历史发展

- **2014年**：双流网络 (Two-Stream Networks) 出现，分别处理空间和时间信息
- **2015年**：C3D 提出 3D 卷积，直接建模时空特征
- **2016年**：TSN (Temporal Segment Networks) 提出稀疏采样策略
- **2018年**：SlowFast 网络结合快速和慢速通道
- **2021年**：Vision Transformer 应用于视频理解 (ViViT, TimeSformer)
- **2023年**：大语言模型与视频理解结合 (Video-LLaMA, VideoChat)

## 2. 核心原理

### 2.1 视频表示方法

**帧级表示**：将视频视为独立的帧序列
```
视频 = [frame_1, frame_2, ..., frame_T]
```

**片段级表示**：将视频分割为短片段
```
视频 = [clip_1, clip_2, ..., clip_N]
```

**特征级表示**：将视频编码为特征向量
```
视频 → CNN → [f_1, f_2, ..., f_T] → 池化 → f_video
```

### 2.2 时序建模方法

**2D CNN + 时序池化**：
- 优点：简单高效，易于训练
- 缺点：忽略时序关系
- 代表：TSN

**3D CNN**：
- 优点：直接建模时空关系
- 缺点：计算量大，参数多
- 代表：C3D, I3D, SlowFast

**Transformer**：
- 优点：强大的建模能力，可处理长距离依赖
- 缺点：需要大量数据和计算资源
- 代表：ViViT, TimeSformer

### 2.3 帧采样策略

| 策略 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 均匀采样 | 等间距选取帧 | 简单，覆盖整个视频 | 可能错过关键时刻 |
| 随机采样 | 随机选取帧 | 增加多样性 | 不确定性高 |
| 密集采样 | 高频率采样 | 保留更多细节 | 计算量大 |
| 关键帧采样 | 基于场景变化选取 | 保留重要信息 | 需要额外计算 |

### 2.4 特征提取方法

**空间特征**：使用 CNN 提取每帧的视觉特征
- ResNet-18/34/50
- VGG-16/19
- EfficientNet

**时间特征**：建模帧间关系
- 光流 (Optical Flow)
- 帧差分 (Frame Difference)
- 时序卷积 (Temporal Convolution)

**时空特征**：同时建模空间和时间
- 3D 卷积
- 时序 Transformer
- 图神经网络

## 3. 关键帧提取

### 3.1 基于视觉差异

**颜色直方图差异**：
1. 计算每帧的 HSV 颜色直方图
2. 计算相邻帧的直方图距离
3. 选取差异较大的帧

**常用距离度量**：
- Bhattacharyya 距离
- 卡方距离
- 相关性距离

### 3.2 基于运动信息

**光流方法**：
1. 计算相邻帧的光流场
2. 计算光流幅度
3. 选取运动变化大的帧

**帧差分方法**：
1. 计算相邻帧的像素差
2. 计算差分的统计量
3. 选取变化显著的帧

### 3.3 基于聚类

**K-Means 聚类**：
1. 提取每帧的特征
2. 对特征进行聚类
3. 选取每个聚类的中心帧

**优势**：
- 自动确定关键帧数量
- 保证关键帧的多样性
- 适用于各种视频类型

## 4. 视频摘要

### 4.1 抽取式摘要

从视频中选取关键帧，组成摘要。

**方法**：
1. 重要性评分：为每帧计算重要性分数
2. 关键帧选取：选取分数最高的帧
3. 去重：避免选取过于相似的帧

### 4.2 生成式摘要

生成描述视频内容的文字摘要。

**方法**：
1. 视频编码：将视频编码为特征向量
2. 文本解码：使用语言模型生成文字
3. 注意力机制：关注视频的关键部分

### 4.3 评价指标

**关键帧质量**：
- 准确率 (Precision)
- 召回率 (Recall)
- F1 分数

**摘要质量**：
- ROUGE 分数
- BLEU 分数
- 人工评价

## 6. 本项目的技术选型

### 6.1 选型理由

**特征提取**：ResNet-18/34/50
- 成熟稳定，预训练权重丰富
- 计算效率高，易于部署
- 支持多种变体

**时序池化**：Mean/Max/Attention
- Mean：简单高效，作为基线
- Max：捕捉最显著特征
- Attention：学习帧重要性

**关键帧提取**：直方图 + 光流 + 聚类
- 直方图：计算简单，效果稳定
- 光流：捕捉运动信息
- 聚类：保证多样性

### 6.2 设计目标

1. **模块化**：各组件独立，易于替换
2. **可扩展**：支持添加新的采样策略、骨干网络等
3. **易用性**：提供统一的分析接口
4. **可测试**：所有组件都有单元测试
