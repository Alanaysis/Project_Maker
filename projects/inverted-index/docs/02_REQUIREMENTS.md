# 02 - 需求文档

## 1. 项目目标

实现一个完整的倒排索引搜索引擎，支持：
- 文档索引构建
- 多种查询方式
- 相关性排序
- 索引压缩

## 2. 功能需求

### 2.1 文档处理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 分词 | 支持中英文分词 | P0 |
| 停用词过滤 | 移除常见无意义词汇 | P0 |
| 词干提取 | 英文词汇还原为词干 | P1 |
| 词形还原 | 词汇还原为词典形式 | P1 |

### 2.2 索引构建

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 倒排索引 | 基本的词汇-文档映射 | P0 |
| 位置索引 | 记录词汇在文档中的位置 | P1 |
| 压缩索引 | 可变字节编码压缩 | P2 |
| 增删改 | 支持动态更新索引 | P1 |

### 2.3 查询处理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 单词查询 | 查找包含特定词汇的文档 | P0 |
| AND查询 | 查找包含所有词汇的文档 | P0 |
| OR查询 | 查找包含任一词汇的文档 | P0 |
| NOT查询 | 排除包含特定词汇的文档 | P1 |
| 短语查询 | 查找包含特定短语的文档 | P1 |
| 通配符查询 | 支持 * 和 ? 通配符 | P2 |
| 模糊查询 | 容错查询（编辑距离） | P2 |

### 2.4 排序

| 功能 | 描述 | 优先级 |
|------|------|--------|
| TF-IDF | 基于词频和逆文档频率排序 | P0 |
| BM25 | 改进的排序算法 | P1 |

## 3. 非功能需求

### 3.1 性能要求

- 索引构建：1000文档 < 5秒
- 查询响应：单次查询 < 100ms
- 内存占用：支持10万文档级别

### 3.2 可扩展性

- 支持自定义分词器
- 支持自定义停用词
- 支持自定义排序算法

### 3.3 易用性

- 简洁的API接口
- 完整的文档和示例
- 良好的错误提示

## 4. 技术约束

- 语言：Python 3.8+
- 依赖：仅使用标准库 + nltk（可选）
- 测试覆盖率：> 80%

## 5. 验收标准

1. 能够索引和检索文档
2. 支持所有定义的查询类型
3. 排序算法正确实现
4. 所有测试通过
5. 文档完整
