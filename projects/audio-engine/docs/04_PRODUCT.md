## 2. 关键要点

### 2.1 音频采样的核心概念

**采样率选择**
```
应用场景          采样率      频率范围
─────────────────────────────────────
电话语音          8kHz       0-4kHz
宽带语音          16kHz      0-8kHz
AM广播           22.05kHz   0-11kHz
CD质量           44.1kHz    0-22kHz
专业音频          48kHz      0-24kHz
高清音频          96kHz      0-48kHz
```

**位深度与动态范围**
```
位深度    动态范围    应用场景
─────────────────────────────
8-bit    48dB       低质量语音
16-bit   96dB       CD质量
24-bit   144dB      专业录音
32-bit   192dB      内部处理
```

### 2.2 FFT 的重要性

**FFT 是音频处理的核心工具**

应用领域：
1. **频谱分析** - 可视化音频频率成分
2. **滤波** - 频域滤波比时域更高效
3. **卷积** - 实现混响等效果
4. **音高检测** - 识别基频
5. **压缩** - MP3/AAC 使用频域变换

**FFT 复杂度对比**
```
方法        时间复杂度      1024点耗时（估算）
────────────────────────────────────────────
DFT         O(N²)          ~1ms
FFT         O(N log N)     ~0.01ms
```

### 2.3 动态处理的原理

**压缩器工作流程**
```
输入信号 → 电平检测 → 增益计算 → 增益平滑 → 增益应用 → 输出信号
```

**关键参数**
- **Threshold（阈值）**: 开始压缩的电平
- **Ratio（比率）**: 压缩比例（如 4:1）
- **Attack（启动）**: 压缩响应时间
- **Release（释放）**: 压缩恢复时间
- **Knee（拐点）**: 压缩曲线形状

### 2.4 混响的实现

**Schroeder 混响架构**
```
                    ┌─── Comb Filter 1 ───┐
                    │                     │
输入 ──────────────┼─── Comb Filter 2 ───┼───── Allpass 1 ──── Allpass 2 ──── 输出
                    │                     │
                    ├─── Comb Filter 3 ───┤
                    │                     │
                    └─── Comb Filter 4 ───┘
```

**参数含义**
- **Room Size（房间大小）**: 控制延迟时间
- **Damping（阻尼）**: 控制高频衰减
- **Mix（干湿比）**: 原始信号与混响信号的比例

### 2.5 音频合成方法

**波形合成基础**
```
正弦波: sin(2πft)
方波:   sign(sin(2πft))
三角波: 2|2(ft - floor(ft + 0.5))| - 1
锯齿波: 2(ft - floor(ft + 0.5))
```

**FM 合成公式**
```
y(t) = A * sin(2πfc*t + I * sin(2πfm*t))
```
- fc: 载波频率
- fm: 调制频率
- I: 调制指数
- A: 振幅

## 4. 技术选型思考

### 4.1 为什么选择 C++？

| 考虑因素 | C++ | Python | Rust |
|----------|-----|--------|------|
| 性能 | 优秀 | 一般 | 优秀 |
| 学习曲线 | 中等 | 简单 | 陡峭 |
| 生态系统 | 成熟 | 丰富 | 发展中 |
| 实时性 | 支持 | 不支持 | 支持 |
| 教学价值 | 高 | 中 | 中 |

**选择 C++ 的原因**
1. 音频处理需要高性能
2. 理解底层实现细节
3. 广泛用于专业音频软件
4. 教学价值高

### 4.2 为什么自己实现 FFT？

**学习价值**
- 理解算法原理
- 掌握蝶形运算
- 了解位反转
- 性能优化实践

**实际应用**
- 专业场景使用 FFTW
- 嵌入式可能需要自定义实现
- 理解才能正确使用库

### 4.3 WAV 格式的选择

**为什么先实现 WAV？**
- 格式简单，易于理解
- 无压缩，直接对应 PCM 数据
- 广泛支持
- 便于调试

**后续扩展**
- MP3: 需要理解心理声学
- AAC: 更复杂的编码
- Opus: 最先进的编解码器

## 5. 常见陷阱与解决方案

### 5.1 采样率相关

**问题**: 采样率不匹配导致音高错误
**解决**: 实现采样率转换

**问题**: 混叠（Aliasing）
**解决**: 采样前进行抗混叠滤波

### 5.2 量化相关

**问题**: 量化噪声
**解决**: 使用抖动（Dither）技术

**问题**: 削波（Clipping）
**解决**: 使用限幅器或降低音量

### 5.3 FFT 相关

**问题**: 频谱泄漏
**解决**: 使用窗函数

**问题**: 频率分辨率不足
**解决**: 增加 FFT 大小或零填充

### 5.4 实时处理

**问题**: 延迟过大
**解决**: 减小缓冲区大小

**问题**: 处理不及时导致断音
**解决**: 优化算法或增加缓冲
