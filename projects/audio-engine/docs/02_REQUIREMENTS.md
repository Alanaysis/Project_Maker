# 需求分析

## 1. 功能需求

### 1.1 音频基础模块

#### 采样 (Sampling)
- **FR-1.1.1**: 生成正弦波、方波、三角波、锯齿波测试信号
- **FR-1.1.2**: 演示不同采样率（8kHz, 44.1kHz, 48kHz, 96kHz）的效果
- **FR-1.1.3**: 展示混叠现象（Aliasing）及其预防

#### 量化 (Quantization)
- **FR-1.2.1**: 实现不同位深度（8-bit, 16-bit, 24-bit）量化
- **FR-1.2.2**: 计算量化噪声和信噪比（SNR）
- **FR-1.2.3**: 演示抖动（Dither）技术

#### 采样率转换 (Sample Rate Conversion)
- **FR-1.3.1**: 实现整数倍上采样（插值）
- **FR-1.3.2**: 实现整数倍下采样（抽取）
- **FR-1.3.3**: 实现非整数倍采样率转换

#### 位深度处理 (Bit Depth)
- **FR-1.4.1**: 位深度转换（16→24, 24→16）
- **FR-1.4.2**: 抖动处理
- **FR-1.4.3**: 噪声整形（Noise Shaping）

#### 声道处理 (Channels)
- **FR-1.5.1**: 立体声转单声道
- **FR-1.5.2**: 单声道转立体声
- **FR-1.5.3**: 声道平衡调节

## 2. 技术清单

### 2.1 数据结构

```cpp
// 音频缓冲区
struct AudioBuffer {
    std::vector<float> data;     // 采样数据
    int sample_rate;             // 采样率
    int channels;                // 声道数
    int bit_depth;               // 位深度
};

// WAV文件头
struct WavHeader {
    char riff[4];                // "RIFF"
    uint32_t file_size;          // 文件大小
    char wave[4];                // "WAVE"
    char fmt[4];                 // "fmt "
    uint32_t fmt_size;           // 格式块大小
    uint16_t audio_format;       // 音频格式
    uint16_t num_channels;       // 声道数
    uint32_t sample_rate;        // 采样率
    uint32_t byte_rate;          // 字节率
    uint16_t block_align;        // 块对齐
    uint16_t bits_per_sample;    // 位深度
    char data[4];                // "data"
    uint32_t data_size;          // 数据大小
};
```

### 2.2 算法清单

| 模块 | 算法 | 复杂度 |
|------|------|--------|
| FFT | Cooley-Tukey | O(N log N) |
| 卷积 | 重叠相加法 | O(N log N) |
| 滤波器 | IIR/FIR | O(N) |
| 混响 | Schroeder/Freeverb | O(N) |
| 压缩 | 动态增益 | O(N) |
| 音高 | 自相关/YIN | O(N²)或O(N) |
| 节拍 | 能量检测 | O(N) |

### 2.3 依赖库

| 库 | 用途 | 必需 |
|----|------|------|
| 标准库 | 数学、容器、IO | 是 |
| ALSA/CoreAudio | 音频输出 | 否（演示用） |
| FFTW | FFT加速 | 否（自己实现） |

### 2.4 性能指标

| 指标 | 目标 |
|------|------|
| FFT延迟 | < 1ms (4096 samples) |
| 处理延迟 | < 10ms |
| CPU使用 | < 50% (单核) |
| 内存使用 | < 100MB |
