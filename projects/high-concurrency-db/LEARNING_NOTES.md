## 第三阶段：B+ 树索引

### 学习目标
- [ ] 理解 B+ 树的结构和性质
- [ ] 实现查找、插入、删除操作

### 核心概念

#### 1. B+ 树 vs B 树
```
B 树：
- 数据存储在所有节点
- 搜索可能在非叶子节点结束

B+ 树：
- 数据只存储在叶子节点
- 叶子节点通过链表连接
- 所有搜索都到达叶子节点
```

你的理解：

#### 2. 节点结构
```
内部节点：
┌─────────────────────────────────────┐
│  n keys: [k0, k1, k2, ..., kn]    │
│  n+1 children: [p0, p1, ..., pn]   │
└─────────────────────────────────────┘

叶子节点：
┌─────────────────────────────────────┐
│  n keys: [k0, k1, k2, ..., kn]    │
│  n values: [v0, v1, ..., vn]       │
│  next pointer                       │
└─────────────────────────────────────┘
```

#### 3. 插入操作
```
步骤：
1. 找到合适的叶子节点
2. 如果有空间，直接插入
3. 如果没有空间，分裂：
   a. 创建新节点
   b. 将一半 key 移到新节点
   c. 将中间 key 提升到父节点
   d. 如果父节点也满，递归分裂
```

**图解**:
```
插入前：        插入后：
    [5]            [5, 8]
   / \            / | \
 [1,3] [7,9]   [1,3] [5,7] [8,9]
```

#### 4. 删除操作
```
步骤：
1. 找到并删除 key
2. 如果节点 key 数量 >= 最小值，完成
3. 如果不足，尝试：
   a. 从兄弟节点借一个 key
   b. 如果兄弟也最少，合并节点
```

### 核心概念

#### 1. 词法分析（Tokenization）
```
SQL: SELECT name FROM users WHERE id = 1

Tokens:
[SELECT] [name] [FROM] [users] [WHERE] [id] [=] [1]
```

**实现要点**:
```
- 使用有限状态机
- 处理空白字符
- 识别关键字 vs 标识符
```

#### 2. 语法分析（Parsing）
```
递归下降解析：

parseSelect()
    ├── match(SELECT)
    ├── parseColumnList()
    ├── match(FROM)
    ├── parseTableName()
    ├── [match(WHERE) → parseWhereClause()]
    └── return SelectStatement
```

#### 3. AST（抽象语法树）
```
SELECT name FROM users WHERE id = 1

        SelectStatement
        ├── columns: [name]
        ├── table: users
        └── where: EqualExpression
                    ├── left: ColumnRef(id)
                    └── right: Literal(1)
```

## 第六阶段：并发控制

### 学习目标
- [ ] 理解事务的 ACID 属性
- [ ] 实现读写锁
- [ ] 理解锁协议和死锁

### 核心概念

#### 1. 事务 ACID
```
A - Atomicity（原子性）：事务要么全部成功，要么全部失败
C - Consistency（一致性）：事务前后数据库保持一致状态
I - Isolation（隔离性）：并发事务互不干扰
D - Durability（持久性）：已提交事务的数据永久保存
```

#### 2. 读写锁
```
共享锁（S Lock）：多个事务可以同时持有
排他锁（X Lock）：只有一个事务可以持有

兼容矩阵：
    S   X
S   ✓   ✗
X   ✗   ✗
```

#### 3. 两阶段锁（2PL）
```
阶段1 - Growing：获取锁
阶段2 - Shrinking：释放锁

规则：
- 在释放任何锁之后，不能再获取新锁
```

#### 4. 死锁
```
示例：
T1 持有 A 的锁，等待 B 的锁
T2 持有 B 的锁，等待 A 的锁
→ 死锁！

解决方案：
- 死锁检测（Wait-For Graph）
- 死锁预防（Wait-Die, Wound-Wait）
```
