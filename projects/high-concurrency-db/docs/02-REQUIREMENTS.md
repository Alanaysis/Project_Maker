# 需求分析文档

## 1. 项目定位

MiniDB 是一个**教学导向**的轻量级数据库引擎，目标是帮助开发者理解数据库内核的核心原理。

## 2. 用户画像

### 主要用户
- **数据库内核初学者**: 想要理解数据库底层实现的学习者
- **计算机科学学生**: 正在学习数据库系统课程
- **后端开发者**: 想要深入了解存储引擎原理
- **数据库爱好者**: 对数据库内部实现好奇

### 用户需求
| 用户类型 | 核心需求 | 次要需求 |
|----------|----------|----------|
| 初学者 | 代码易读，文档清晰 | 可视化工具 |
| 学生 | 涵盖课程知识点 | 练习题 |
| 后端开发者 | 架构清晰，可扩展 | 性能对比 |
| 爱好者 | 完整功能演示 | 源码注释 |

## 3. 功能需求

### 3.1 核心功能（P0 - 必须实现）

#### SQL 解析
- [ ] 词法分析器（Tokenizer）
- [ ] 语法分析器（Parser）
- [ ] AST（抽象语法树）生成

#### 支持的 SQL 语句
```sql
-- DDL
CREATE TABLE table_name (
    col1 INT PRIMARY KEY,
    col2 VARCHAR(100),
    col3 FLOAT
);

-- DML
INSERT INTO table_name VALUES (1, 'hello', 3.14);
SELECT col1, col2 FROM table_name WHERE col1 > 0;
UPDATE table_name SET col2 = 'world' WHERE col1 = 1;
DELETE FROM table_name WHERE col1 = 1;
```

#### 存储引擎
- [ ] 磁盘管理器（Disk Manager）
- [ ] 缓冲池管理器（Buffer Pool Manager）
- [ ] B+ 树索引

#### 并发控制
- [ ] 读写锁
- [ ] 简单事务支持

### 3.2 重要功能（P1 - 应该实现）

- [ ] 查询优化（谓词下推）
- [ ] 多表查询（简单 JOIN）
- [ ] 聚合函数（COUNT, SUM, AVG）
- [ ] 排序（ORDER BY）

### 3.3 扩展功能（P2 - 可以实现）

- [ ] 索引自动创建
- [ ] 查询计划可视化
- [ ] 性能基准测试
- [ ] WAL 日志

## 4. 非功能需求

### 4.1 性能需求
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 单行查询延迟 | < 1ms | 内存命中 |
| 批量插入 | > 10K rows/s | 顺序写入 |
| 并发读 | > 100 QPS | 读写锁 |

### 4.2 可靠性需求
- 数据持久化到磁盘
- 崩溃恢复（基础）
- 无数据损坏

### 4.3 可维护性需求
- 代码注释覆盖率 > 30%
- 模块间低耦合
- 清晰的错误处理

## 5. 数据类型支持

| 类型 | 存储大小 | 范围 | 说明 |
|------|----------|------|------|
| INT | 4 bytes | -2^31 ~ 2^31-1 | 32位整数 |
| FLOAT | 4 bytes | IEEE 754 | 单精度浮点 |
| VARCHAR(n) | 2 + len bytes | 0 ~ n | 变长字符串 |

## 6. 约束条件

### 6.1 技术约束
- 使用 C++17 标准
- 不依赖第三方库（除测试框架）
- 支持 Linux 和 macOS

### 6.2 设计约束
- 单机数据库
- 单文件存储
- 行级锁（简化为表级锁）

## 7. 验收标准

### 7.1 功能验收
```bash
# 创建表
> CREATE TABLE users (id INT PRIMARY KEY, name VARCHAR(50), age INT);
OK

# 插入数据
> INSERT INTO users VALUES (1, 'Alice', 25);
OK

# 查询数据
> SELECT name, age FROM users WHERE id = 1;
name    | age
--------+-----
Alice   | 25

# 更新数据
> UPDATE users SET age = 26 WHERE id = 1;
OK

# 删除数据
> DELETE FROM users WHERE id = 1;
OK
```

### 7.2 并发验收
- 多个读操作可以并发执行
- 写操作互斥
- 无死锁

### 7.3 测试验收
- 单元测试覆盖率 > 80%
- 所有测试通过
- 无内存泄漏

## 9. 风险分析

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| B+ 树实现复杂 | 高 | 高 | 参考现有实现，分步实现 |
| 并发 bug | 高 | 中 | 充分测试，使用 sanitizer |
| SQL 解析困难 | 中 | 中 | 使用递归下降解析器 |
| 性能不达标 | 低 | 低 | 先功能后性能 |
