# 产品思维文档

## 1. 产品定位

### 1.1 一句话描述
MiniDB 是一个面向学习者的轻量级数据库内核实现，帮助开发者深入理解数据库核心原理。

### 1.2 核心价值主张

| 维度 | 价值 |
|------|------|
| 学习价值 | 通过实际代码理解数据库内核 |
| 实践价值 | 动手实现核心组件 |
| 面试价值 | 掌握数据库底层原理 |
| 职业价值 | 为数据库开发打下基础 |

## 2. 目标用户

### 2.1 用户画像

**用户 A: 数据库初学者**
- 背景：计算机科学大三学生
- 目标：理解课程中的数据库概念
- 痛点：理论和实践脱节，不知道如何实现
- 场景：学习 B+ 树、事务、查询优化

**用户 B: 后端开发者**
- 背景：3 年 Java/Go 后端开发经验
- 目标：深入理解存储引擎原理
- 痛点：使用数据库但不理解内部实现
- 场景：优化查询性能，理解索引原理

**用户 C: 数据库爱好者**
- 背景：对数据库内核感兴趣
- 目标：参与开源数据库项目
- 痛点：大型项目代码量太大，难以入手
- 场景：为贡献 TiDB/CockroachDB 做准备

### 2.2 用户旅程

```
发现 → 了解 → 尝试 → 学习 → 深入 → 分享
  │      │      │      │      │      │
  ▼      ▼      ▼      ▼      ▼      ▼
GitHub  README 克隆   跟踪  修改  写博客
搜索    文档   运行   代码  扩展  贡献
```

## 3. 竞品分析

### 3.1 竞品对比

| 项目 | 目标用户 | 代码量 | 学习曲线 | 文档 | 活跃度 |
|------|----------|--------|----------|------|--------|
| MiniDB | 初学者 | 适中 | ⭐⭐ | 丰富 | - |
| BusTub | CMU学生 | 适中 | ⭐⭐⭐ | 英文 | 高 |
| MiniOB | 入门者 | 较大 | ⭐⭐⭐ | 中文 | 高 |
| SQLite | 开发者 | 大 | ⭐⭐⭐⭐ | 丰富 | 高 |
| LevelDB | 高级用户 | 中等 | ⭐⭐⭐ | 一般 | 中 |

### 3.2 竞争优势

**MiniDB 的差异化优势**:

1. **中文优先**
   - 所有文档和注释使用中文
   - 降低国内学习者的语言门槛

2. **学习导向**
   - 代码结构清晰，便于理解
   - 重点难点有详细标注
   - 包含学习笔记模板

3. **完整链路**
   - 从 SQL 到存储的完整实现
   - 不是单一组件，而是完整数据库

4. **渐进式学习**
   - 可以从简单模块开始
   - 逐步深入复杂模块

### 3.3 竞争劣势

| 劣势 | 影响 | 缓解措施 |
|------|------|----------|
| 性能不如工业级 | 低 | 教学项目不需要极致性能 |
| 功能不如完整数据库 | 中 | 明确定位为教学项目 |
| 社区小 | 中 | 通过优质内容吸引用户 |

## 4. 用户吸引力

### 4.1 吸引策略

**内容驱动**:
- 高质量的技术博客
- 详细的代码注释
- 清晰的架构图

**社区驱动**:
- GitHub Discussions
- 学习交流群
- 定期分享会

**成果驱动**:
- 学习成果展示
- 面试经验分享
- 贡献者认可

### 4.2 用户留存

**学习路径设计**:
```
Level 1: 环境搭建 + Hello World
    ↓
Level 2: 磁盘管理 + 页面管理
    ↓
Level 3: 缓冲池 + LRU 替换
    ↓
Level 4: B+ 树索引
    ↓
Level 5: SQL 解析
    ↓
Level 6: 查询执行
    ↓
Level 7: 并发控制
    ↓
Level 8: 查询优化
```

**学习激励**:
- 完成每个 Level 有徽章
- 贡献代码有贡献者证书
- 优秀学习笔记有展示

## 9. 总结

MiniDB 的核心价值在于**降低数据库内核学习门槛**。通过：
1. 清晰的中文文档
2. 渐进式学习路径
3. 完整的功能实现
4. 活跃的社区支持

帮助更多开发者理解数据库核心原理，为职业发展打下坚实基础。
