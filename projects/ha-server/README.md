# 高可用高及时性服务器框架 (HA-Server)

## 项目简介

一个从零实现的高可用、低延迟服务器框架，支持故障转移和负载均衡。通过本项目，深入理解高可用架构设计的核心原理。

## 学习目标

- **高可用架构设计**：理解主从、多活、集群等高可用模式
- **故障检测和转移**：掌握心跳检测、超时机制、自动故障转移策略
- **负载均衡算法**：实现轮询、加权轮询、最少连接等经典算法
- **连接池管理**：理解连接复用、连接池大小调优
- **网络编程**：掌握 socket 编程、非阻塞 I/O、事件驱动模型

## 技术栈

| 技术 | 选择 | 学习难度 | 说明 |
|------|------|----------|------|
| 主语言 | C++ | ⭐⭐⭐⭐ | 性能关键场景首选 |
| 网络模型 | epoll (Linux) | ⭐⭐⭐⭐⭐ | 高性能 I/O 多路复用 |
| 线程模型 | 线程池 | ⭐⭐⭐ | 并发请求处理 |
| 协议 | HTTP/1.1 | ⭐⭐⭐ | 应用层协议实现 |

> 💡 **为什么选择 C++？**
> - 性能关键：高可用服务器对延迟极其敏感
> - 底层控制：需要直接操作 socket、内存等系统资源
> - 学习价值：理解高性能服务器的核心实现原理
> - 行业参考：Nginx、HAProxy、Envoy 均使用 C/C++

## 核心循环

```
请求接入 → 健康检查 → 负载均衡 → 请求转发 → 响应返回 → 故障转移
```

## 重点难点

### ⭐ 重点
1. **事件驱动模型**：理解 epoll 的工作原理和事件循环
2. **负载均衡算法**：每种算法的适用场景和实现细节
3. **健康检查机制**：主动检查 vs 被动检查的权衡
4. **连接池管理**：连接复用、超时回收、池大小调优

### ⭐ 难点
1. **故障检测的准确性**：如何避免误判（网络抖动 vs 真实故障）
2. **优雅降级**：部分后端故障时如何保证服务可用
3. **并发安全**：多线程环境下的数据一致性
4. **性能优化**：在保证正确性的前提下追求极致性能

## 💡 值得思考

1. **为什么 Nginx 选择单进程多线程而不是多进程？**（提示：共享内存、锁竞争）
2. **加权轮询中权重如何动态调整？**（提示：根据响应时间、CPU 负载）
3. **健康检查间隔设多少合适？**（提示：频率 vs 开销的权衡）
4. **连接池大小如何确定？**（提示：Little's Law、排队论）
5. **如何实现零宕机时间的配置热更新？**（提示：优雅重启、信号处理）
