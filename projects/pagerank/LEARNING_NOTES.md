# PageRank 算法学习笔记

## 1. PageRank 核心概念

### 1.1 什么是 PageRank

PageRank 是 Google 创始人 Larry Page 和 Sergey Brin 在 1998 年提出的算法，用于衡量网页的重要性。

**核心思想**：
- 一个页面被越多页面链接，它就越重要
- 被重要页面链接比被普通页面链接更有价值
- 链接数量越少的页面，传递的价值越高

### 1.2 数学公式

```
PR(i) = (1-d)/N + d * Σ(PR(j)/L(j))
```

**参数说明**：
- `PR(i)`：页面 i 的 PageRank 值
- `d`：阻尼因子（Damping Factor），通常取 0.85
- `N`：网页总数
- `L(j)`：页面 j 的出链数量
- `Σ(PR(j)/L(j))`：所有指向页面 i 的页面的 PageRank 贡献之和

### 1.3 阻尼因子

**作用**：模拟用户随机点击链接的概率

- `d = 0.85`：85% 概率点击链接，15% 概率随机跳转
- `d = 1.0`：完全跟随链接（可能出现死循环）
- `d = 0.0`：完全随机（所有页面同等重要）

## 2. 算法实现

### 2.1 迭代法

**步骤**：
1. 初始化所有页面的 PR 值为 1/N
2. 对每个页面，根据公式计算新的 PR 值
3. 重复步骤 2 直到收敛（变化小于阈值）

**收敛条件**：
- 达到最大迭代次数
- L1 范数变化小于阈值（如 1e-6）

### 2.2 稀疏矩阵优化

**为什么使用稀疏矩阵**：
- 网页图通常非常稀疏（每个页面只链接到少量页面）
- 稀疏矩阵节省内存
- 稀疏矩阵运算更高效

**SciPy 稀疏矩阵类型**：
- CSR（Compressed Sparse Row）：按行压缩
- CSC（Compressed Sparse Column）：按列压缩
- COO（Coordinate）：坐标格式

### 2.3 转移矩阵

**构建步骤**：
1. 构建邻接矩阵 A（A[i,j] = 1 如果页面 i 链接到页面 j）
2. 归一化列（使每列和为 1）
3. 处理悬挂节点（没有出链的页面）

**悬挂节点处理**：
- 将其视为链接到所有页面
- 或者将其 PageRank 均匀分配给所有页面

## 3. 关键实现细节

### 3.1 WebGraph 类

**功能**：
- 管理页面和链接
- 构建邻接矩阵
- 构建转移矩阵

**设计决策**：
- 使用字典存储页面名称到索引的映射
- 使用列表存储边
- 使用稀疏矩阵构建图结构

### 3.2 PageRank 类

**功能**：
- 迭代法计算 PageRank
- 幂迭代法计算 PageRank
- 代数法计算 PageRank

**参数**：
- `damping_factor`：阻尼因子
- `max_iterations`：最大迭代次数
- `tolerance`：收敛阈值

### 3.3 PageRankResult 类

**属性**：
- `scores`：PageRank 分数数组
- `iterations`：迭代次数
- `converged`：是否收敛
- `final_diff`：最终差异
- `page_names`：页面名称映射

**方法**：
- `ranked_pages`：按分数排序的页面列表
- `get_score`：获取特定页面的分数

## 5. 性能优化

### 5.1 稀疏矩阵

- 使用 CSR 格式存储邻接矩阵
- 使用稀疏矩阵乘法
- 避免稠密矩阵运算

### 5.2 收敛加速

- 使用良好的初始值
- 调整收敛阈值
- 使用更高效的求解方法

### 5.3 内存优化

- 使用稀疏矩阵
- 避免存储不必要的中间结果
- 使用原地操作

## 7. 常见问题

### 7.1 为什么不收敛？

- 阻尼因子过大（接近 1）
- 图结构特殊（如孤立节点）
- 最大迭代次数不足

### 7.2 如何选择阻尼因子？

- 经验值：0.85
- 范围：0.5 - 0.95
- 根据具体应用调整

### 7.3 如何处理大规模图？

- 使用稀疏矩阵
- 分布式计算
- 近似算法
