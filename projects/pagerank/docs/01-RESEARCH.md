# PageRank 算法研究文档

## 1. 算法背景

### 1.1 历史背景

PageRank 由 Google 创始人 Larry Page 和 Sergey Brin 于 1998 年在斯坦福大学开发，是 Google 搜索引擎的核心算法之一。

**关键里程碑**：
- 1996 年：Larry Page 和 Sergey Brin 开始研究网页排名
- 1998 年：发表论文 "The Anatomy of a Large-Scale Hypertextual Web Search Engine"
- 1998 年 9 月：Google 公司成立
- 2001 年：PageRank 获得美国专利

### 1.2 算法动机

**问题**：如何从海量网页中找到最相关的结果？

**传统方法的局限**：
- 基于关键词匹配：容易被滥用
- 基于人工分类：无法扩展
- 基于访问量：不代表质量

**PageRank 的创新**：
- 利用网页之间的链接关系
- 将链接视为"投票"
- 递归定义重要性

## 2. 算法原理

### 2.1 基本思想

**核心假设**：
1. 一个页面被越多页面链接，它就越重要
2. 被重要页面链接比被普通页面链接更有价值
3. 链接数量越少的页面，传递的价值越高

**类比**：
- 学术论文引用：被引用次数多的论文更重要
- 社交网络：被更多人关注的人更有影响力
- 网页链接：被更多页面链接的页面更重要

### 2.2 数学定义

**PageRank 公式**：

```
PR(i) = (1-d)/N + d * Σ(PR(j)/L(j))
```

**参数说明**：
- `PR(i)`：页面 i 的 PageRank 值
- `d`：阻尼因子（Damping Factor），通常取 0.85
- `N`：网页总数
- `L(j)`：页面 j 的出链数量
- `Σ(PR(j)/L(j))`：所有指向页面 i 的页面的 PageRank 贡献之和

**矩阵形式**：

```
PR = (1-d)/N * e + d * M * PR
```

其中：
- `PR`：PageRank 向量
- `e`：全 1 向量
- `M`：转移矩阵

### 2.3 阻尼因子

**定义**：用户继续点击链接的概率

**作用**：
- 防止排名被少数页面垄断
- 模拟用户随机跳转行为
- 确保算法收敛

**取值**：
- 经验值：0.85
- 范围：[0, 1]
- 含义：85% 概率点击链接，15% 概率随机跳转

### 2.4 转移矩阵

**定义**：表示页面之间链接关系的矩阵

**构建方法**：
1. 构建邻接矩阵 A（A[i,j] = 1 如果页面 i 链接到页面 j）
2. 归一化列（使每列和为 1）
3. 处理悬挂节点（没有出链的页面）

**悬挂节点处理**：
- 将其视为链接到所有页面
- 或者将其 PageRank 均匀分配给所有页面

## 3. 算法变种

### 3.1 Weighted PageRank

**改进**：考虑链接权重

**公式**：
```
PR(i) = (1-d)/N + d * Σ(w(j,i) * PR(j) / L(j))
```

**应用场景**：
- 考虑链接位置（正文 vs 侧边栏）
- 考虑链接文本
- 考虑链接类型（nofollow）

### 3.2 Topic-Sensitive PageRank

**改进**：考虑主题相关性

**方法**：
1. 为每个主题计算一组 PageRank
2. 根据查询主题选择相应的 PageRank
3. 结合传统 PageRank 和主题 PageRank

**应用场景**：
- 个性化搜索
- 主题相关排名

### 3.3 Personalized PageRank

**改进**：个性化排名

**方法**：
1. 为每个用户或查询定义个性化向量
2. 使用个性化向量替代均匀分布的随机跳转
3. 计算个性化的 PageRank

**应用场景**：
- 推荐系统
- 社交网络分析
- 个性化搜索

### 3.4 TrustRank

**改进**：信任排名

**方法**：
1. 人工标记一组可信页面
2. 从可信页面开始传播信任
3. 距离可信页面越近，信任度越高

**应用场景**：
- 反垃圾网页
- 信任传播

## 4. 算法实现

### 4.1 迭代法

**步骤**：
1. 初始化所有页面的 PR 值为 1/N
2. 对每个页面，根据公式计算新的 PR 值
3. 重复步骤 2 直到收敛（变化小于阈值）

**收敛条件**：
- 达到最大迭代次数
- L1 范数变化小于阈值（如 1e-6）

**时间复杂度**：O(E * k)，其中 E 是边数，k 是迭代次数

**空间复杂度**：O(V)，其中 V 是顶点数

### 4.2 幂迭代法

**步骤**：
1. 构建 Google 矩阵：G = d * M + (1-d)/N * ones
2. 初始化 PageRank 向量
3. 迭代：PR = G * PR
4. 归一化：PR = PR / sum(PR)

**优点**：
- 数学形式简洁
- 易于理解

**缺点**：
- 需要存储稠密矩阵
- 计算量大

### 4.3 代数法

**步骤**：
1. 构建线性方程组：(I - d * M) * PR = (1-d)/N * e
2. 使用稀疏矩阵求解器求解

**优点**：
- 直接求解，不需要迭代
- 精确解

**缺点**：
- 需要求解大型线性方程组
- 内存消耗大

## 5. 性能优化

### 5.1 稀疏矩阵

**为什么使用稀疏矩阵**：
- 网页图通常非常稀疏（每个页面只链接到少量页面）
- 稀疏矩阵节省内存
- 稀疏矩阵运算更高效

**SciPy 稀疏矩阵类型**：
- CSR（Compressed Sparse Row）：按行压缩
- CSC（Compressed Sparse Column）：按列压缩
- COO（Coordinate）：坐标格式

### 5.2 收敛加速

**方法**：
- 使用良好的初始值
- 调整收敛阈值
- 使用更高效的求解方法

### 5.3 分布式计算

**方法**：
- MapReduce 实现
- 图分区
- 异步迭代

## 7. 局限性

### 7.1 计算复杂度

**问题**：
- 大规模图计算量大
- 需要迭代多次才能收敛
- 内存消耗大

**解决方案**：
- 使用稀疏矩阵
- 分布式计算
- 近似算法

### 7.2 悬挂节点

**问题**：
- 没有出链的页面会"吸收" PageRank
- 影响其他页面的排名

**解决方案**：
- 将其视为链接到所有页面
- 或者将其 PageRank 均匀分配给所有页面

### 7.3 作弊问题

**问题**：
- 链接农场：大量虚假链接
- 链接买卖：购买高权重链接
- 链接隐藏：隐藏链接以提高排名

**解决方案**：
- TrustRank：信任排名
- 反作弊算法
- 人工审核
