# 外部排序研究文档

## 1. 概述

外部排序（External Sorting）是指在排序过程中，数据无法全部装入内存，需要借助外部存储（如磁盘）进行排序的算法。它是处理大规模数据的核心技术之一。

## 2. 问题背景

### 2.1 内部排序的局限性

内部排序算法（如快速排序、归并排序）假设所有数据都在内存中。当数据量超过内存容量时，这些算法无法直接应用。

**典型场景**:
- 数据库系统需要排序超过内存容量的表
- 日志分析系统需要按时间排序大量日志文件
- 数据处理管道需要对大规模数据集排序

### 2.2 外部存储特性

外部存储（磁盘）与内存有显著差异:

| 特性 | 内存 | 磁盘 |
|------|------|------|
| 访问方式 | 随机访问 | 顺序/随机访问 |
| 访问延迟 | ~100ns | ~10ms |
| 带宽 | ~50GB/s | ~200MB/s |
| 容量 | 8-64GB | TB级 |

**关键洞察**: 磁盘顺序访问比随机访问快 100-1000 倍。

## 3. 外部排序算法

### 3.1 基本外部归并排序

外部排序的基本思路是"分而治之":

1. **阶段 1: 生成初始归并段**
   - 将数据分块，每块大小不超过内存容量
   - 对每块在内存中排序
   - 将排序后的块写入临时文件（称为归并段）

2. **阶段 2: 多路归并**
   - 将多个归并段合并为一个有序文件
   - 使用最小堆维护每个归并段的当前最小元素
   - 每次输出堆中最小元素，并从对应归并段读取下一个元素

**时间复杂度**: O(N log N)
**I/O 复杂度**: O(N log_{M/B}(N/M))

其中:
- N: 数据总量
- M: 内存容量
- B: 块大小

### 3.2 置换选择排序

置换选择排序（Replacement Selection）是一种优化的初始归并段生成算法。

**算法流程**:

```
1. 从输入读取 M 个元素到最小堆
2. 输出堆顶元素（当前最小值），记为 last_output
3. 从输入读取下一个元素 new_item:
   - 如果 new_item >= last_output:
     将 new_item 加入堆（属于当前归并段）
   - 否则:
     标记 new_item 为下一个归并段
4. 当堆为空时，当前归并段结束
5. 重复步骤 1-4，直到所有数据处理完毕
```

**关键特性**:
- 平均归并段长度为 2M（是内部排序法的两倍）
- 当输入数据部分有序时，归并段长度可能更长
- 最坏情况（数据完全逆序）下，归并段长度为 M

**数学证明**:

假设输入数据是随机排列的，置换选择排序生成的归并段平均长度为 2M。

证明思路:
- 考虑一个"滑雪"模型：从山脚开始，每次向上滑到比当前位置高的点
- 滑行的平均距离等于 2M（M 为堆的大小）

### 3.3 多路归并

多路归并（K-way Merge）是将 K 个有序序列合并为一个有序序列的过程。

**基于最小堆的实现**:

```
1. 初始化: 将每个序列的第一个元素加入最小堆
2. 重复以下步骤直到堆为空:
   a. 弹出堆顶元素（最小元素）
   b. 输出该元素
   c. 从该元素所属的序列读取下一个元素
   d. 如果有下一个元素，将其加入堆
```

**时间复杂度**: O(N log K)
- N: 总元素数
- K: 归并路数

**空间复杂度**: O(K)

### 3.4 多趟归并

当归并路数 K 很大时，堆操作的开销会增加。多趟归并通过分阶段归并来解决这个问题。

**策略**:
- 每趟最多归并 P 个归并段
- 如果总归并段数为 R，则需要 log_P(R) 趟
- 每趟的输出成为下一趟的输入

**优化目标**: 最小化总 I/O 次数

总 I/O 次数 = 2 × N × (1 + ⌈log_P(R)⌉)

其中:
- P: 每趟最大归并路数
- R: 初始归并段数

## 4. 优化技术

### 4.1 缓冲 I/O

使用内存缓冲区减少磁盘 I/O 次数:

**写缓冲区**:
- 数据先写入内存缓冲区
- 当缓冲区满时，一次性写入磁盘
- 减少写操作次数

**读缓冲区**:
- 一次性从磁盘读取多个记录到缓冲区
- 后续从缓冲区读取记录
- 减少读操作次数

**双缓冲区**:
- 使用两个缓冲区交替读写
- 一个缓冲区用于当前处理，另一个用于预读取
- 实现 I/O 与计算的重叠

### 4.2 并行 I/O

现代存储设备支持并行 I/O:

**多线程读取**:
- 多个线程同时读取不同的归并段
- 利用 SSD 的并行特性

**异步 I/O**:
- 使用异步 I/O 操作
- 在等待 I/O 完成时进行计算

### 4.3 最小堆优化

**二叉堆**:
- 标准实现，简单高效
- 每次操作 O(log K)

**配对堆**:
- 更好的缓存局部性
- 摊还 O(1) 的 decrease-key 操作

**斐波那契堆**:
- 理论上更优的复杂度
- 实际中由于常数因子较大，不一定更快

### 4.4 归并段合并策略

**平衡归并**:
- 每次合并相同数量的归并段
- 简单但不一定最优

**非平衡归并**:
- 根据归并段大小调整合并策略
- 可以减少总 I/O 次数

**最佳归并树**:
- 使用 Huffman 树的思想
- 最小化总 I/O 次数

## 5. 性能分析

### 5.1 I/O 复杂度

外部排序的主要开销是 I/O 操作。

**初始归并段生成**:
- 读: N 次
- 写: N 次
- 总计: 2N 次 I/O

**多路归并**:
- 每趟: 读 N 次 + 写 N 次 = 2N 次 I/O
- 趟数: ⌈log_P(R)⌉
- 总计: 2N × ⌈log_P(R)⌉ 次 I/O

**总 I/O 次数**:
2N × (1 + ⌈log_P(R)⌉)

### 5.2 内存使用

**堆内存**: O(K) - K 为归并路数
**缓冲区内存**: O(B × K) - B 为缓冲区大小

**总内存**: O(M) = O(K × B)

### 5.3 优化效果

**置换选择排序**:
- 归并段长度从 M 增加到 2M
- 归并段数从 N/M 减少到 N/(2M)
- 减少约 50% 的归并趟数

**缓冲 I/O**:
- 减少 I/O 次数
- 提高磁盘利用率
- 典型加速比: 10-100 倍

**多路归并**:
- 增加归并路数可以减少趟数
- 但会增加堆操作开销
- 最优归并路数通常在 8-64 之间

## 6. 实际应用

### 6.1 数据库系统

数据库系统使用外部排序进行:
- ORDER BY 操作
- GROUP BY 操作
- JOIN 操作（排序-合并连接）
- 索引构建

**PostgreSQL 外部排序**:
- 使用改进的置换选择排序
- 支持增量排序
- 自动调整内存分配

### 6.2 搜索引擎

搜索引擎使用外部排序进行:
- 倒排索引构建
- 查询结果排序
- PageRank 计算

### 6.3 数据处理框架

**MapReduce**:
- Map 阶段生成排序的中间结果
- Reduce 阶段进行多路归并

**Spark**:
- 使用外部排序处理 shuffle 数据
- 支持自定义排序函数

### 6.4 日志处理

日志处理系统使用外部排序进行:
- 按时间戳排序日志
- 合并多个日志文件
- 日志分析和查询

## 7. 算法变种

### 7.1 自然归并排序

利用输入数据中已有的有序段:
- 查找自然有序段
- 对自然有序段进行归并
- 减少初始归并段生成的开销

### 7.2 多相归并排序

使用不同大小的归并段:
- 初始归并段大小不同
- 通过精心安排归并顺序
- 最小化总 I/O 次数

### 7.3 置换选择排序变种

**带松弛的置换选择排序**:
- 允许归并段之间有一定重叠
- 进一步增加归并段长度

**多堆置换选择排序**:
- 使用多个堆
- 每个堆对应一个归并段
- 可以并行生成多个归并段
