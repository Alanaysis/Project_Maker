# 外部排序需求文档

## 1. 项目概述

### 1.1 项目目标

开发一个高效、可靠的外部排序库，用于处理无法全部装入内存的大规模数据排序。

### 1.2 目标用户

- 数据工程师
- 数据库开发人员
- 科学计算研究人员
- 系统开发人员

### 1.3 使用场景

- 大文件排序
- 日志文件处理
- 数据库排序操作
- ETL 数据处理
- 科学计算数据排序

## 2. 功能需求

### 2.1 核心功能

#### FR-001: 基本外部排序

**描述**: 支持对超过内存容量的文件进行排序

**输入**:
- 输入文件路径
- 输出文件路径
- 内存限制（可选）

**输出**:
- 排序后的文件
- 排序统计信息

**验收标准**:
- 能够排序超过内存容量的文件
- 排序结果正确
- 内存使用不超过指定限制

#### FR-002: 初始归并段生成

**描述**: 支持两种初始归并段生成策略

**功能点**:
- 内部排序法: 将数据分块，每块在内存中排序
- 置换选择排序: 使用最小堆生成更长的归并段

**验收标准**:
- 内部排序法生成的归并段长度约为 M（内存容量）
- 置换选择排序生成的归并段长度约为 2M
- 生成的归并段都是有序的

#### FR-003: 多路归并

**描述**: 支持将多个有序归并段合并为一个有序文件

**功能点**:
- 基于最小堆的多路归并
- 支持任意数量的归并段
- 支持多趟归并

**验收标准**:
- 归并结果正确
- 支持至少 1000 路归并
- 多趟归并结果正确

#### FR-004: 缓冲 I/O

**描述**: 使用内存缓冲区减少磁盘 I/O 次数

**功能点**:
- 读缓冲区
- 写缓冲区
- 自动刷新机制

**验收标准**:
- 减少 I/O 次数
- 缓冲区大小可配置
- 数据完整性保证

#### FR-005: 自定义排序

**描述**: 支持自定义排序规则

**功能点**:
- 自定义键提取函数
- 自定义比较函数
- 多键排序

**验收标准**:
- 支持任意可比较类型
- 支持复合排序键
- 支持降序排序

### 2.2 应用功能

#### FR-006: 大文件排序

**描述**: 专门针对超大文件的排序优化

**功能点**:
- 自动内存管理
- 进度报告
- 断点续排（可选）

**验收标准**:
- 能够处理 GB 级文件
- 内存使用稳定
- 提供进度信息

#### FR-007: 日志排序

**描述**: 按时间戳或字段排序日志文件

**功能点**:
- 时间戳解析
- 字段提取
- 自定义分隔符

**验收标准**:
- 正确解析常见时间格式
- 支持多种日志格式
- 排序结果正确

#### FR-008: 数据库排序

**描述**: 模拟数据库的外部排序操作

**功能点**:
- CSV 文件排序
- 多列排序
- 表头处理

**验收标准**:
- 支持标准 CSV 格式
- 多列排序正确
- 保留表头信息

### 2.3 辅助功能

#### FR-009: 统计信息

**描述**: 提供排序过程的统计信息

**输出信息**:
- 总记录数
- 归并段数
- 归并趟数
- I/O 次数
- 时间统计

**验收标准**:
- 统计信息准确
- 包含所有关键指标

#### FR-010: 错误处理

**描述**: 处理各种错误情况

**错误类型**:
- 文件不存在
- 磁盘空间不足
- 内存不足
- 数据格式错误

**验收标准**:
- 提供清晰的错误信息
- 不会导致数据损坏
- 优雅降级

## 3. 非功能需求

### 3.1 性能需求

#### NFR-001: 排序速度

**目标**: 在标准硬件上达到合理的排序速度

**基准**:
- 1GB 数据排序时间 < 5 分钟（机械硬盘）
- 1GB 数据排序时间 < 1 分钟（SSD）

**测试条件**:
- 内存: 1GB
- 磁盘: 7200RPM HDD 或 SSD
- 数据: 随机整数

#### NFR-002: 内存使用

**目标**: 内存使用不超过指定限制

**要求**:
- 内存使用稳定
- 不随数据量增长
- 支持最小 1MB 内存限制

#### NFR-003: I/O 效率

**目标**: 最小化磁盘 I/O 次数

**要求**:
- 使用缓冲 I/O
- 顺序读写为主
- I/O 次数接近理论最优

### 3.2 可靠性需求

#### NFR-004: 数据完整性

**目标**: 排序结果 100% 正确

**要求**:
- 不丢失数据
- 不重复数据
- 排序顺序正确

#### NFR-005: 异常处理

**目标**: 优雅处理各种异常

**要求**:
- 不会导致程序崩溃
- 提供有意义的错误信息
- 清理临时文件

### 3.3 可用性需求

#### NFR-006: 易用性

**目标**: API 简单易用

**要求**:
- 最少的参数配置
- 合理的默认值
- 清晰的文档

#### NFR-007: 可配置性

**目标**: 支持灵活的配置

**配置项**:
- 内存限制
- 缓冲区大小
- 归并路数
- 临时目录

### 3.4 可维护性需求

#### NFR-008: 代码质量

**目标**: 高质量的代码

**要求**:
- 模块化设计
- 清晰的接口
- 完整的注释

#### NFR-009: 测试覆盖

**目标**: 高测试覆盖率

**要求**:
- 单元测试覆盖率 > 80%
- 集成测试
- 性能测试

## 4. 技术约束

### 4.1 平台约束

- 操作系统: Linux, macOS, Windows
- Python 版本: 3.8+
- 无第三方依赖（仅使用标准库）

### 4.2 文件约束

- 支持 UTF-8 编码
- 支持大文件（> 4GB）
- 支持临时文件管理

### 4.3 数据约束

- 支持基本数据类型（整数、浮点数、字符串）
- 支持自定义数据类型
- 单条记录大小 < 内存限制

## 6. 数据需求

### 6.1 输入数据格式

**数值文件**:
```
123
456
789
```

**CSV 文件**:
```
name,age,city
alice,30,beijing
bob,25,shanghai
```

**日志文件**:
```
2024-01-01 10:00:00 INFO Message
2024-01-01 10:01:00 ERROR Error occurred
```

### 6.2 输出数据格式

与输入格式相同，但数据已排序。

### 6.3 临时文件格式

内部使用的临时文件:
- 归并段文件: run_0000.txt, run_0001.txt, ...
- 中间结果文件: pass_000_batch_000.txt, ...
