# 产品思维文档

## 1. 产品定位

**一句话定位**: 面向学习者的、从零实现的大模型后训练框架

**核心价值**: 通过可运行的代码，帮助开发者深入理解 LoRA 微调和 PPO 强化学习的原理

## 2. 目标用户吸引力

### 2.1 为什么用户会选择本项目？

| 用户需求 | 本项目的解决方案 | 对比竞品的优势 |
|----------|-----------------|---------------|
| 想理解 LoRA 底层原理 | 从零实现 LoRA 层，代码有详细注释 | PEFT 封装太深，难以理解 |
| 想学习 PPO 算法 | PPO 核心算法的清晰实现 | TRL 实现复杂，包含大量工程细节 |
| 想快速上手实验 | 提供可运行的小模型示例 | 论文代码通常不完整 |
| 想系统学习 | 完整的文档体系和学习笔记模板 | 散落的博客缺乏系统性 |

### 2.2 核心吸引力

1. **可读性优先**
   - 每个核心函数都有数学公式说明
   - 变量命名清晰，注释使用中文
   - 代码结构与论文描述对应

2. **可运行**
   - 使用 GPT-2 等小模型，单卡即可运行
   - 提供示例数据，无需额外准备
   - 一键运行脚本

3. **可实验**
   - 所有超参数通过配置文件可调
   - 支持快速对比不同配置的结果
   - 日志记录详细，便于分析

## 3. 竞品对比

### 3.1 与 PEFT 的对比

| 维度 | PEFT | 本项目 |
|------|------|--------|
| **定位** | 生产级 PEFT 库 | 学习框架 |
| **LoRA 实现** | 高度优化，支持多种变体 | 清晰易懂，便于理解 |
| **代码复杂度** | 高（抽象层次多） | 低（扁平结构） |
| **适用场景** | 生产环境 | 学习和实验 |
| **文档** | API 文档 | 学习导向文档 |

### 3.2 与 TRL 的对比

| 维度 | TRL | 本项目 |
|------|-----|--------|
| **定位** | 完整的 RLHF 工具库 | 学习框架 |
| **PPO 实现** | 工程完善，支持多种优化 | 核心逻辑清晰 |
| **算法覆盖** | PPO/DPO/KTO/ORPO... | PPO（专注） |
| **代码量** | 大 | 小 |
| **上手难度** | 中-高 | 低 |

### 3.3 与 LLaMA-Factory 的对比

| 维度 | LLaMA-Factory | 本项目 |
|------|---------------|--------|
| **定位** | 统一微调框架 | 学习框架 |
| **用户体验** | Web UI，无代码 | 代码级，可定制 |
| **模型支持** | 100+ 架构 | 基础架构 |
| **学习价值** | 低（抽象层次高） | 高（底层实现） |
| **社区规模** | 35K+ stars | 学习项目 |

## 4. 差异化策略

### 4.1 核心差异化

```
本项目 = 清晰的底层实现 + 完整的学习文档 + 可运行的示例
```

不同于现有框架追求生产级性能和功能覆盖，本项目聚焦于：
- **理解优先**: 每个设计决策都有解释
- **学习路径**: 从理论到实践的完整路径
- **实验友好**: 鼓励修改代码进行实验

### 4.2 学习体验设计

```
理论学习          代码阅读          动手实验          总结反思
   │                │                │                │
   ▼                ▼                ▼                ▼
阅读文档  ──→  阅读核心代码  ──→  运行示例  ──→  填写学习笔记
   │                │                │                │
   ▼                ▼                ▼                ▼
理解原理      理解实现细节     观察实际效果     形成知识体系
```

## 5. 使用场景

### 场景 1: 个人学习
- AI 工程师想理解 LoRA 原理
- 阅读 LoRA 层的实现代码
- 修改 rank 参数进行实验
- 记录学习笔记

### 场景 2: 团队培训
- 技术团队进行内部培训
- 使用项目作为教材
- 团队成员各自完成实验
- 讨论和分享学习心得

### 场景 3: 研究参考
- 算法研究者需要 PPO 的参考实现
- 基于本项目进行算法改进实验
- 对比不同超参数配置的效果

## 6. 成功指标

### 6.1 学习效果指标
- 用户能独立解释 LoRA 的数学原理
- 用户能修改 PPO 的关键参数并预测其影响
- 用户能基于本项目实现新的微调方法

### 6.2 项目质量指标
- 所有核心模块有单元测试
- 所有示例可以一键运行
- 文档覆盖率达到 100%
