# 需求分析文档

## 1. 项目概述

本项目是一个面向学习者的大模型后训练框架，核心目标是让开发者通过实际代码理解 LoRA 微调和 PPO 强化学习的原理与实现。

## 2. 用户画像

### 主要用户

#### 用户 A：AI 工程师（初级-中级）
- **背景**: 有 Python 和 PyTorch 基础，了解 Transformer 架构
- **目标**: 理解 LoRA 微调的底层原理，能够在实际项目中应用
- **痛点**: 现有框架（PEFT、TRL）封装太深，难以理解底层实现
- **期望**: 清晰的代码、详细的注释、可运行的示例

#### 用户 B：算法研究者
- **背景**: 深度学习背景，熟悉 NLP/CV 领域
- **目标**: 理解 PPO 在 LLM 对齐中的应用，为研究做准备
- **痛点**: 论文描述的算法与实际实现之间存在差距
- **期望**: 可修改的代码框架，便于实验不同配置

#### 用户 C：全栈开发者（转 AI 方向）
- **背景**: 软件工程背景，正在学习 AI/ML
- **目标**: 了解大模型训练的完整流程
- **痛点**: 不知道从哪里开始，缺乏系统性的学习路径
- **期望**: 完整的学习路径，从理论到实践

## 3. 功能需求

### 3.1 LoRA 微调模块

#### 核心功能
- [x] 实现 LoRA 层（低秩矩阵 A 和 B）
- [x] 将 LoRA 层注入到 Transformer 注意力层
- [x] LoRA 参数的前向传播和反向传播
- [x] LoRA 权重的保存和加载
- [x] LoRA 权重合并到基础模型

#### 配置项
- `rank` (r): 低秩矩阵的秩，典型值 4/8/16/32/64
- `alpha` (α): 缩放因子，控制 LoRA 更新的幅度
- `dropout`: LoRA 层的 dropout 率
- `target_modules`: 要注入 LoRA 的模块名称列表

#### 验收标准
- LoRA 参数量远小于全参数量（< 1%）
- 微调后的模型在目标任务上有明显提升
- 合并后的模型推理结果与微调模型一致

### 3.2 PPO 训练模块

#### 核心功能
- [x] 策略模型（Policy Model）生成回答
- [x] 奖励模型（Reward Model）评分
- [x] 参考模型（Reference Model）计算 KL 散度
- [x] Value Head 计算状态价值
- [x] PPO 裁剪目标函数
- [x] GAE 优势估计
- [x] 训练循环管理

#### 配置项
- `learning_rate`: 学习率
- `clip_range`: PPO 裁剪范围（典型值 0.2）
- `kl_penalty`: KL 散度惩罚系数
- `gamma`: 折扣因子
- `lam`: GAE 的 lambda 参数
- `batch_size`: 批大小
- `mini_batch_size`: 小批大小
- `ppo_epochs`: 每批数据的 PPO 更新轮数

#### 验收标准
- PPO 训练能够收敛（奖励信号上升）
- KL 散度控制在合理范围内
- 生成质量有可观察的提升

### 3.3 数据处理模块

- [x] 支持 JSON/JSONL 格式数据加载
- [x] 支持指令微调数据格式
- [x] 数据 Tokenization 和 Padding
- [x] 数据集划分（训练/验证）

### 3.4 评估模块

- [x] 训练损失监控
- [x] 学习率调度
- [x] 生成质量评估（BLEU/ROUGE 或自定义指标）
- [x] KL 散度监控
- [x] 奖励分数统计

### 3.5 分布式训练支持

- [x] 数据并行训练
- [x] 多 GPU 支持
- [x] 梯度累积
- [x] 混合精度训练

## 4. 非功能需求

### 4.1 性能需求
- 单卡可训练 7B 参数模型（LoRA 模式）
- 多卡训练时线性加速比 > 80%（2-4卡）

### 4.2 可用性需求
- 所有核心模块有详细的中文注释
- 提供完整的使用示例
- 配置文件使用 YAML 格式，易于修改

### 4.3 可维护性需求
- 模块化设计，各组件松耦合
- 有完整的单元测试覆盖
- 代码风格统一（遵循 PEP 8）

## 5. 用户故事

### 故事 1：LoRA 微调学习
> 作为一名 AI 工程师，我想通过修改 LoRA 的 rank 参数来观察其对模型性能的影响，以便理解低秩分解的权衡。

**验收条件**:
1. 可以通过配置文件修改 rank
2. 训练过程中显示 LoRA 参数量和占比
3. 不同 rank 的训练结果可以对比

### 故事 2：PPO 算法理解
> 作为一名算法研究者，我想观察 PPO 训练过程中 KL 散度和奖励的变化，以便理解 PPO 的收敛特性。

**验收条件**:
1. 训练日志中记录 KL 散度、奖励、loss 等指标
2. 可以调整 clip_range 观察训练稳定性变化
3. 提供可视化工具或日志分析方法

### 故事 3：端到端流程体验
> 作为一名转行开发者，我想运行一个完整的从 SFT 到 RLHF 的流程，以便理解大模型训练的全貌。

**验收条件**:
1. 提供完整的一键运行示例
2. 使用小模型（如 GPT-2）确保可本地运行
3. 每个步骤有清晰的输出和说明
