# 需求分析文档

## 1. 项目概述

### 1.1 项目目标

开发一个功能完整、易于使用的约束满足问题 (CSP) 求解器，支持:
- 多种约束类型
- 高效的约束传播算法
- 智能的搜索策略
- 实际应用场景

### 1.2 目标用户

- 研究人员: CSP 算法研究和实验
- 开发者: 需要求解约束问题的应用
- 学生: 学习 CSP 算法和 AI 概念

## 2. 功能需求

### 2.1 核心功能

#### FR-01: 变量管理
- 支持创建变量，指定名称和取值域
- 支持动态修改变量域
- 支持变量赋值和撤销

#### FR-02: 约束定义
- 支持 AllDifferent 约束
- 支持线性约束 (a*x + b*y op c)
- 支持表约束 (枚举允许的元组)
- 支持自定义约束

#### FR-03: 约束传播
- 实现 AC-3 弧相容算法
- 实现路径相容算法
- 支持传播过程的可视化/日志

#### FR-04: 搜索策略
- 回溯搜索
- 最小剩余值 (MRV) 启发式
- 度启发式 (Degree Heuristic)
- 最小约束值 (LCV) 启发式

#### FR-05: 求解功能
- 求解单个解
- 求解所有解
- 支持初始赋值
- 返回求解统计信息

### 2.2 应用功能

#### FR-06: 数独求解
- 输入 9x9 盘面
- 输出完整解
- 验证解的有效性

#### FR-07: N 皇后问题
- 支持任意 N
- 求解单个/所有解
- 可视化棋盘

#### FR-08: 排课问题
- 支持教师约束
- 支持学生组约束
- 支持时间槽约束

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 数独求解 | < 100ms |
| 8 皇后 | < 1s |
| 16 皇后 | < 10s |

### 3.2 可用性需求

- 清晰的 API 设计
- 完整的文档和示例
- 有意义的错误信息

### 3.3 可维护性需求

- 模块化设计
- 完整的测试覆盖
- 代码注释和文档

### 3.4 可扩展性需求

- 易于添加新的约束类型
- 易于添加新的搜索策略
- 易于添加新的应用场景
