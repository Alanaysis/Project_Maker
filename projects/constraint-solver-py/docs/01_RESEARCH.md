# 约束满足问题研究背景

## 1. 问题定义

约束满足问题 (Constraint Satisfaction Problem, CSP) 是一类重要的组合优化问题，由三个要素组成:

1. **变量集合** X = {X₁, X₂, ..., Xₙ}
2. **域集合** D = {D₁, D₂, ..., Dₙ}，每个变量 Xᵢ 有对应的取值域 Dᵢ
3. **约束集合** C = {C₁, C₂, ..., Cₘ}，限制变量的取值组合

**目标**: 为每个变量分配一个值，使得所有约束同时满足。

## 2. 理论基础

### 2.1 相容性

相容性是 CSP 求解的核心概念，用于在搜索前缩减搜索空间。

| 相容性级别 | 定义 | 复杂度 |
|-----------|------|--------|
| 节点相容 | 每个变量的域满足一元约束 | O(nd) |
| 弧相容 | 对于每个变量的每个值，邻居域中存在支持 | O(ed³) |
| 路径相容 | 任意两变量的值对可通过第三变量扩展 | O(n²d³) |
| k-相容 | 任意 k-1 个变量的相容赋值可扩展到第 k 个 | O(nᵏdᵏ) |

### 2.2 弧相容算法

**AC-3 算法** (Mackworth, 1977):
- 维护一个弧队列
- 迭代地从队列中取出弧 (Xi, Xj)
- 修订 Xi 的域，移除没有 Xj 支持的值
- 如果 Xi 的域被修改，将涉及 Xi 的其他弧加入队列

**复杂度**: O(ed³)，其中 e 是约束数，d 是最大域大小。

### 2.3 路径相容算法

**PC-2 算法**:
- 检查所有三元组 (Xi, Xj, Xk)
- 确保 R_ij 中的每个值对都能通过 Xk 扩展
- 比弧相容更强，但复杂度更高

## 3. 搜索策略

### 3.1 回溯搜索

基本的深度优先搜索 + 回溯:
1. 选择一个未赋值变量
2. 尝试其域中的每个值
3. 检查与已赋值变量的一致性
4. 如果一致，递归求解剩余变量
5. 如果所有值都失败，回溯

### 3.2 变量选择启发式

**最小剩余值 (MRV)**:
- 优先选择域最小的变量
- 直觉: 域越小，越可能失败，尽早失败可以剪枝

**度启发式 (Degree Heuristic)**:
- MRV 平局时，选择约束最多的变量
- 直觉: 约束越多，越可能影响其他变量

### 3.3 值选择启发式

**最小约束值 (LCV)**:
- 优先尝试对邻居约束最小的值
- 直觉: 保留更多选择给后续变量

## 4. 约束类型

### 4.1 AllDifferent 约束

所有变量取不同值。常见于:
- 数独: 每行/列/宫的数字不同
- N 皇后: 所有皇后在不同列

### 4.2 线性约束

形如 a₁x₁ + a₂x₂ + ... + aₙxₙ op c，其中 op 是比较操作符。

应用:
- 资源分配: 总量约束
- 排程: 时间约束

### 4.3 表约束

通过枚举允许的值元组定义约束。

优点:
- 表达力强，可表示任意约束
- 易于理解和维护

缺点:
- 存储空间大
- 传播效率低

## 5. 实际应用

### 5.1 数独

- 81 个变量 (格子)
- 域: 1-9
- 约束: 27 个 AllDifferent (9 行 + 9 列 + 9 宫)

### 5.2 N 皇后

- N 个变量 (每行一个皇后)
- 域: 1 到 N (列位置)
- 约束: AllDifferent + 对角线约束

### 5.3 排课问题

- 变量: 每门课程的时间槽
- 约束: 教师冲突、教室冲突、学生冲突
