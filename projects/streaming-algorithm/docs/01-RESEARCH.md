# 市场调研报告

## 1. 问题定义

### 要解决的问题
在数据实时产生的场景下，无法存储所有历史数据，需要设计只能遍历一次数据（One-pass）的流式算法来处理无限数据流。

### 为什么这个问题重要
实时监控、推荐系统、网络流量分析、物联网传感器数据处理等场景都需要流式算法。

## 2. 同类型项目概览

| 项目 | 核心特点 | 技术栈 |
|------|---------|--------|
| Apache Flink | 分布式流处理框架 | Java |
| Kafka Streams | Kafka 生态流处理 | Java |
| StreamLib | 轻量级流算法库 | Go |
| AlgoDS | 算法数据结构合集 | Go |

## 3. 技术变体分析

### 核心循环的变体

**基础版本**：
```
数据流入 → 窗口更新 → 聚合计算 → 结果输出
```

**变体1：滑动窗口**
- 维护固定大小的窗口，新数据加入时旧数据移除
- 适用于均值、最大值等聚合计算

**变体2：蓄水池采样**
- 从未知长度的数据流中均匀随机采样
- 适用于随机抽样

**变体3：概率数据结构**
- HyperLogLog 基数估计、Bloom Filter
- 适用于大数据量去重统计

**变体4：Top-K 频繁项**
- Lossy Counting 算法
- 适用于热点数据统计

## 4. 技术演进路径

```
精确算法 → 近似算法 → 概率数据结构 → 分布式流处理
```

## 5. 我们的选择

基于调研，实现轻量级流式算法库：
- 滑动窗口聚合
- 蓄水池采样
- HyperLogLog 基数估计
- Top-K 频繁项

### 选择理由
- 覆盖最常用的流式算法场景
- 零外部依赖
- 清晰的 API 设计
