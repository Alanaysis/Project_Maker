# 需求分析文档

## 1. 用户画像

### 主要用户

**用户类型**：Web3 初学者 / Solidity 学习者

**用户特征**：
- 背景：有 JavaScript/TypeScript 基础，想学习区块链开发
- 技能水平：编程基础扎实，但 Solidity 零基础
- 学习目标：理解 ERC20 标准，能独立编写和部署智能合约

**用户痛点**：
1. 找不到一个完整的学习项目（从调研到部署）
2. 现有教程要么太简单要么太复杂
3. 不理解为什么要这样设计合约
4. 不知道如何测试和部署合约

**用户需求**：
1. 一个可以运行的完整项目
2. 详细的文档解释每个设计决策
3. 从简单到复杂的学习路径
4. 实际可部署的代码

### 次要用户

**用户类型**：面试准备者

**用户特征**：
- 背景：有一定 Solidity 基础
- 需求：能清晰解释 ERC20 的设计和实现

## 2. 使用场景

### 场景1：学习入门
**用户**：Web3 初学者
**目标**：理解 ERC20 是什么，如何实现
**步骤**：
1. 阅读 README 了解项目概览
2. 阅读 01-RESEARCH.md 了解背景
3. 运行 examples/basic-usage.js 看效果
4. 阅读 LearningToken.sol 源代码
5. 运行测试理解验证方式

**预期结果**：能理解 ERC20 的核心接口和实现逻辑

### 场景2：动手实践
**用户**：想动手写代码的学习者
**目标**：自己修改合约并测试
**步骤**：
1. 修改 LearningToken.sol 添加新功能
2. 编写新的测试用例
3. 运行测试验证修改
4. 部署到本地网络测试

**预期结果**：能独立修改和扩展 ERC20 合约

### 场景3：面试准备
**用户**：Web3 开发面试者
**目标**：能清晰解释 ERC20 的设计决策
**步骤**：
1. 阅读 03-DESIGN.md 理解设计决策
2. 阅读 LEARNING_NOTES.md 复习关键点
3. 能口述 approve/transferFrom 的工作原理
4. 能解释为什么需要 increaseAllowance

**预期结果**：能自信地回答 ERC20 相关面试问题

## 3. 功能需求

### 核心功能（必须）

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F001 | ERC20 基础接口 | 实现 name, symbol, decimals, totalSupply, balanceOf, transfer | P0 |
| F002 | 授权机制 | 实现 approve, transferFrom, allowance | P0 |
| F003 | 事件发射 | 实现 Transfer 和 Approval 事件 | P0 |
| F004 | 安全检查 | 零地址检查、余额检查、授权额度检查 | P0 |
| F005 | 单元测试 | 覆盖所有核心功能的测试用例 | P0 |
| F006 | 部署脚本 | 可部署到本地和测试网的脚本 | P0 |

### 扩展功能（可选）

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F101 | 销毁功能 | 允许持有者销毁自己的代币 | P1 |
| F102 | 铸造功能 | 仅 owner 可以铸造新代币 | P1 |
| F103 | 增减授权 | increaseAllowance/decreaseAllowance | P1 |
| F104 | Permit | EIP-2612 无 Gas 授权 | P2 |

## 4. 非功能需求

| 需求 | 描述 | 目标 |
|------|------|------|
| 安全性 | 合约无漏洞 | 通过所有测试用例 |
| 可读性 | 代码有充分注释 | 新手能理解每一行 |
| 可测试性 | 所有功能可测试 | 测试覆盖率 > 90% |
| 文档完整性 | 文档覆盖所有方面 | 包含调研、设计、开发文档 |

## 5. 边界定义

### 包含范围
- ERC20 标准的完整实现
- 单元测试和集成测试
- 部署脚本
- 交互脚本
- 完整的文档

### 不包含范围
- 前端界面（仅提供交互脚本）
- 多链部署
- 代币经济学设计
- 治理机制
- 跨链桥接
