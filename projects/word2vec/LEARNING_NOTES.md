# Word2Vec 学习笔记

## 1. 词向量基础

### 1.1 什么是词向量

词向量（Word Embedding）是将词语映射到低维稠密向量空间的技术。相比传统的 one-hot 编码，词向量能够：
- 捕获语义信息
- 降低维度
- 使词语之间可以计算相似度

### 1.2 词向量的核心思想

**分布式假设**：一个词的含义由它的上下文决定（You shall know a word by the company it keeps）。

相似上下文的词语具有相似的词向量。

### 1.3 词表示方法演进

| 方法 | 维度 | 特点 | 缺点 |
|------|------|------|------|
| One-hot | |V| | 简单 | 维度高、无语义 |
| TF-IDF | |V| | 考虑词频 | 维度高、稀疏 |
| Word2Vec | d (50-300) | 语义丰富 | 需要大量语料 |
| GloVe | d | 全局统计 | 需要共现矩阵 |
| BERT | 768+ | 上下文相关 | 计算量大 |

## 2. Word2Vec 模型

### 2.1 两种架构

**CBOW (Continuous Bag of Words)**
- 输入：上下文词
- 输出：中心词
- 隐藏层：上下文词向量的平均
- 适合：频繁词，训练速度快

**Skip-gram**
- 输入：中心词
- 输出：上下文词
- 无隐藏层
- 适合：稀有词，语义准确度高

### 2.2 Skip-gram 详解

给定中心词 w_t，预测其上下文窗口内的词 w_{t+j}：

P(w_{t+j} | w_t) = exp(v'_w_j · v_w_t) / sum_w exp(v'_w · v_w_t)

其中：
- v_w：词 w 的输入向量
- v'_w：词 w 的输出向量

### 2.3 CBOW 详解

给定上下文词 {w_{t-c}, ..., w_{t-1}, w_{t+1}, ..., w_{t+c}}，预测中心词 w_t：

h = mean(v_{w_{t-c}}, ..., v_{w_{t+c}})  # 隐藏层

P(w_t | context) = softmax(W_out · h)

### 2.4 训练目标

最大化对数似然：

J = sum_{t=1}^{T} sum_{-c<=j<=c, j!=0} log P(w_{t+j} | w_t)

## 3. 训练优化技术

### 3.1 负采样（Negative Sampling）

**为什么需要负采样**：原始 Softmax 需要遍历整个词汇表，计算量 O(|V|) 太大。

**原理**：将多分类问题转化为二分类问题：
- 正样本：真实的 (中心词, 上下文词) 对
- 负样本：随机采样的 (中心词, 随机词) 对

**目标函数**：
L = log σ(v'_o · v_w) + sum_{i=1}^{k} E_{w_i ~ P_n(w)} [log σ(-v'_w_i · v_w)]

**噪声分布**：使用词频的 3/4 次方作为采样概率：
P_n(w) = f(w)^{3/4} / sum_{w'} f(w')^{3/4}

### 3.2 层次 Softmax（Hierarchical Softmax）

**原理**：使用 Huffman 树表示词汇表，将计算复杂度从 O(V) 降到 O(log V)。

**特点**：
- 每个词对应树中的一条从根到叶的路径
- 每一步是一个二分类决策（左/右）
- 高频词路径短（靠近根节点）
- 使用 Huffman 编码确保最优

**优点**：
- 不需要负采样
- 计算效率高
- 适合稀有词

### 3.3 降采样（Subsampling）

**问题**：高频词（如 "the", "a", "is"）在语料中出现次数远多于低频词，但它们携带的语义信息较少。

**解决方案**：根据词频概率性地丢弃高频词。

**采样概率公式**：
P(keep) = sqrt(t / f(w)) + t / f(w)

其中 f(w) 是词频（归一化），t 是阈值（通常 1e-3 到 1e-5）。

**效果**：
- 加速训练（减少训练样本）
- 提升低频词的表示质量
- 平衡词频差异

### 3.4 其他训练技巧

**动态窗口**：随机选择窗口大小（1 到 window），增加训练的多样性。

**学习率衰减**：线性衰减学习率，从初始值衰减到初始值的 1%。

**梯度裁剪**：限制梯度范数，防止梯度爆炸。

## 4. 实现细节

### 4.1 词汇表构建

1. 统计词频
2. 过滤低频词
3. 构建词到索引的映射

### 4.2 训练数据生成

**Skip-gram**：
1. 滑动窗口遍历语料
2. 生成 (中心词, 上下文词) 对
3. 对每个正样本生成 k 个负样本

**CBOW**：
1. 滑动窗口遍历语料
2. 收集上下文词索引
3. 生成 (上下文词列表, 中心词) 对

### 4.3 参数更新

使用随机梯度下降（SGD）更新参数：

θ = θ - η · ∇_θ J

## 5. 评估方法

### 5.1 内在评估

- **词相似度**：计算词对相似度与人工评分的相关性（Spearman 相关系数）
- **类比任务**：man:king = woman:? (king - man + woman = queen)

### 5.2 外在评估

- 下游任务性能提升（文本分类、情感分析等）

### 5.3 可视化评估

- **PCA 降维**：快速预览词向量分布
- **t-SNE 降维**：更精确地展示聚类结构

## 6. 实际应用

### 6.1 文本分类

使用词向量的平均作为文档表示，然后用最近邻或逻辑回归进行分类。

### 6.2 情感分析

基于情感词典和词向量相似度，计算句子的情感得分。

### 6.3 词聚类

使用 K-Means 对词向量进行聚类，发现语义相近的词组。

## 7. 常见问题

### 7.1 训练不收敛

- 学习率过大/过小
- 负样本数量不合适
- 语料量不足

### 7.2 向量质量差

- 语料预处理不充分
- 窗口大小不合适
- 训练轮数不足

### 7.3 CBOW vs Skip-gram 选择

- 数据量小、追求训练速度 -> CBOW
- 数据量大、追求质量 -> Skip-gram
- 有大量低频词 -> Skip-gram

### 7.4 负采样 vs 层次 Softmax 选择

- 实现简单、效果好 -> 负采样
- 对稀有词效果好 -> 层次 Softmax
