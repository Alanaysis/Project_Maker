# Word2Vec 调研文档

## 1. 技术背景

### 1.1 词表示方法演进

| 方法 | 维度 | 特点 | 缺点 |
|------|------|------|------|
| One-hot | |V| | 简单 | 维度高、无语义 |
| TF-IDF | |V| | 考虑词频 | 维度高、稀疏 |
| Word2Vec | d (50-300) | 语义丰富 | 需要大量语料 |
| GloVe | d | 全局统计 | 需要共现矩阵 |
| BERT | 768+ | 上下文相关 | 计算量大 |

### 1.2 Word2Vec 的优势

1. **低维稠密**：通常 50-300 维
2. **语义捕获**：能够编码语义关系
3. **计算高效**：训练和推理速度快
4. **通用性强**：可用于多种下游任务

## 2. 算法调研

### 2.1 CBOW vs Skip-gram

**CBOW (Continuous Bag of Words)**
```
输入: [The, cat, on, mat]
输出: sat
```
- 平均上下文词向量
- 训练速度快
- 适合高频词
- 对小型数据集效果好

**Skip-gram**
```
输入: sat
输出: [The, cat, on, mat]
```
- 分别预测每个上下文词
- 语义更准确
- 适合稀有词
- 对大型数据集效果好

**选择建议**：
- 数据量小、追求训练速度 -> CBOW
- 数据量大、追求质量 -> Skip-gram
- 有大量低频词 -> Skip-gram

### 2.2 训练优化技术

#### 层次 Softmax

使用 Huffman 树表示词汇表，将计算复杂度从 O(V) 降到 O(log V)。

**原理**：
- 每个词对应树中的一条从根到叶的路径
- 每一步是一个二分类决策（左/右）
- 高频词路径短（靠近根节点）
- 使用 Huffman 编码确保最优

**优点**：
- 不需要负采样
- 计算效率高
- 适合稀有词

**缺点**：
- 实现复杂
- 对高频词效果不如负采样

#### 负采样

将多分类问题转化为二分类问题。

**原理**：
- 正样本：真实的 (中心词, 上下文词) 对
- 负样本：随机采样的 (中心词, 随机词) 对
- 每次只更新 k+1 个样本的梯度

**噪声分布**：
使用词频的 3/4 次方作为采样概率：
P_n(w) = f(w)^{3/4} / sum(f(w')^{3/4})

**优点**：
- 实现简单
- 计算效率高
- 效果好

**缺点**：
- 需要选择合适的负样本数量
- 对高频词可能采样不足

#### 降采样（Subsampling）

高频词（如 "the", "a", "is"）在语料中出现次数远多于低频词，但它们携带的语义信息较少。

**采样概率公式**：
P(keep) = sqrt(t / f(w)) + t / f(w)

其中 f(w) 是词频（归一化），t 是阈值（通常 1e-3 到 1e-5）。

**效果**：
- 加速训练（减少训练样本）
- 提升低频词的表示质量
- 平衡词频差异

## 3. 实现方案调研

### 3.1 纯 NumPy 实现

**优点**：
- 学习原理最透彻
- 无外部依赖
- 代码可控性强

**缺点**：
- 实现复杂度高
- 性能较低

**选择理由**：作为学习项目，理解原理最重要

### 3.2 关键数据结构

```python
# 词汇表
vocab: Dict[str, int]  # word -> index
word_freq: Dict[str, int]  # word -> frequency

# 模型参数
W_in: np.ndarray  # 输入矩阵 (vocab_size, dim)
W_out: np.ndarray  # 输出矩阵 (vocab_size, dim)
```

### 3.3 模型对比

| 特性 | Skip-gram | CBOW |
|------|-----------|------|
| 输入 | 中心词 | 上下文词 |
| 输出 | 上下文词 | 中心词 |
| 隐藏层 | 无 | 上下文平均 |
| 训练速度 | 较慢 | 较快 |
| 语义质量 | 较好 | 一般 |
| 适合场景 | 大数据、稀有词 | 小数据、高频词 |

## 4. 性能指标

### 4.1 训练性能

| 指标 | 目标值 |
|------|--------|
| 训练速度 | > 1000 words/sec |
| 内存占用 | < 100MB (小语料) |
| 收敛轮数 | 50-200 epochs |

### 4.2 质量指标

| 评估任务 | 目标 |
|----------|------|
| 词相似度相关性 | > 0.5 |
| 类比准确率 | > 30% |

## 5. 风险评估

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 训练不收敛 | 高 | 调整学习率、增加语料 |
| 内存溢出 | 中 | 限制词汇表大小 |
| 训练过慢 | 中 | 使用负采样、减少维度 |
| 向量质量差 | 中 | 增加训练轮数、调整窗口 |

## 6. 结论

采用纯 NumPy 实现 Skip-gram + CBOW 双架构，支持负采样和层次 Softmax 两种优化方式，既能深入理解原理，又能保证实现可行性。
