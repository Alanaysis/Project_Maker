# 产品思考：C++ 内存模型和并发

## 学习目标

### 知识目标
1. 理解 C++ 内存模型的核心概念
2. 掌握五种内存序的含义和使用场景
3. 理解原子操作的原理和实现
4. 掌握常见并发数据结构的设计

### 技能目标
1. 能够选择正确的内存序
2. 能够实现简单的无锁数据结构
3. 能够使用 C++20 并发原语
4. 能够诊断和解决并发问题

### 素养目标
1. 培养并发思维
2. 建立性能意识
3. 养成安全编程习惯

## 关键要点

### 1. 内存序不是魔法
内存序是编译器和硬件的优化指导，不是神秘咒语。理解其本质是"允许省略哪些内存屏障"。

### 2. 无锁不等于高性能
无锁编程有额外开销（CAS 重试、缓存行弹跳），在低竞争场景下可能比有锁更慢。

### 3. 正确性优先于性能
先保证正确，再优化性能。过早优化是万恶之源。

### 4. 测试不能证明正确
并发 bug 难以复现，测试只能发现 bug，不能证明没有 bug。需要形式化验证或仔细推理。

## 设计决策

### 为什么选择 C++20？
- std::latch、std::barrier 等新原语简化了并发编程
- std::jthread 和 std::stop_token 提供了更好的线程管理
- 协程支持为异步编程提供了新范式

### 为什么包含性能测试？
- 理论知识需要实践验证
- 不同内存序的性能差异需要量化
- 帮助建立性能直觉

### 为什么强调无锁数据结构？
- 是并发编程的高级主题
- 深入理解内存模型的最佳途径
- 在高性能场景有实际应用

## 学习建议

### 第一阶段：理解基础
- 先理解内存模型基础
- 不要急于实现无锁结构
- 用画图辅助理解内存序

### 第二阶段：动手实践
- 每个示例都亲手运行
- 尝试修改参数观察变化
- 用 ThreadSanitizer 检测错误

### 第三阶段：深入研究
- 阅读标准文档
- 研究真实项目的并发实现
- 参与开源项目贡献

## 常见陷阱

### 1. 过度使用 seq_cst
seq_cst 是最保守的内存序，但不是万能的。过度使用会导致不必要的性能损失。

### 2. 忽视 ABA 问题
CAS 操作可能遇到 ABA 问题，需要使用版本号或 hazard pointer 解决。

### 3. 混淆原子性和顺序性
原子性保证操作不被中断，顺序性保证操作的可见顺序。两者是不同的概念。

### 4. 忘记内存对齐
未对齐的原子操作可能导致未定义行为或性能下降。
