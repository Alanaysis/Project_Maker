# 需求分析文档

## 1. 项目概述

### 1.1 项目背景

社交聊天应用是现代互联网最基础的通信工具之一。本项目旨在实现一个功能完整的即时通讯系统，涵盖用户管理、消息传递、社交互动等核心功能。

### 1.2 项目目标

1. 实现实时双向通信
2. 支持多种消息类型
3. 提供完整的社交功能
4. 保证消息的可靠传递
5. 提供良好的用户体验

### 1.3 目标用户

- 个人用户：日常社交聊天
- 小型团队：内部沟通协作
- 开发者：学习即时通讯技术

## 2. 功能需求

### 2.1 用户系统

#### 2.1.1 用户注册

**需求描述**: 新用户可以创建账号

**功能点**:
- 用户名注册（唯一性校验）
- 邮箱注册（格式校验）
- 密码设置（强度要求）
- 昵称设置（可选）

**输入验证**:
- 用户名：3-20位，字母数字下划线
- 邮箱：有效邮箱格式
- 密码：6-50位

**业务规则**:
- 用户名不可重复
- 邮箱不可重复
- 密码需加密存储

#### 2.1.2 用户登录

**需求描述**: 已注册用户可以登录系统

**功能点**:
- 用户名/密码登录
- JWT令牌生成
- 登录状态维护

**安全要求**:
- 密码传输加密
- 登录失败限制
- 令牌过期机制

#### 2.1.3 用户资料

**需求描述**: 用户可以管理个人信息

**功能点**:
- 查看个人资料
- 修改昵称
- 修改头像
- 修改个人简介

#### 2.1.4 用户状态

**需求描述**: 显示用户在线状态

**状态类型**:
- 在线（online）
- 离线（offline）
- 离开（away）
- 忙碌（busy）

### 2.2 好友管理

#### 2.2.1 添加好友

**需求描述**: 用户可以添加其他用户为好友

**功能点**:
- 搜索用户（用户名/邮箱）
- 发送好友请求
- 附带验证消息

**业务规则**:
- 不能添加自己
- 不能重复添加
- 好友请求需对方确认

#### 2.2.2 好友请求处理

**需求描述**: 用户可以处理收到的好友请求

**功能点**:
- 查看待处理请求
- 接受请求
- 拒绝请求

**业务规则**:
- 接受后双向建立好友关系
- 拒绝后请求标记为已拒绝

#### 2.2.3 好友列表

**需求描述**: 用户可以查看好友列表

**功能点**:
- 显示所有好友
- 显示好友在线状态
- 点击好友开始聊天

#### 2.2.4 删除好友

**需求描述**: 用户可以删除好友

**业务规则**:
- 双向删除好友关系
- 删除后聊天记录保留

### 2.3 聊天功能

#### 2.3.1 单聊

**需求描述**: 两个用户之间的一对一聊天

**功能点**:
- 发送文本消息
- 接收实时消息
- 消息历史记录
- 消息分页加载

**消息格式**:
```json
{
  "id": 1,
  "sender_id": 1,
  "receiver_id": 2,
  "content": "Hello!",
  "message_type": "text",
  "created_at": "2024-01-01T00:00:00Z"
}
```

#### 2.3.2 群聊

**需求描述**: 多个用户组成的群组聊天

**功能点**:
- 创建群组
- 邀请成员
- 移除成员
- 退出群组
- 群组消息

**群组信息**:
```json
{
  "id": 1,
  "name": "我的群组",
  "description": "群组描述",
  "owner_id": 1,
  "member_count": 10,
  "created_at": "2024-01-01T00:00:00Z"
}
```

#### 2.3.3 消息类型

**需求描述**: 支持多种消息类型

**消息类型**:
1. **文本消息**: 纯文本内容
2. **图片消息**: 图片文件
3. **文件消息**: 任意文件

**文件消息格式**:
```json
{
  "message_type": "file",
  "content": "文件描述",
  "file_url": "https://example.com/file.pdf",
  "file_name": "document.pdf",
  "file_size": 1024000
}
```

### 2.4 消息管理

#### 2.4.1 消息存储

**需求描述**: 所有消息需要持久化存储

**存储要求**:
- 消息内容
- 发送者信息
- 接收者信息
- 时间戳
- 消息状态

#### 2.4.2 消息搜索

**需求描述**: 用户可以搜索历史消息

**搜索功能**:
- 关键词搜索
- 按时间范围搜索
- 按发送者搜索

#### 2.4.3 离线消息

**需求描述**: 用户离线时接收的消息需要保存

**功能点**:
- 离线消息存储
- 上线后同步
- 消息状态更新

### 2.5 实时通知

#### 2.5.1 在线状态通知

**需求描述**: 好友上线/下线时通知用户

**通知内容**:
- 用户ID
- 用户名
- 状态变更（上线/下线）
- 时间戳

#### 2.5.2 消息推送

**需求描述**: 新消息实时推送给用户

**推送策略**:
- 在线用户：WebSocket直接推送
- 离线用户：存储为离线消息

#### 2.5.3 已读回执

**需求描述**: 消息被阅读后通知发送者

**功能点**:
- 标记消息为已读
- 通知发送者
- 统计未读数量

#### 2.5.4 输入状态

**需求描述**: 显示对方正在输入的状态

**功能点**:
- 开始输入通知
- 停止输入通知
- 超时自动隐藏

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 响应时间 | < 100ms |
| 并发连接 | > 1000 |
| 消息延迟 | < 200ms |
| 数据库查询 | < 50ms |

### 3.2 可用性需求

- 系统可用性 > 99.9%
- 故障恢复时间 < 5分钟
- 数据备份策略

### 3.3 安全需求

- 密码加密存储（bcrypt）
- JWT令牌认证
- 输入数据验证
- SQL注入防护
- XSS防护

### 3.4 可扩展性需求

- 支持水平扩展
- 模块化设计
- 接口标准化

## 4. 用户故事

### 4.1 用户注册

```
作为一个新用户
我希望能够注册账号
以便使用聊天功能

验收标准：
- 可以使用用户名、邮箱、密码注册
- 用户名和邮箱唯一性校验
- 密码强度验证
- 注册成功后自动登录
```

### 4.2 添加好友

```
作为一个用户
我希望能够添加好友
以便与他们聊天

验收标准：
- 可以搜索用户
- 可以发送好友请求
- 可以附带验证消息
- 对方可以接受或拒绝
```

### 4.3 发送消息

```
作为一个用户
我希望能够发送消息
以便与好友交流

验收标准：
- 可以发送文本消息
- 消息实时送达
- 显示消息发送状态
- 支持消息历史查看
```

### 4.4 创建群组

```
作为一个用户
我希望能够创建群组
以便多人聊天

验收标准：
- 可以创建群组
- 可以邀请好友加入
- 群组内可以发送消息
- 可以管理群组成员
```

## 5. 数据需求

### 5.1 数据实体

1. **User**: 用户信息
2. **Message**: 消息记录
3. **Group**: 群组信息
4. **Friendship**: 好友关系
5. **GroupMember**: 群组成员
6. **FriendRequest**: 好友请求
7. **OfflineMessage**: 离线消息

### 5.2 数据关系

```
User 1:N Message (发送者)
User 1:N Message (接收者)
User N:N User (好友关系)
User 1:N Group (拥有的群组)
User N:N Group (加入的群组)
Group 1:N Message (群组消息)
```

### 5.3 数据量预估

| 数据类型 | 初始数据量 | 增长速度 |
|----------|------------|----------|
| 用户 | 100 | 10/天 |
| 消息 | 1000 | 100/天 |
| 群组 | 10 | 1/天 |

## 6. 接口需求

### 6.1 HTTP API接口

| 接口 | 方法 | 路径 | 描述 |
|------|------|------|------|
| 注册 | POST | /api/register | 用户注册 |
| 登录 | POST | /api/login | 用户登录 |
| 搜索用户 | GET | /api/users/search | 搜索用户 |

### 6.2 WebSocket消息接口

| 消息类型 | 方向 | 描述 |
|----------|------|------|
| send_message | C->S | 发送消息 |
| new_message | S->C | 新消息通知 |
| typing | C->S | 输入状态 |
| read_receipt | C->S | 已读回执 |
| heartbeat | C->S | 心跳检测 |
