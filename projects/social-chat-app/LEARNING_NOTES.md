**WebSocket vs HTTP 对比**：

| 特性 | HTTP | WebSocket |
|------|------|-----------|
| 通信方式 | 单向（请求-响应） | 全双工 |
| 连接方式 | 短连接 | 长连接 |
| 服务器推送 | 不支持 | 支持 |
| 协议开销 | 每次请求都有头部 | 建立连接后开销小 |

**关键概念**：
- 握手过程：HTTP Upgrade 请求
- 消息帧：文本帧、二进制帧、控制帧

#### 笔记

**加密类型**：
- 对称加密：AES，速度快，密钥分发难
- 非对称加密：RSA，速度慢，密钥分发易
- 混合加密：结合两者优点

**端到端加密流程**：
1. 生成密钥对
2. 交换公钥
3. 使用对方公钥加密
4. 使用自己的私钥解密

**思考**：

## 技术深度笔记

### Go 并发编程

**Goroutine**：
```go
go func() {
    // 并发执行
}()
```

**Channel**：
```go
ch := make(chan int, 100)  // 带缓冲的 channel
ch <- 1                    // 发送
value := <-ch              // 接收
```

**Select**：
```go
select {
case msg := <-ch1:
    // 处理 ch1
case msg := <-ch2:
    // 处理 ch2
default:
    // 默认处理
}
```

**思考**：
1. Goroutine 与线程的区别？
2. 什么时候使用带缓冲的 Channel？
3. 如何避免 Goroutine 泄漏？

### 设计模式

**观察者模式**：
- 用于消息推送
- 用户订阅消息频道
- 新消息时通知所有订阅者

**连接池模式**：
- 复用数据库连接
- 减少连接开销
- 控制并发数

**工厂模式**：
- 创建不同类型的处理器
- 解耦对象创建和使用
