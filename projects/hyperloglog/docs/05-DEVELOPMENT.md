### 阶段 2: HyperLogLog 核心实现

**目标**: 实现 HyperLogLog 数据结构

**关键决策**:
- 使用 `uint8` 存储寄存器
- 预计算 alpha 常数
- 使用位运算优化

**遇到的问题**:

1. **前导零计数的边界情况**
   - 问题: 当剩余位全为 0 时，前导零数量应该是 64-p
   - 原因: `bits.LeadingZeros64(0)` 返回 64
   - 解决: 加 1 避免 log(0) 问题

2. **分桶索引的位运算**
   - 问题: `hash >> (64 - p)` 可能溢出
   - 原因: 当 p=0 时，右移 64 位
   - 解决: 验证 p >= 4

**代码量**: ~250 行

### 阶段 3: 基数估算实现

**目标**: 实现基数估算算法

**关键决策**:
- 使用调和平均数
- 实现小基数校正 (Linear Counting)
- 实现大基数校正

**遇到的问题**:

1. **浮点精度问题**
   - 问题: 大基数估算可能溢出
   - 原因: `2^64` 超出 float64 范围
   - 解决: 使用大基数校正公式

2. **Linear Counting 的阈值**
   - 问题: 何时使用 Linear Counting?
   - 原经: 当 estimate <= 2.5 * m 时
   - 解决: 参考论文中的建议

**代码量**: ~50 行

## Bug 修复记录

### Bug 1: 前导零计数错误

**现象**: 小基数估算不准确

**原因**: 当剩余位全为 0 时，前导零数量应该是 64-p，但代码返回 0

**修复**: 加 1 避免 log(0) 问题

```go
// 修复前
leadingZeros := uint8(bits.LeadingZeros64(remainingBits))

// 修复后
leadingZeros := uint8(bits.LeadingZeros64(remainingBits)) + 1
```

### Bug 2: 大基数估算溢出

**现象**: 大基数估算返回 0

**原因**: `2^64` 超出 float64 范围，导致溢出

**修复**: 使用大基数校正公式

```go
// 修复前
estimate = h.alpha * float64(h.m) * float64(h.m) / sum

// 修复后
if estimate > 143165576.5 { // 2^32 / 30
    estimate = -math.Pow(2, 32) * math.Log(1-estimate/math.Pow(2, 32))
}
```
