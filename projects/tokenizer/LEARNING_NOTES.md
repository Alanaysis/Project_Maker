# 学习笔记

## 中文分词学习总结

### 1. 分词算法理解

#### 正向最大匹配 (FMM)

**核心思想**：
- 从左到右扫描文本
- 每次取最长可能的词
- 如果词典中存在则切分，否则减少一个字符继续尝试

**示例**：
```
文本：我爱北京天安门
词典：我、爱、北京、天安门

步骤：
1. 取 "我爱北京天" -> 不在词典
2. 取 "我爱北京" -> 不在词典
3. 取 "我爱北" -> 不在词典
4. 取 "我爱" -> 不在词典
5. 取 "我" -> 在词典，切分
6. 继续处理 "爱北京天安门"
...
```

**优点**：
- 实现简单
- 速度快

**缺点**：
- 无法处理歧义
- 依赖词典质量

#### 逆向最大匹配 (BMM)

**核心思想**：
- 从右到左扫描文本
- 每次取最长可能的词

**与 FMM 的区别**：
- 扫描方向相反
- 对某些歧义处理不同

**示例**：
```
文本：研究生命的起源
FMM：研究生/命/的/起源
BMM：研究/生命/的/起源
```

**学习心得**：
- BMM 在某些情况下比 FMM 效果好
- 实际应用中通常结合两者

### 2. HMM 分词

#### 核心概念

**状态定义**：
- B (Begin)：词的开始
- M (Middle)：词的中间
- E (End)：词的结束
- S (Single)：单字词

**示例**：
```
文本：我爱北京
状态：S S B E
分词：我/爱/北京
```

**模型要素**：
1. **初始概率 π**：句子以某个状态开头的概率
2. **转移概率 A**：从状态 i 转移到状态 j 的概率
3. **发射概率 B**：状态 i 生成观测值 o 的概率

#### 维特比算法

**目的**：找到最可能的状态序列

**算法步骤**：
1. 初始化：计算每个状态的初始概率
2. 递推：对于每个时间步，计算每个状态的最大概率
3. 回溯：从最后一个状态开始，回溯找到最优路径

**时间复杂度**：O(T * N^2)
- T：序列长度
- N：状态数量

**学习心得**：
- 维特比算法是动态规划的应用
- 使用对数概率可以避免数值下溢
- 路径回溯是关键步骤

### 4. 分词评估

#### 评估指标

**精确率 (Precision)**：
```
P = 正确切分的词数 / 切分出的词数
```

**召回率 (Recall)**：
```
R = 正确切分的词数 / 标准答案的词数
```

**F1 值**：
```
F1 = 2 * P * R / (P + R)
```

**示例**：
```
标准：我/爱/北京/天安门
切分：我/爱/北京天/安门

正确：我、爱、北京
切分出：我、爱、北京天、安门（4个）
标准答案：我、爱、北京、天安门（4个）

P = 3/4 = 0.75
R = 3/4 = 0.75
F1 = 0.75
```

#### 常见问题

1. **歧义问题**
   - "研究生命的起源"
   - FMM 和 BMM 结果不同
   - 解决：结合两种算法或使用统计方法

2. **未登录词**
   - 词典中没有的词
   - 解决：使用 HMM 等统计方法

3. **分词粒度**
   - "中华人民共和国"
   - 可以切分为 "中华/人民/共和国" 或 "中华人民共和国"
   - 解决：根据应用需求调整
