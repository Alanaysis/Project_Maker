# 需求分析文档

> Industrial Vision Detection - Requirements Analysis

## 1. 项目定位

### 1.1 项目性质

本项目是一个**学习型项目**，旨在：

1. 理解工业视觉检测的核心技术
2. 掌握目标检测模型的实现原理
3. 学习从训练到部署的完整流程
4. 培养工程实践能力

### 1.2 目标用户画像

#### 主要用户：AI 学习者

| 特征 | 描述 |
|------|------|
| 技术背景 | 有 Python 基础，了解深度学习概念 |
| 学习目标 | 理解目标检测原理，掌握工程实践 |
| 使用场景 | 学习、实验、课程项目 |
| 痛点 | 理论与实践脱节，缺乏完整项目经验 |

#### 次要用户：工业视觉工程师

| 特征 | 描述 |
|------|------|
| 技术背景 | 有视觉检测经验，需要学习深度学习方法 |
| 学习目标 | 将 DL 技术应用到工业检测 |
| 使用场景 | 技术预研、方案验证 |
| 痛点 | 缺乏 DL 模型训练和优化经验 |

## 2. 功能需求

### 2.1 核心功能

#### F1: 数据处理

**需求描述**: 支持工业图像数据的加载、预处理和增强

**功能点**:
- [ ] 支持 COCO 格式数据加载
- [ ] 支持 YOLO 格式数据加载
- [ ] 支持 VOC 格式数据加载
- [ ] 图像预处理 (归一化、resize)
- [ ] 基础数据增强 (翻转、旋转、缩放)
- [ ] 高级数据增强 (Mosaic、MixUp)

**验收标准**:
- 能正确加载三种格式的标注
- 数据增强后标注同步变换
- 增强策略可配置

#### F2: 模型定义

**需求描述**: 实现 YOLOv8 风格的目标检测模型

**功能点**:
- [ ] 骨干网络 (Backbone) 实现
- [ ] 特征融合 (Neck) 实现
- [ ] 检测头 (Head) 实现
- [ ] 完整模型组装
- [ ] 模型配置可定制

**验收标准**:
- 模型能接受图像输入并输出检测结果
- 支持不同大小的模型配置
- 代码结构清晰、模块化

#### F3: 损失函数

**需求描述**: 实现目标检测常用损失函数

**功能点**:
- [ ] 分类损失 (BCE/Focal Loss)
- [ ] 回归损失 (CIoU/DIoU/GIoU)
- [ ] 目标性损失
- [ ] 总损失组合

**验收标准**:
- 损失函数计算正确
- 支持梯度反向传播
- 有数学公式文档

#### F4: 训练流程

**需求描述**: 实现完整的模型训练流程

**功能点**:
- [ ] 数据加载器配置
- [ ] 优化器配置 (SGD/Adam)
- [ ] 学习率调度
- [ ] 训练循环
- [ ] 验证评估
- [ ] 模型保存/加载
- [ ] 训练日志记录

**验收标准**:
- 能完成完整的训练流程
- 训练过程有日志输出
- 支持断点续训

#### F5: 评估指标

**需求描述**: 实现目标检测评估指标

**功能点**:
- [ ] IoU 计算
- [ ] Precision/Recall
- [ ] AP/mAP 计算
- [ ] 混淆矩阵

**验收标准**:
- 指标计算与标准实现一致
- 支持不同 IoU 阈值
- 输出可读的评估报告

#### F6: 推理预测

**需求描述**: 实现单张图像推理

**功能点**:
- [ ] 图像预处理
- [ ] 模型推理
- [ ] 后处理 (NMS)
- [ ] 结果格式化

**验收标准**:
- 能对单张图像进行推理
- 输出包含类别、置信度、边界框
- 推理时间可测量

#### F7: 模型导出

**需求描述**: 支持 ONNX 格式导出

**功能点**:
- [ ] PyTorch 到 ONNX 转换
- [ ] ONNX 模型验证
- [ ] ONNX 推理测试

**验收标准**:
- 导出的 ONNX 模型可被 ONNX Runtime 加载
- ONNX 推理结果与 PyTorch 一致

## 3. 数据需求

### 3.1 数据格式

#### COCO 格式

```json
{
  "images": [
    {
      "id": 1,
      "file_name": "image1.jpg",
      "width": 640,
      "height": 640
    }
  ],
  "annotations": [
    {
      "id": 1,
      "image_id": 1,
      "category_id": 1,
      "bbox": [x, y, width, height],
      "area": 1000,
      "iscrowd": 0
    }
  ],
  "categories": [
    {
      "id": 1,
      "name": "defect"
    }
  ]
}
```

#### YOLO 格式

```
# 每行一个目标
# class_id center_x center_y width height (归一化坐标)
0 0.5 0.5 0.3 0.4
1 0.2 0.3 0.1 0.2
```

### 3.2 数据集要求

| 项目 | 要求 |
|------|------|
| 图像格式 | JPG/PNG |
| 图像尺寸 | 任意 (训练时统一 resize) |
| 标注格式 | COCO/YOLO/VOC |
| 类别数 | 可配置 |
| 最小样本数 | 每类 100+ (建议) |

### 3.3 示例数据集

为方便学习，提供以下数据集选项：

1. **模拟数据集**: 项目生成的简单几何图形
2. **公开数据集**: MVTec AD、GC10-DET
3. **自定义数据集**: 用户自己的工业图像

## 4. 技术约束

### 4.1 环境约束

| 项目 | 最低要求 | 推荐配置 |
|------|----------|----------|
| Python | 3.8 | 3.10 |
| PyTorch | 1.12 | 2.0+ |
| 内存 | 8GB | 16GB+ |
| GPU | 无 (CPU可运行) | NVIDIA GPU 8GB+ |
| 磁盘 | 2GB | 10GB+ |

### 4.2 性能约束

| 指标 | 要求 |
|------|------|
| 训练速度 | 可接受 (不要求实时) |
| 推理速度 | 单张 < 100ms (GPU) |
| 内存占用 | < 4GB |

### 4.3 兼容性约束

- 支持 Linux/macOS/Windows
- 支持 CPU 和 GPU
- 兼容主流 Python 版本
