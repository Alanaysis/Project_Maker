# 学习笔记

## 1. RISC-V 指令集基础

### 1.1 什么是 RISC-V?

RISC-V 是一个开源的精简指令集架构 (ISA)。与 x86 (CISC) 和 ARM (RISC 但闭源) 不同，RISC-V 是完全开源和免费的。

**核心思想**: 精简指令集 = 每条指令做的事情少，但指令数量多，执行速度快。

### 1.2 RV32I 指令格式

RISC-V 有 6 种基本格式，所有指令都是 32 位宽:

```
R-type: 用于寄存器-寄存器运算 (ADD, SUB, AND, OR...)
I-type: 用于立即数运算和加载 (ADDI, LW...)
S-type: 用于存储 (SW, SH, SB...)
B-type: 用于条件分支 (BEQ, BNE, BLT...)
U-type: 用于高位立即数 (LUI, AUIPC)
J-type: 用于跳转 (JAL)
```

**关键理解**: 每种格式的立即数位置不同，需要根据格式来提取和符号扩展。

### 1.3 寄存器

RISC-V 有 32 个通用寄存器 (x0-x31):
- **x0 (zero)**: 硬连线为零，写入无效
- **x1 (ra)**: 返回地址
- **x2 (sp)**: 栈指针
- **x10-x11 (a0-a1)**: 函数参数/返回值
- **x5-x7, x28-x31 (t0-t6)**: 临时寄存器
- **x8-x9, x18-x27 (s0-s11)**: 保存寄存器

**设计亮点**: x0 硬连线为零，简化了很多指令的设计。例如，`MV rd, rs` 实际上是 `ADDI rd, rs, 0` 的别名。

## 2. 指令解码

### 2.1 操作码识别

指令的低 7 位是操作码 (opcode):
```
0x33 (0110011) = R-type ALU
0x13 (0010011) = I-type ALU
0x03 (0000011) = Load
0x23 (0100011) = Store
0x63 (1100011) = Branch
0x6F (1101111) = JAL
0x67 (1100111) = JALR
0x37 (0110111) = LUI
0x17 (0010111) = AUIPC
0x73 (1110111) = System
```

### 2.2 立即数符号扩展

不同格式的立即数位置不同:

- **I-type**: bits[31:20] -> 12 位符号扩展
- **S-type**: bits[31:25] + bits[11:7] -> 12 位符号扩展
- **B-type**: bits[31|7|30:25|11:8] -> 13 位符号扩展 (最低位为 0)
- **U-type**: bits[31:12] -> 直接使用 (高 20 位)
- **J-type**: bits[31|19:12|20|30:21] -> 21 位符号扩展 (最低位为 0)

**踩坑记录**: B-type 的立即数位排列非常不连续，很容易出错。

### 2.3 符号扩展的重要性

符号扩展是将 N 位有符号数扩展为 M 位 (M > N) 的过程:
```
12 位 -1 = 0xFFF
符号扩展为 32 位 = 0xFFFFFFFF
```

如果不正确地进行符号扩展，负数会变成正数，导致分支和跳转地址错误。

## 3. 指令执行

### 3.1 ALU 运算

算术和逻辑运算相对简单:
```c
// ADD: result = rs1 + rs2
// SUB: result = rs1 - rs2
// AND: result = rs1 & rs2
// OR:  result = rs1 | rs2
// XOR: result = rs1 ^ rs2
```

移位运算需要注意:
- SLL: 逻辑左移 (低位补零)
- SRL: 逻辑右移 (高位补零)
- SRA: 算术右移 (高位补符号位)

### 3.2 分支指令

分支指令比较 rs1 和 rs2，如果条件成立则跳转:
```c
BEQ:  if (rs1 == rs2) pc += imm
BNE:  if (rs1 != rs2) pc += imm
BLT:  if ((int32_t)rs1 < (int32_t)rs2) pc += imm  // 有符号
BGE:  if ((int32_t)rs1 >= (int32_t)rs2) pc += imm
BLTU: if (rs1 < rs2) pc += imm  // 无符号
BGEU: if (rs1 >= rs2) pc += imm
```

**关键点**: BLT/BGE 是有符号比较，BLTU/BGEU 是无符号比较。

### 3.3 内存访问

Load 指令从内存读取数据:
```c
LW:  rd = memory[rs1 + imm]         // 32 位
LH:  rd = sign_extend(memory[rs1 + imm])  // 16 位符号扩展
LHU: rd = zero_extend(memory[rs1 + imm])  // 16 位零扩展
LB:  rd = sign_extend(memory[rs1 + imm])  // 8 位符号扩展
LBU: rd = zero_extend(memory[rs1 + imm])  // 8 位零扩展
```

Store 指令将数据写入内存:
```c
SW: memory[rs1 + imm] = rs2  // 32 位
SH: memory[rs1 + imm] = rs2  // 16 位 (低 16 位)
SB: memory[rs1 + imm] = rs2  // 8 位 (低 8 位)
```

### 3.4 跳转指令

JAL (Jump And Link):
```c
rd = pc + 4      // 保存返回地址
pc = pc + imm    // 跳转
```

JALR (Jump And Link Register):
```c
rd = pc + 4              // 保存返回地址
pc = (rs1 + imm) & ~1   // 跳转 (清除最低位)
```

**设计模式**: JAL 用于函数调用 (保存返回地址到 ra)，JALR 用于函数返回。

## 4. 汇编器设计

### 4.1 两遍汇编

```
第一遍: 扫描标签
  - 记录每个标签的地址
  - 计算每条指令的 PC

第二遍: 生成代码
  - 汇编每条指令
  - 解析标签引用为相对偏移
```

### 4.2 标签解析

标签引用需要转换为相对于当前 PC 的偏移:
```c
offset = label_addr - current_pc
```

**踩坑记录**: 地址使用 `u32` 而非 `i32`，因为基地址 0x80000000 在 `i32` 中是负数。

### 4.3 LUI 立即数

LUI 指令的立即数是高 20 位，用户提供的值需要左移 12 位:
```asm
LUI t0, 0x800FF    # 用户写 0x800FF
# 实际编码: imm = 0x800FF << 12 = 0x800FF000
# t0 = 0x800FF000
```

## 5. 测试策略

### 5.1 单元测试
- 内存模块: 读写、对齐、边界
- 解码器: 各类型指令、立即数符号扩展
- 汇编器: 寄存器解析、指令编码

### 5.2 集成测试
- 算术运算: ADD, SUB, AND, OR, XOR, SLL, SRL
- 控制流: BEQ, BNE, BLT, BGE, JAL, JALR
- 内存访问: LUI, Load/Store
- 比较运算: SLT, SLTU
- 特殊: x0 硬连线零

### 5.3 测试中的教训
- 循环边界: BLT 是 "小于"，用 N 作为上界时需要 N+1
- 内存地址: 测试数据地址必须在内存范围内
- 立即数范围: ADDI 的 12 位有符号立即数范围是 -2048..2047

## 6. 总结

通过实现 RISC-V 模拟器，我深入理解了:

1. **指令集架构**: RISC-V 的精简设计和模块化思想
2. **指令编码**: 如何将汇编指令编码为二进制，以及如何反向解码
3. **CPU 工作原理**: 取指-解码-执行循环的每个步骤
4. **内存模型**: 字节寻址、小端序、对齐访问
5. **汇编器设计**: 两遍汇编、标签解析、符号表

这个项目是理解计算机体系结构的绝佳练习。
