## 关键要点

### 1. 内存管理要点

#### 核心原则
- **RAII**：资源获取即初始化
- **所有权明确**：每个资源有明确的所有者
- **自动管理**：优先使用智能指针

#### 关键技巧
```cpp
// 优先使用 unique_ptr
auto ptr = std::make_unique<int>(42);

// 需要共享时使用 shared_ptr
auto shared = std::make_shared<int>(42);

// 避免裸指针管理内存
int* raw = new int(42);  // 避免
```

### 2. 并发编程要点

#### 核心原则
- **最小化共享**：减少共享数据
- **使用同步**：正确使用同步原语
- **避免锁嵌套**：减少死锁风险

#### 关键技巧
```cpp
// 使用 mutex 保护共享数据
std::mutex mtx;
std::lock_guard<std::mutex> lock(mtx);

// 使用原子操作
std::atomic<int> counter{0};
counter.fetch_add(1, std::memory_order_relaxed);
```

### 3. 类型安全要点

#### 核心原则
- **类型明确**：避免隐式转换
- **边界检查**：检查数值边界
- **空值检查**：检查指针和引用

#### 关键技巧
```cpp
// 使用 explicit 防止隐式转换
class MyInt {
public:
    explicit MyInt(int val) : value(val) {}
private:
    int value;
};

// 使用 std::optional 表示可能为空
std::optional<int> find_value(int key);
```

### 4. 异常安全要点

#### 核心原则
- **异常安全级别**：明确异常安全保证
- **资源管理**：使用 RAII 管理资源
- **异常传播**：正确处理异常

#### 关键技巧
```cpp
// 使用 RAII 管理资源
class Resource {
public:
    Resource() { /* 获取资源 */ }
    ~Resource() { /* 释放资源 */ }
};

// 使用 noexcept 标记不抛出异常的函数
void safe_function() noexcept {
    // 不会抛出异常
}
```
