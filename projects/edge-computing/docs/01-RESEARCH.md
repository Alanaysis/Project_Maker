# 01 - 研究阶段

## 边缘计算概述

### 什么是边缘计算？

边缘计算（Edge Computing）是一种分布式计算范式，将计算和数据存储靠近数据源的位置，而不是依赖于中心化的数据中心。这种架构减少了数据传输的延迟和带宽使用，同时提高了应用的响应速度。

### 核心概念

1. **边缘节点**: 位于网络边缘的计算设备，可以是路由器、网关、专用服务器等
2. **任务卸载**: 将计算任务从终端设备迁移到边缘节点执行
3. **资源调度**: 合理分配边缘节点的计算资源
4. **负载均衡**: 在多个边缘节点之间均匀分配工作负载
5. **数据处理**: 在边缘侧进行数据过滤、聚合和分析
6. **云端同步**: 边缘与云端的数据和配置同步

### 与云计算的区别

| 特性 | 云计算 | 边缘计算 |
|------|--------|----------|
| 位置 | 集中式数据中心 | 分布式网络边缘 |
| 延迟 | 较高（100ms+） | 较低（1-10ms） |
| 带宽 | 需要大量带宽 | 减少数据传输 |
| 计算能力 | 强大 | 相对有限 |
| 适用场景 | 大规模计算 | 实时处理 |

## 任务卸载策略

### 1. 本地执行

任务在终端设备上本地执行，不进行卸载。

**适用场景**:
- 任务计算量小
- 网络连接不稳定
- 数据隐私要求高

### 2. 完全卸载

任务完全迁移到边缘节点执行。

**适用场景**:
- 终端设备计算能力不足
- 任务计算量大
- 需要访问边缘节点的特定资源

### 3. 部分卸载

任务被分解，部分在本地执行，部分在边缘节点执行。

**适用场景**:
- 任务可以并行化
- 需要平衡延迟和计算资源

## 负载均衡算法

### 1. 轮询（Round Robin）

按顺序将请求分配给各个服务器。

**算法**:
```
next_server = (current_server + 1) % total_servers
```

**优点**: 实现简单，分配公平
**缺点**: 不考虑服务器实际负载

### 2. 加权轮询（Weighted Round Robin）

根据服务器权重分配请求。

**算法**:
```
weight[i] = server_capacity[i] / sum(all_capacities)
```

**优点**: 考虑服务器能力差异
**缺点**: 权重配置需要准确

### 3. 最少连接（Least Connections）

选择当前连接数最少的服务器。

**算法**:
```
selected = server_with_min(active_connections)
```

**优点**: 动态均衡
**缺点**: 需要实时监控连接数

### 4. 最短响应时间（Least Response Time）

选择响应时间最短的服务器。

**算法**:
```
selected = server_with_min(avg_response_time)
```

**优点**: 用户体验最佳
**缺点**: 响应时间可能波动

### 5. IP 哈希（IP Hash）

根据客户端 IP 地址的哈希值选择服务器。

**算法**:
```
selected = hash(client_ip) % total_servers
```

**优点**: 同一客户端总是访问同一服务器
**缺点**: 可能导致分配不均

## 资源调度策略

### 1. 先来先服务（FCFS）

按照任务到达的顺序执行。

**优点**: 公平，实现简单
**缺点**: 可能导致长任务阻塞短任务

### 2. 短作业优先（SJF）

优先执行预计执行时间最短的任务。

**优点**: 平均等待时间最短
**缺点**: 可能导致长任务饥饿

### 3. 优先级调度

根据任务优先级执行。

**优点**: 保证重要任务优先处理
**缺点**: 低优先级任务可能饥饿

### 4. 时间片轮转（RR）

每个任务执行固定时间片后切换到下一个任务。

**优点**: 响应时间短
**缺点**: 上下文切换开销

### 5. 多级反馈队列（MFQ）

结合多种调度策略，根据任务行为动态调整优先级。

**优点**: 适应性强
**缺点**: 实现复杂

## 数据处理技术

### 数据过滤

边缘设备产生的数据通常包含噪声和冗余信息，需要在边缘侧进行过滤。

**过滤策略**:
1. **阈值过滤**: 过滤超出正常范围的值
2. **去重过滤**: 移除重复的数据点
3. **采样过滤**: 按比例或时间间隔采样
4. **质量过滤**: 基于数据质量分数过滤

### 数据聚合

将多个数据点聚合为更有意义的统计信息。

**聚合方式**:
1. **时间聚合**: 按时间窗口聚合（如每分钟平均值）
2. **空间聚合**: 按地理位置聚合
3. **设备聚合**: 按设备类型聚合
4. **维度聚合**: 按多个维度交叉聚合

### 规则引擎

根据预定义规则自动处理数据，支持复杂的业务逻辑。

**规则类型**:
1. **阈值规则**: 超过阈值触发动作
2. **模式规则**: 检测特定数据模式
3. **关联规则**: 多个条件组合触发
4. **时序规则**: 基于时间序列的规则

## IoT 数据处理

### 数据特点

1. **高频产生**: 传感器可能每秒产生多个数据点
2. **格式多样**: 不同传感器数据格式不同
3. **质量参差**: 数据可能包含噪声和异常值
4. **实时性要求**: 许多应用需要实时处理

### 处理流程

```
传感器数据 → 数据验证 → 数据清洗 → 数据转换 → 数据存储
                ↓           ↓           ↓           ↓
            格式检查    去噪去重    单位转换    时序数据库
                ↓           ↓           ↓           ↓
            异常检测    缺失填充    特征提取    数据聚合
```

### 实时分析

1. **统计分析**: 计算均值、方差、极值等
2. **趋势检测**: 识别数据上升/下降趋势
3. **异常检测**: 发现异常数据点
4. **预测分析**: 基于历史数据预测未来值

## 云端同步技术

### 同步模式

1. **实时同步**: 数据产生后立即同步
2. **批量同步**: 积累一定量数据后批量同步
3. **定时同步**: 按固定时间间隔同步
4. **事件触发同步**: 特定事件触发同步

### 冲突解决

当边缘和云端数据冲突时的解决策略：
1. **本地优先**: 使用边缘侧数据
2. **远程优先**: 使用云端数据
3. **最新优先**: 使用时间戳最新的数据
4. **手动解决**: 由用户手动选择

### 数据一致性

1. **最终一致性**: 允许短暂不一致，最终达到一致
2. **强一致性**: 任何时候数据都是一致的
3. **因果一致性**: 保证因果关系的数据一致

## 边缘计算架构

### 三层架构

```
+------------------+
|   终端设备层     |  (IoT 设备、手机、传感器)
+------------------+
        |
        v
+------------------+
|   边缘计算层     |  (边缘节点、网关)
+------------------+
        |
        v
+------------------+
|   云计算层       |  (数据中心)
+------------------+
```

### 关键组件

1. **边缘节点管理器**: 管理边缘节点的注册、发现和监控
2. **任务调度器**: 决定任务在哪个节点执行
3. **负载均衡器**: 在节点之间分配工作负载
4. **结果收集器**: 收集和聚合任务执行结果
5. **监控系统**: 监控系统状态和性能
6. **数据处理器**: 过滤、聚合和分析边缘数据
7. **同步管理器**: 管理边缘与云端的数据同步

## 技术选型

1. **简洁易学**: Python 语法简洁，适合学习和原型开发
2. **丰富的库**: 拥有大量成熟的库和框架

2. **灵活性**: 可以根据需求定制
3. **轻量级**: 减少依赖和复杂度
