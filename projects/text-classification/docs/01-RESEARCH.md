# 01 - 调研文档：文本分类

## 1. 文本分类概述

文本分类是自然语言处理（NLP）中最基础和最重要的任务之一。它的目标是将文本文档自动分配到一个或多个预定义的类别中。

### 1.1 应用场景

| 应用 | 描述 | 示例 |
|------|------|------|
| 情感分析 | 判断文本的情感倾向 | "这部电影很棒" → 正面 |
| 垃圾邮件检测 | 识别垃圾邮件 | "免费领取大奖" → 垃圾邮件 |
| 新闻分类 | 将新闻归类到主题 | "股市上涨" → 财经 |
| 话题分类 | 识别文本主题 | "Python编程" → 技术 |
| 语言检测 | 识别文本语言 | "Hello" → 英语 |

### 1.2 发展历程

```
规则方法 → 统计方法 → 机器学习 → 深度学习 → 预训练模型
(1960s)    (1980s)    (1990s)    (2010s)    (2018+)
```

## 2. 文本表示方法

### 2.1 词袋模型 (Bag of Words)

**原理**：将文本表示为词的集合，忽略词序，只关注词频。

**优点**：
- 简单直观
- 计算效率高
- 适合大多数分类任务

**缺点**：
- 丢失词序信息
- 高维稀疏
- 无法捕捉语义

**示例**：
```
文档1: "the cat sat on the mat"
文档2: "the dog sat on the log"

词汇表: [cat, dog, log, mat, on, sat, the]

文档1向量: [1, 0, 0, 1, 1, 1, 2]
文档2向量: [0, 1, 1, 0, 1, 1, 2]
```

### 2.2 TF-IDF (Term Frequency-Inverse Document Frequency)

**原理**：综合考虑词频和文档频率，既考虑词的重要性，又考虑词的区分能力。

**公式**：
```
TF(t, d) = 词t在文档d中出现的次数 / 文档d的总词数
IDF(t) = log(文档总数 / 包含词t的文档数)
TF-IDF(t, d) = TF(t, d) × IDF(t)
```

**核心思想**：
- **TF (词频)**：一个词在文档中出现的频率越高，它对文档越重要
- **IDF (逆文档频率)**：一个词在越少的文档中出现，它对区分文档越重要

**示例**：
```
文档1: "the cat sat on the mat"
文档2: "the dog sat on the log"

TF("the", 文档1) = 2/6 = 0.333
IDF("the") = log(2/2) = 0  # "the"出现在所有文档中
TF-IDF("the", 文档1) = 0.333 × 0 = 0

TF("cat", 文档1) = 1/6 = 0.167
IDF("cat") = log(2/1) = 0.693  # "cat"只出现在1个文档中
TF-IDF("cat", 文档1) = 0.167 × 0.693 = 0.116
```

### 2.3 N-gram

**原理**：将连续N个词作为一个特征单元，捕捉局部词序信息。

**类型**：
- **Unigram (n=1)**: 单个词 ["the", "cat", "sat"]
- **Bigram (n=2)**: 词对 ["the cat", "cat sat", "sat on"]
- **Trigram (n=3)**: 三元组 ["the cat sat", "cat sat on"]

**优点**：
- 捕捉局部词序
- 提高特征表达能力

**缺点**：
- 特征空间急剧增大
- 数据稀疏问题

## 3. 传统分类算法

### 3.1 朴素贝叶斯 (Naive Bayes)

**原理**：基于贝叶斯定理，假设特征之间条件独立。

**贝叶斯定理**：
```
P(y|X) = P(X|y) × P(y) / P(X)
```

**朴素假设**：
```
P(X|y) = P(x₁|y) × P(x₂|y) × ... × P(xₙ|y)
```

**优点**：
- 训练和预测速度快
- 适合高维数据
- 对小样本效果好
- 不容易过拟合

**缺点**：
- 特征独立性假设往往不成立
- 对特征相关性敏感

**变体**：
- **多项式朴素贝叶斯**：适合离散特征（词频、TF-IDF）
- **伯努利朴素贝叶斯**：适合二值特征
- **高斯朴素贝叶斯**：适合连续特征

### 3.2 逻辑回归 (Logistic Regression)

**原理**：使用逻辑（sigmoid）函数将线性组合映射到概率。

**公式**：
```
P(y=1|x) = sigmoid(w^T * x + b) = 1 / (1 + exp(-(w^T * x + b)))
```

**优化目标**：最小化对数损失（交叉熵）
```
L = -1/n * Σ(y * log(p) + (1-y) * log(1-p))
```

**优点**：
- 输出概率值
- 可解释性强
- 训练效率高
- 支持正则化

**缺点**：
- 线性决策边界
- 对特征工程依赖大

**多分类扩展**：
- **One-vs-Rest (OvR)**：为每个类别训练一个二分类器
- **Softmax回归**：直接建模多分类概率

### 3.3 支持向量机 (SVM)

**原理**：找到最大间隔超平面来分隔不同类别的样本。

**核心概念**：
- **超平面**：决策边界
- **间隔**：超平面到最近样本的距离
- **支持向量**：距离超平面最近的样本点
- **核技巧**：将数据映射到高维空间

**优化目标**：
```
min 1/2 ||w||^2 + C * Σξᵢ
s.t. yᵢ(w^T * xᵢ + b) >= 1 - ξᵢ
```

**核函数**：
- **线性核**：K(x, y) = x^T * y
- **多项式核**：K(x, y) = (x^T * y + c)^d
- **RBF核**：K(x, y) = exp(-γ||x - y||^2)

**优点**：
- 有效处理高维数据
- 内存效率高（只存储支持向量）
- 核技巧处理非线性问题

**缺点**：
- 对大规模数据训练慢
- 对参数和核函数选择敏感

## 4. 深度学习方法

### 4.1 TextCNN

**原理**：使用卷积神经网络捕捉文本的局部特征。

**架构**：
```
输入 → 嵌入层 → 多尺度卷积 → 最大池化 → 全连接 → 输出
```

**关键组件**：
- **嵌入层**：将词索引映射为稠密向量
- **卷积层**：使用不同大小的卷积核捕捉不同n-gram特征
- **池化层**：最大池化提取最显著特征
- **全连接层**：分类决策

**优点**：
- 并行计算效率高
- 捕捉局部特征
- 参数共享减少参数量

**缺点**：
- 无法捕捉长距离依赖
- 对词序敏感度有限

### 4.2 LSTM (Long Short-Term Memory)

**原理**：使用门控机制解决RNN的梯度消失问题，捕捉长距离依赖。

**门控机制**：
- **遗忘门**：决定丢弃哪些信息
- **输入门**：决定添加哪些新信息
- **输出门**：决定输出哪些信息

**公式**：
```
fₜ = σ(Wf · [hₜ₋₁, xₜ] + bf)  # 遗忘门
iₜ = σ(Wi · [hₜ₋₁, xₜ] + bi)  # 输入门
c̃ₜ = tanh(Wc · [hₜ₋₁, xₜ] + bc)  # 候选值
cₜ = fₜ * cₜ₋₁ + iₜ * c̃ₜ  # 更新细胞状态
oₜ = σ(Wo · [hₜ₋₁, xₜ] + bo)  # 输出门
hₜ = oₜ * tanh(cₜ)  # 输出隐藏状态
```

**优点**：
- 捕捉长距离依赖
- 解决梯度消失问题
- 适合序列数据

**缺点**：
- 训练速度慢（顺序计算）
- 参数量大

### 4.3 BiLSTM + Attention

**原理**：双向LSTM捕捉上下文信息，注意力机制聚焦重要部分。

**架构**：
```
输入 → 嵌入层 → 双向LSTM → 注意力层 → 全连接 → 输出
```

**注意力机制**：
```
eₜ = v^T * tanh(W * hₜ)  # 计算注意力分数
αₜ = softmax(eₜ)  # 归一化权重
context = Σ(αₜ * hₜ)  # 加权求和
```

**优点**：
- 双向捕捉上下文
- 注意力机制提高可解释性
- 自动聚焦重要信息

**缺点**：
- 计算复杂度高
- 参数量大

## 5. 评估指标

### 5.1 基础指标

| 指标 | 公式 | 含义 |
|------|------|------|
| 准确率 (Accuracy) | (TP+TN)/(TP+TN+FP+FN) | 整体正确率 |
| 精确率 (Precision) | TP/(TP+FP) | 预测为正的样本中真正为正的比例 |
| 召回率 (Recall) | TP/(TP+FN) | 真正为正的样本中被正确预测的比例 |
| F1分数 | 2*P*R/(P+R) | 精确率和召回率的调和平均 |

### 5.2 混淆矩阵

```
              预测为正    预测为负
实际为正        TP          FN
实际为负        FP          TN
```

### 5.3 多分类评估

- **Macro平均**：各类别指标的算术平均
- **Micro平均**：汇总所有类别后计算指标
- **Weighted平均**：按类别样本数加权平均
