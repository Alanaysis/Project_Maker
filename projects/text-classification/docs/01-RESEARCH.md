# 01 - 调研文档：文本分类

## 1. 文本分类简介

文本分类是自然语言处理（NLP）中最基础和最重要的任务之一。它的目标是将文本文档自动分配到一个或多个预定义的类别中。

### 1.1 核心思想

文本分类的核心流程：
```
文本 → 特征提取 → 分类器 → 类别
```

1. **文本预处理**：清洗、分词、去除停用词等
2. **特征提取**：将文本转换为数值特征向量
3. **分类器**：使用机器学习算法进行分类
4. **输出**：预测的类别标签

### 1.2 应用场景

| 应用场景 | 描述 | 示例 |
|----------|------|------|
| 情感分析 | 判断文本的情感倾向 | 电影评论正面/负面 |
| 垃圾邮件检测 | 识别垃圾邮件 | 邮件分类为 spam/ham |
| 新闻分类 | 将新闻分到不同类别 | 财经/科技/体育 |
| 语言识别 | 识别文本的语言 | 英文/中文/法文 |
| 主题分类 | 识别文本主题 | 论文分类到不同学科 |
| 意图识别 | 识别用户意图 | 聊天机器人意图分类 |

## 2. 文本表示方法

### 2.1 词袋模型 (Bag of Words)

最简单的文本表示方法，忽略词序，只关注词频。

**表示方式**：
```python
文档: "the cat sat on the mat"
词表: {the: 2, cat: 1, sat: 1, on: 1, mat: 1}
向量: [2, 1, 1, 1, 1]
```

**优点**：
- 简单直观
- 计算效率高

**缺点**：
- 忽略词序信息
- 无法捕捉语义相似性
- 高维稀疏向量

### 2.2 TF-IDF (Term Frequency-Inverse Document Frequency)

TF-IDF 是一种统计方法，用于评估一个词对一个文档集或一个语料库中的其中一份文档的重要程度。

**公式**：
```
TF(t, d) = 词t在文档d中出现的次数 / 文档d的总词数
IDF(t) = log(文档总数 / 包含词t的文档数)
TF-IDF(t, d) = TF(t, d) × IDF(t)
```

**核心思想**：
- **TF (词频)**：一个词在文档中出现的频率越高，它对文档越重要
- **IDF (逆文档频率)**：一个词在越少的文档中出现，它对区分文档越重要
- **TF-IDF**：综合考虑词频和文档频率，既考虑词的重要性，又考虑词的区分能力

**示例**：
```python
文档1: "the cat sat on the mat"
文档2: "the dog sat on the log"

TF("the", 文档1) = 2/6 = 0.333
IDF("the") = log(2/2) = 0  # "the"出现在所有文档中
TF-IDF("the", 文档1) = 0.333 × 0 = 0

TF("cat", 文档1) = 1/6 = 0.167
IDF("cat") = log(2/1) = 0.693  # "cat"只出现在1个文档中
TF-IDF("cat", 文档1) = 0.167 × 0.693 = 0.116
```

**优点**：
- 简单有效
- 考虑了词的区分能力
- 广泛用于信息检索和文本分类

**缺点**：
- 仍然忽略词序
- 无法捕捉语义相似性
- 高维稀疏向量

### 2.3 词嵌入 (Word Embeddings)

将词映射到低维稠密向量空间，捕捉语义相似性。

**常见方法**：
- Word2Vec
- GloVe
- FastText

**优点**：
- 捕捉语义相似性
- 低维稠密向量
- 可以利用预训练模型

**缺点**：
- 需要大量训练数据
- 训练时间长
- 无法处理未登录词（OOV）

### 2.4 预训练语言模型

使用大规模预训练模型提取文本特征。

**常见模型**：
- BERT
- GPT
- RoBERTa

**优点**：
- 强大的语义理解能力
- 可以处理上下文
- 迁移学习效果好

**缺点**：
- 计算资源需求大
- 模型复杂
- 推理速度慢

## 3. 分类算法

### 3.1 朴素贝叶斯 (Naive Bayes)

基于贝叶斯定理的分类器，假设特征之间条件独立。

**贝叶斯定理**：
```
P(y|X) = P(X|y) × P(y) / P(X)
```

**朴素假设**：
```
P(X|y) = P(x₁|y) × P(x₂|y) × ... × P(xₙ|y)
```

**优点**：
- 训练和预测速度快
- 适合高维数据
- 对小样本效果好
- 不容易过拟合

**缺点**：
- 特征独立性假设往往不成立
- 对特征相关性敏感

**变体**：
- 多项式朴素贝叶斯：适用于离散特征（词频）
- 伯努利朴素贝叶斯：适用于二值特征
- 高斯朴素贝叶斯：适用于连续特征

### 3.2 逻辑回归 (Logistic Regression)

线性模型，使用 sigmoid 函数输出概率。

**公式**：
```
P(y=1|X) = σ(wᵀX + b)
σ(z) = 1 / (1 + e⁻ᶻ)
```

**优点**：
- 简单高效
- 输出概率
- 可解释性强

**缺点**：
- 线性模型，无法捕捉非线性关系
- 对特征工程依赖大

### 3.3 支持向量机 (SVM)

寻找最大间隔超平面进行分类。

**优点**：
- 在高维空间效果好
- 对过拟合有较好的控制
- 核技巧可以处理非线性问题

**缺点**：
- 训练时间长
- 对参数敏感
- 不直接输出概率

### 3.4 随机森林 (Random Forest)

集成学习方法，组合多个决策树。

**优点**：
- 不容易过拟合
- 能处理高维数据
- 可以评估特征重要性

**缺点**：
- 模型较大
- 预测速度较慢

### 3.5 深度学习方法

使用神经网络进行文本分类。

**常见架构**：
- CNN (卷积神经网络)
- RNN (循环神经网络)
- Transformer

**优点**：
- 强大的表示能力
- 可以自动学习特征
- 在大数据集上效果好

**缺点**：
- 需要大量数据
- 计算资源需求大
- 可解释性差

## 4. 评估指标

### 4.1 准确率 (Accuracy)

```
Accuracy = 正确预测数 / 总样本数
```

**适用场景**：类别平衡的数据集

### 4.2 精确率 (Precision)

```
Precision = TP / (TP + FP)
```

**适用场景**：关注假正例的任务（如垃圾邮件检测）

### 4.3 召回率 (Recall)

```
Recall = TP / (TP + FN)
```

**适用场景**：关注假负例的任务（如疾病检测）

### 4.4 F1 分数

```
F1 = 2 × (Precision × Recall) / (Precision + Recall)
```

**适用场景**：需要平衡精确率和召回率

### 4.5 混淆矩阵

```
              预测正例    预测负例
实际正例      TP          FN
实际负例      FP          TN
```

## 5. 文本预处理

### 5.1 分词 (Tokenization)

将文本分割成词或子词。

**英文分词**：
```python
"I love NLP" → ["I", "love", "NLP"]
```

**中文分词**：
```python
"我爱自然语言处理" → ["我", "爱", "自然语言处理"]
```

### 5.2 去除停用词

移除常见但无意义的词。

**英文停用词**：
```
the, a, an, is, are, was, were, ...
```

**中文停用词**：
```
的，了，在，是，我，有，和，就，不，人，都，一，一个，上，也，很，到，说，要，去，你，会，着，没有，看，好，自己，这
```

### 5.3 词干提取 (Stemming)

将词还原为词干形式。

```python
"running" → "run"
"cats" → "cat"
"better" → "good" (词形还原)
```

### 5.4 词形还原 (Lemmatization)

将词还原为词典形式。

```python
"ran" → "run"
"better" → "good"
"mice" → "mouse"
```

## 6. 特征工程

### 6.1 N-gram 特征

考虑连续 N 个词的组合。

**Unigram**: 单个词
```python
"I love NLP" → ["I", "love", "NLP"]
```

**Bigram**: 两个连续词
```python
"I love NLP" → ["I love", "love NLP"]
```

**Trigram**: 三个连续词
```python
"I love NLP" → ["I love NLP"]
```

### 6.2 词性标注特征

使用词性信息作为特征。

```python
"I love NLP" → [("I", "PRP"), ("love", "VBP"), ("NLP", "NNP")]
```

### 6.3 命名实体特征

识别文本中的实体。

```python
"Apple released iPhone" → [("Apple", "ORG"), ("iPhone", "PRODUCT")]
```

## 7. 实践建议

### 7.1 算法选择

| 数据规模 | 推荐算法 |
|----------|----------|
| 小规模 (<1K) | 朴素贝叶斯、逻辑回归 |
| 中等规模 (1K-100K) | SVM、随机森林 |
| 大规模 (>100K) | 深度学习 |

### 7.2 特征选择

| 方法 | 适用场景 |
|------|----------|
| TF-IDF | 通用文本分类 |
| 词袋模型 | 简单任务 |
| 词嵌入 | 需要语义理解 |
| 预训练模型 | 高精度要求 |

### 7.3 常见问题

1. **类别不平衡**
   - 过采样少数类
   - 欠采样多数类
   - 使用类别权重

2. **高维稀疏特征**
   - 特征选择
   - 降维（PCA、LDA）
   - 使用正则化

3. **过拟合**
   - 增加训练数据
   - 使用正则化
   - 交叉验证
