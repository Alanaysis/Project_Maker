# 04 - 产品思维：解释器

## 1. 用户画像

### 1.1 目标用户

**主要用户**：编程语言学习者

- 想了解编程语言如何工作
- 对编译原理感兴趣
- 希望通过实践加深理解

**次要用户**：需要嵌入脚本的开发者

- 需要在Go应用中嵌入脚本能力
- 需要简单的DSL（领域特定语言）
- 需要配置和自动化脚本

### 1.2 用户需求

| 需求 | 优先级 | 说明 |
|------|--------|------|
| 理解解释器原理 | 高 | 学习词法、语法、AST、执行 |
| 代码简洁易读 | 高 | 便于学习和理解 |
| 完整的文档 | 高 | 解释设计决策和实现细节 |
| 可运行的示例 | 中 | 展示语言能力 |
| 可扩展性 | 中 | 便于添加新特性 |

## 2. 产品定位

### 2.1 核心价值

**教育优先**：这是一个学习项目，不是生产工具。

- 代码清晰 > 性能
- 可读性 > 功能完整
- 理解原理 > 实际应用

### 2.2 差异化

与其他解释器实现的区别：

| 特点 | 本项目 | 其他实现 |
|------|--------|----------|
| 语言 | Go | 多种 |
| 复杂度 | 低 | 中-高 |
| 文档 | 详细 | 简略 |
| 测试 | 完整 | 部分 |
| 示例 | 丰富 | 少量 |

## 3. 功能设计

### 3.1 核心功能

**词法分析**
- 支持所有基本Token类型
- 清晰的错误信息
- 行号和列号追踪

**语法分析**
- 递归下降解析
- 运算符优先级
- 详细的错误信息

**解释执行**
- 树遍历执行
- 变量作用域
- 函数和闭包

### 3.2 用户体验

**REPL模式**
```
>> let x = 5;
>> print x;

>> fn add(a, b) { return a + b; }
>> print add(1, 2);

>> help
```

## 4. 使用场景

### 4.1 学习场景

**场景1**：学习词法分析
```
用户阅读lexer.go，理解如何将字符转换为Token
运行lexer_test.go，验证理解
修改代码，添加新的Token类型
```

**场景2**：学习语法分析
```
用户阅读parser.go，理解递归下降
运行parser_test.go，验证理解
修改代码，添加新的语法结构
```

**场景3**：学习作用域
```
用户阅读env.go，理解环境链
运行scope相关的测试
编写自己的闭包示例
```

### 4.2 嵌入场景

**场景4**：在Go应用中嵌入脚本
```go
import "interpreter"

func main() {
    interp := interpreter.NewInterpreter()

    // 执行脚本
    err := interp.Run(`
        let result = 1 + 2;
        print result;
    `)

    if err != nil {
        log.Fatal(err)
    }
}
```

### 4.3 DSL场景

**场景5**：创建领域特定语言
```
// 配置文件DSL
let server_host = "localhost";
let server_port = 8080;
let max_connections = 100;

fn isProduction() {
    return server_host != "localhost";
}

if isProduction() {
    print "Running in production mode";
}
```

## 8. 总结

本项目的核心价值是**教育**。通过实现一个简单但完整的解释器，学习者可以：

1. 理解编程语言是如何工作的
2. 掌握词法分析、语法分析、AST等核心概念
3. 学会变量作用域和闭包
4. 获得设计和实现语言的经验

这不是一个生产级的解释器，而是一个**学习工具**。它的价值在于帮助学习者建立对编程语言实现的深入理解。
