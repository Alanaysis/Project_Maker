## 2. 梯度下降基础

### 2.1 什么是梯度下降？

梯度下降是一种迭代优化算法，用于寻找函数的最小值。

**核心思想**:
- 从一个初始点开始
- 计算当前位置的梯度（函数增长最快的方向）
- 沿梯度的反方向移动（函数下降最快的方向）
- 重复直到收敛

**数学表达**:
```
θ_{t+1} = θ_t - η * ∇J(θ_t)
```

其中:
- `θ`: 模型参数
- `η`: 学习率（步长）
- `∇J(θ)`: 损失函数的梯度

### 2.2 为什么需要梯度下降？

在机器学习中，我们需要最小化损失函数来训练模型。对于复杂模型，无法直接求解最优参数，需要使用迭代优化方法。

**应用场景**:
- 线性回归
- 逻辑回归
- 神经网络
- 各种机器学习模型

### 2.3 梯度下降的挑战

1. **学习率选择**: 太大导致发散，太小导致收敛慢
2. **局部最优**: 可能陷入局部最小值
3. **鞍点**: 在鞍点处梯度为零但不是最优
4. **病态问题**: 条件数大导致收敛慢

## 3. SGD 及其变体

### 3.1 批量梯度下降 (Batch GD)

**特点**:
- 使用全部训练样本计算梯度
- 收敛稳定
- 计算开销大

**数学表达**:
```
∇J(θ) = (1/N) * Σ_{i=1}^{N} ∇J_i(θ)
```

### 3.2 随机梯度下降 (SGD)

**特点**:
- 每次使用一个样本计算梯度
- 计算速度快
- 收敛不稳定
- 有逃离局部最优的能力

**数学表达**:
```
∇J(θ) ≈ ∇J_i(θ)
```

### 3.3 小批量梯度下降 (Mini-batch GD)

**特点**:
- 使用小批量样本计算梯度
- 结合了批量和随机梯度下降的优点
- 是深度学习中最常用的方法

**数学表达**:
```
∇J(θ) = (1/B) * Σ_{i=1}^{B} ∇J_i(θ)
```

### 3.4 动量法 (Momentum)

**核心思想**: 引入动量项来加速收敛并减少震荡

**数学表达**:
```
v_t = γ * v_{t-1} + η * ∇J(θ_t)
θ_{t+1} = θ_t - v_t
```

**优势**:
- 加速收敛
- 减少震荡
- 逃离局部最优

**关键洞察**: 动量项像一个球在山上滚动，会积累速度

### 3.5 Nesterov 加速梯度 (NAG)

**改进**: 在计算梯度前先"向前看一步"

**数学表达**:
```
v_t = γ * v_{t-1} + η * ∇J(θ_t - γ * v_{t-1})
θ_{t+1} = θ_t - v_t
```

**优势**:
- 更准确的梯度估计
- 更快的收敛速度

## 4. 自适应学习率算法

### 4.1 为什么需要自适应学习率？

不同参数可能需要不同的学习率：
- 频繁更新的参数 → 较小学习率
- 稀疏更新的参数 → 较大学习率

### 4.2 AdaGrad

**核心思想**: 为每个参数自适应调整学习率

**数学表达**:
```
G_t = G_{t-1} + (∇J(θ_t))^2
θ_{t+1} = θ_t - (η / √(G_t + ε)) * ∇J(θ_t)
```

**特点**:
- 对稀疏特征友好
- 学习率单调递减
- 可能过早停止学习

**关键洞察**: 频繁更新的参数会获得更小的学习率

### 4.3 RMSProp

**改进**: 使用移动平均代替累积平方和

**数学表达**:
```
G_t = β * G_{t-1} + (1-β) * (∇J(θ_t))^2
θ_{t+1} = θ_t - (η / √(G_t + ε)) * ∇J(θ_t)
```

**优势**:
- 解决 AdaGrad 学习率过早衰减的问题
- 适合非平稳目标函数

### 4.4 Adam (Adaptive Moment Estimation)

**核心思想**: 结合动量法和 RMSProp 的优点

**数学表达**:
```
m_t = β1 * m_{t-1} + (1-β1) * ∇J(θ_t)      # 一阶矩估计
v_t = β2 * v_{t-1} + (1-β2) * (∇J(θ_t))^2    # 二阶矩估计
m̂_t = m_t / (1 - β1^t)                        # 偏差修正
v̂_t = v_t / (1 - β2^t)                        # 偏差修正
θ_{t+1} = θ_t - (η / (√v̂_t + ε)) * m̂_t
```

**关键组件**:
1. **一阶矩 (m_t)**: 梯度的指数移动平均（动量）
2. **二阶矩 (v_t)**: 梯度平方的指数移动平均（自适应学习率）
3. **偏差修正**: 修正初始化偏差

**优势**:
- 自适应学习率
- 偏差修正机制
- 适用于大多数问题
- 对超参数不敏感

### 4.5 AdamW (Adam with Weight Decay)

**改进**: 解耦权重衰减与梯度更新

**数学表达**:
```
θ_{t+1} = θ_t - η * (m̂_t / (√v̂_t + ε) + λ * θ_t)
```

**优势**:
- 更好的泛化性能
- 现代深度学习的默认选择

**关键洞察**: L2 正则化和权重衰减在 Adam 中不等价

## 5. 学习率调度

### 5.1 为什么需要学习率调度？

- 训练初期：需要较大学习率快速收敛
- 训练后期：需要较小学习率精细调整
- 逃离局部最优：需要动态调整学习率

### 5.2 阶梯衰减 (Step Decay)

**策略**: 每隔固定 epoch 降低学习率

**数学表达**:
```
lr = lr_0 * gamma^(epoch // step_size)
```

**特点**:
- 简单有效
- 需要预先设定衰减点

### 5.3 指数衰减 (Exponential Decay)

**策略**: 每个 epoch 乘以衰减因子

**数学表达**:
```
lr = lr_0 * gamma^epoch
```

**特点**:
- 平滑衰减
- 连续下降

### 5.4 余弦退火 (Cosine Annealing)

**策略**: 使用余弦函数平滑地降低学习率

**数学表达**:
```
lr = lr_min + 0.5 * (lr_max - lr_min) * (1 + cos(π * t / T))
```

**特点**:
- 平滑的周期性衰减
- 有助于跳出局部最优
- 可以实现"重启"

### 5.5 Warmup

**策略**: 训练初期使用较小学习率，逐渐增加到目标学习率

**优势**:
- 稳定训练初期
- 防止梯度爆炸
- 通常与其他调度策略结合使用

## 6. 测试函数

### 6.1 二次函数 (Quadratic Function)

**数学表达**:
```
f(x, y) = a * (x - x0)^2 + b * (y - y0)^2
```

**特点**:
- 凸函数
- 有唯一最小值
- 梯度是线性的
- 适合验证算法基本功能

### 6.2 Rosenbrock 函数

**数学表达**:
```
f(x, y) = (a - x)^2 + b * (y - x^2)^2
```

**特点**:
- 非凸函数
- 有狭长的"峡谷"
- 全局最小值在 (1, 1)
- 优化难度大

**学习价值**: 测试算法处理峡谷型地形的能力

### 6.3 Himmelblau 函数

**数学表达**:
```
f(x, y) = (x^2 + y - 11)^2 + (x + y^2 - 7)^2
```

**特点**:
- 非凸函数
- 有四个全局最小值
- 所有最小值 f(x) = 0
- 适合测试多起点优化

## 7. 可视化分析

### 7.1 等高线图

**用途**: 展示函数地形和优化轨迹

**观察要点**:
- 等高线密度 → 梯度大小
- 轨迹方向 → 优化路径

### 7.2 收敛曲线

**用途**: 展示优化过程的收敛情况

**观察要点**:
- 下降速度 → 收敛快慢
- 最终值 → 优化质量
- 震荡程度 → 稳定性

### 7.3 梯度范数曲线

**用途**: 展示梯度的变化趋势

**观察要点**:
- 梯度大小 → 收敛状态
- 下降速度 → 优化效率
- 稳定性 → 算法鲁棒性

## 8. 实验观察

### 8.1 优化器对比

**实验**: 在不同函数上对比各种优化器

**观察**:
1. **SGD**: 收敛稳定但慢
2. **Momentum**: 收敛更快，震荡减少
3. **AdaGrad**: 对稀疏特征友好
4. **RMSProp**: 解决 AdaGrad 的衰减问题
5. **Adam**: 通常收敛最快

**结论**: 没有万能的优化器，需要根据问题选择

### 8.2 学习率影响

**实验**: 测试不同学习率的影响

**观察**:
1. 学习率太大 → 发散
2. 学习率太小 → 收敛慢
3. 学习率适中 → 收敛稳定

**结论**: 学习率是最重要的超参数之一

### 8.3 动量系数影响

**实验**: 测试不同动量系数的影响

**观察**:
1. 动量为 0 → 无动量效果
2. 动量太小 → 加速不明显
3. 动量太大 → 可能震荡
4. 动量适中 → 最佳效果

**结论**: 动量系数通常设为 0.9

## 9. 关键概念

### 9.1 收敛性

**定义**: 算法是否能找到最优解

**影响因素**:
- 学习率
- 梯度质量
- 函数性质
- 初始化

### 9.2 收敛速度

**定义**: 达到最优解所需的迭代次数

**影响因素**:
- 算法选择
- 学习率调度
- 函数条件数
- 动量系数

### 9.3 数值稳定性

**定义**: 算法是否会产生数值错误

**问题**:
- 梯度爆炸
- 梯度消失
- 数值溢出

**解决方案**:
- 梯度裁剪
- 学习率限制
- 数值稳定实现

### 9.4 泛化性能

**定义**: 在未见数据上的表现

**影响因素**:
- 正则化
- 权重衰减
- 优化算法选择

## 10. 实践建议

### 10.1 选择优化器

**场景**:
- 简单问题 → SGD + Momentum
- 稀疏数据 → AdaGrad, Adam
- 快速原型 → Adam
- 追求泛化 → SGD + 学习率调度

### 10.2 调整超参数

**学习率**:
- Adam: 1e-3 到 1e-4
- SGD: 1e-1 到 1e-3

**动量系数**:
- Momentum: 0.9
- Adam β1: 0.9
- Adam β2: 0.999

### 10.3 学习率调度

**策略**:
- 余弦退火: 通用
- Warmup + 余弦退火: 大模型
- 阶梯衰减: 简单问题

### 10.4 调试技巧

**问题**:
- 不收敛 → 降低学习率
- 收敛慢 → 增加学习率或使用动量
- 震荡 → 使用动量或降低学习率
- 梯度爆炸 → 梯度裁剪

## 15. 常见问题

### 15.1 为什么不收敛？

**可能原因**:
- 学习率太大
- 梯度计算错误
- 函数有多个局部最优

**解决方案**:
- 降低学习率
- 检查梯度计算
- 使用更好的优化器

### 15.2 为什么收敛慢？

**可能原因**:
- 学习率太小
- 函数条件数大
- 没有使用动量

**解决方案**:
- 增加学习率
- 使用自适应学习率
- 添加动量

### 15.3 为什么有震荡？

**可能原因**:
- 学习率太大
- 没有使用动量
- 梯度噪声大

**解决方案**:
- 降低学习率
- 使用动量
- 使用更大的批量
