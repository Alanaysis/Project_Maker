# 需求分析

## 1. 项目概述

### 1.1 项目目标
实现一个从零开始的量化交易系统，支持：
- 策略回测
- 实盘交易模拟
- 风险管理
- 历史数据加载

### 1.2 学习目标
- 理解量化交易策略的设计与实现
- 掌握回测框架的事件驱动架构
- 学会风险管理和订单管理的工程实现

---

## 2. 用户画像

### 2.1 目标用户

**用户类型 1：量化交易初学者**
- 背景：有 Python 基础，对量化交易感兴趣
- 需求：学习量化交易的基本概念和实现
- 痛点：缺乏系统的学习路径和实践项目

**用户类型 2：金融从业者**
- 背景：有金融知识，想学习编程实现
- 需求：将理论知识转化为可运行的代码
- 痛点：技术实现与金融理论的结合

**用户类型 3：程序员**
- 背景：有编程基础，想进入金融科技领域
- 需求：了解量化交易的技术架构
- 痛点：缺乏金融领域知识

---

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 数据管理
- **历史数据加载**
  - 支持 CSV 文件加载
  - 支持 DataFrame 直接传入
  - 数据格式标准化
  - 数据验证和清洗

- **模拟数据生成**
  - 几何布朗运动（GBM）
  - 多标的相关数据
  - 趋势数据（上升/下降/震荡）
  - 可重现的随机种子

#### 3.1.2 策略引擎
- **策略抽象**
  - 统一的策略接口
  - 策略生命周期管理
  - 参数化配置

- **内置策略**
  - 均线交叉策略
  - 动量策略
  - 可扩展的策略框架

#### 3.1.3 回测引擎
- **事件驱动架构**
  - 事件总线
  - 事件类型系统
  - 事件处理流程

- **回测流程**
  - 数据推送
  - 信号生成
  - 订单执行
  - 成交确认

#### 3.1.4 订单管理
- **订单类型**
  - 市价单
  - 限价单

- **订单状态**
  - 待处理
  - 已提交
  - 已成交
  - 已取消
  - 已拒绝

#### 3.1.5 风险管理
- **风险规则**
  - 最大仓位限制
  - 最大订单规模
  - 止损规则
  - 最大回撤限制
  - 现金储备要求

- **风险检查**
  - 订单前检查
  - 实时监控
  - 风险事件记录

#### 3.1.6 投资组合管理
- **持仓管理**
  - 持仓跟踪
  - 成本计算
  - 盈亏计算

- **资金管理**
  - 现金管理
  - 权益曲线
  - 交易记录

---

### 3.2 非功能需求

#### 3.2.1 性能需求
- 回测速度：1 年日线数据 < 1 秒
- 内存使用：支持 10 年历史数据
- 响应时间：事件处理 < 1ms

#### 3.2.2 可扩展性
- 支持自定义策略
- 支持自定义风险规则
- 支持自定义数据源

#### 3.2.3 可维护性
- 清晰的代码结构
- 完善的文档
- 单元测试覆盖

#### 3.2.4 易用性
- 简洁的 API
- 详细的示例
- 友好的错误提示

## 4. 用例分析

### 4.1 用例 1：运行简单回测

**前置条件**：
- 用户已安装 Python 和依赖库
- 用户有历史数据或使用模拟数据

**基本流程**：
1. 用户创建回测引擎
2. 用户加载历史数据
3. 用户创建策略
4. 用户添加策略到引擎
5. 用户运行回测
6. 系统输出回测结果

**后置条件**：
- 生成回测报告
- 记录权益曲线
- 记录交易历史

---

### 4.2 用例 2：自定义策略

**前置条件**：
- 用户理解策略接口
- 用户有策略逻辑

**基本流程**：
1. 用户继承策略基类
2. 用户实现 `on_init` 方法
3. 用户实现 `on_market_data` 方法
4. 用户定义信号生成逻辑
5. 用户测试策略

**后置条件**：
- 策略可正常使用
- 策略可添加到引擎

---

### 4.3 用例 3：风险控制

**前置条件**：
- 用户理解风险规则
- 用户有风险管理需求

**基本流程**：
1. 用户创建风险管理器
2. 用户添加风险规则
3. 用户配置规则参数
4. 系统自动检查订单
5. 系统记录风险事件

**后置条件**：
- 违规订单被拒绝
- 风险事件被记录

---

## 5. 数据需求

### 5.1 输入数据

#### 5.1.1 历史数据格式

| 字段 | 类型 | 说明 |
|------|------|------|
| date | datetime | 交易日期 |
| symbol | string | 标的代码 |
| open | float | 开盘价 |
| high | float | 最高价 |
| low | float | 最低价 |
| close | float | 收盘价 |
| volume | int | 成交量 |

#### 5.1.2 策略参数

| 参数 | 类型 | 说明 |
|------|------|------|
| name | string | 策略名称 |
| symbols | list | 交易标的 |
| params | dict | 策略参数 |

### 5.2 输出数据

#### 5.2.1 回测结果

| 指标 | 类型 | 说明 |
|------|------|------|
| initial_capital | float | 初始资金 |
| final_equity | float | 最终权益 |
| total_return | float | 总收益率 |
| annualized_return | float | 年化收益率 |
| max_drawdown | float | 最大回撤 |
| total_trades | int | 总交易次数 |
| win_rate | float | 胜率 |

#### 5.2.2 权益曲线

| 字段 | 类型 | 说明 |
|------|------|------|
| timestamp | datetime | 时间戳 |
| equity | float | 总权益 |
| cash | float | 现金 |
| positions_value | float | 持仓市值 |

## 6. 约束条件

### 6.1 技术约束
- 使用 Python 3.10+
- 从零实现，不依赖量化框架
- 使用 pandas/numpy 进行数据处理

### 6.2 业务约束
- 仅支持日线数据
- 仅支持股票市场
- 不支持做空

### 6.3 时间约束
- 项目周期：2 周
- 核心功能优先
