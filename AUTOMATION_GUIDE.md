# 🤖 自动化执行指南

## 概述

本指南说明如何使用自动化工作流端到端地执行愿望单中的所有任务。

## 工作流程

```
┌─────────────────────────────────────────────────────────────┐
│                    自动化执行流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 启动工作流                                               │
│     └── 解析 WISHLIST.md                                     │
│                                                              │
│  2. 并行执行任务                                             │
│     ├── 任务1 → 子代理实现                                   │
│     ├── 任务2 → 子代理实现                                   │
│     └── 任务3 → 子代理实现                                   │
│                                                              │
│  3. 错误处理                                                 │
│     ├── 成功 → 记录日志，继续                                 │
│     └── 失败 → 记录错误，继续下一个                           │
│                                                              │
│  4. 第一轮完成                                               │
│     └── 生成统计报告                                         │
│                                                              │
│  5. 第二轮开始                                               │
│     └── 重新调研，抛弃旧选型                                 │
│                                                              │
│  6. 全部完成                                                 │
│     └── 生成最终报告                                         │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 使用方法

### 方法1: 使用 Claude Code 工作流（推荐）

在 Claude Code 中执行：

```bash
# 启动工作流
/workflow auto-project-generator
```

工作流会自动：
1. 解析愿望单
2. 并行启动子代理
3. 实现每个项目
4. 记录执行日志
5. 进行第二轮迭代

### 方法2: 手动执行

如果你想手动控制执行过程：

```bash
# 1. 查看愿望单
cat WISHLIST.md

# 2. 选择一个任务，例如 "HTTP 协议基础"
# 告诉 Claude: "请实现 HTTP 协议基础 这个项目"

# 3. Claude 会启动子代理实现项目

# 4. 完成后，继续下一个任务
```

### 方法3: 使用 Python 脚本

```bash
# 运行 Python 脚本（用于解析和统计）
python3 auto_runner.py
```

## 配置说明

在 `workflow.js` 中可以调整配置：

```javascript
const CONFIG = {
  maxRounds: 2,           // 最大迭代轮数
  maxConcurrency: 3,      // 最大并行任务数
  retryOnFailure: false,  // 失败时是否重试
  logLevel: 'standard'    // 日志级别: 'simple', 'standard', 'detailed'
}
```

## 错误处理

### 常见错误及处理

| 错误类型 | 处理方式 |
|----------|----------|
| 子代理超时 | 记录错误，继续下一个任务 |
| 代码生成失败 | 记录错误，继续下一个任务 |
| 测试失败 | 记录警告，标记为部分完成 |
| 依赖安装失败 | 记录错误，继续下一个任务 |

### 错误恢复

如果某一轮执行中断：

1. 查看日志文件了解执行状态
2. 手动修复问题
3. 重新运行工作流

## 最佳实践

1. **首次运行建议串行**
   - 先用小并发测试
   - 确认流程正确后再增加并发

2. **监控资源使用**
   - 并行任务会消耗较多资源
   - 根据机器性能调整并发数

3. **定期检查日志**
   - 及时发现和处理问题
   - 了解执行进度

4. **备份重要项目**
   - 完成的项目建议备份
   - 避免重复生成

## 常见问题

### Q: 工作流执行到一半中断了怎么办？

A: 查看 `round_X_log.json` 了解已完成的任务，手动修复问题后重新运行。

### Q: 如何只执行部分任务？

A: 修改 `WISHLIST.md`，只保留要执行的任务，或手动选择任务执行。

### Q: 如何调整执行速度？

A: 减少 `maxConcurrency` 可以降低资源消耗，但会增加总执行时间。

### Q: 第二轮会重新生成所有项目吗？

A: 是的，第二轮会重新调研并生成新版本，旧版本会被覆盖。

## 监控执行

### 实时查看进度

```bash
# 查看最新日志
tail -f round_1_log.json

# 统计完成情况
jq '.tasks | map(select(.status == "completed")) | length' round_1_log.json
```

### 查看错误

```bash
# 查看失败的任务
jq '.tasks | map(select(.status == "failed"))' round_1_log.json
```
